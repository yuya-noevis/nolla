# SNSコンテンツネタ発掘リサーチ

## ミッション

Nollaの潜在ユーザー層（発達障害・知的障害・ダウン症・グレーゾーン・境界知能の子を持つ親）をフォロワーとして集めるためのSNS投稿に使える「コンテンツネタ」を多角的に発掘する。
プロダクトローンチ前フェーズ。目的は**コミュニティ形成とブランディングの同時達成**。

## 対象プラットフォーム

- **X** — 共感・啓発・議論喚起
- **Instagram** — ビジュアル・カルーセル・保護者コミュニティ
- **TikTok** — 短尺・インパクト・感覚刺激
- **YouTube Shorts** — 解説・アニメーション・ハウツー

## ネタカテゴリ（A〜F）

全カテゴリに最低5本ずつ、合計30本以上のアイデアを出すこと。偏りチェック必須。

### A. 情報・知識系
- 発達特性の解説（親が「うちの子これだ」と思うもの）
- 制度・手続きの実用情報（受給者証、放デイ選び、就学相談など）
- 療育テクニックの紹介
- 研究・エビデンスの噛み砕き紹介

### B. 共感・感情系
- 親が「わかる…」と思わずシェアする日常あるあるネタ
- 定型発達の子と比較されたときの気持ち
- 「頑張ってるのに報われない」感覚への寄り添い
- 親同士の「一人じゃない」という連帯感を生むもの

### C. 癒し・ポジティブ系
- 子どもの小さな成長エピソード（「初めて○○できた」系）
- 障害があるからこその愛おしさ・面白さ
- 親自身のセルフケア・リフレッシュ系

### D. エンタメ・クリエイティブ系
- **子ども向けアニメーション動画（AI生成含む）**
  - 発達特性のあるキャラクターが主人公
  - 言葉を使わない / 少ないセリフでストーリーが成立する構造
  - 感覚過敏・こだわり・パニックをネガティブに描かず個性として描く
- 感覚刺激系コンテンツ（ASMR、スライム、色水遊び → 感覚統合と接続）
- ゲーム実況・知育アプリレビュー的フォーマット

### E. 驚き・データ系
- 「発達障害の子どもは○○万人」のようなインパクトのある数字
- 海外の先進的な支援事例
- 意外な研究結果・統計

### F. ストーリー・ナラティブ系
- 成長の物語（Before/After）
- 「あのとき諦めなくてよかった」系
- 当事者（成人した元・障害児）の回顧

## NGルール

- ❌ 投稿者本人が出演していることが前提のネタ（再現性がないため）
- ❌ 「障害児」をNGワードとして使用するコンテンツ案
- ❌ 悲壮感・お涙頂戴に振り切ったコンテンツ案（共感はOK、消費的な感動ポルノはNG）
- ❌ 特定の障害名をセンセーショナルに煽る構成
- ❌ 数値の根拠がないまま「バズる」と主張すること
- ❌ 医学的に不正確・エビデンスのない療法の推奨

## ワークフロー

### Step 1: テーマ確認

ユーザーに確認する（1〜2問だけ）：
- 「どのプラットフォーム向け？（全部でもOK）」
- 「特に狙いたいテーマや切り口は？」

「おまかせ」と言われたら、直近トレンドから判断してスキップ。

### Step 2: リサーチ設計

テーマを以下の**4つの調査軸**に分解する：

| 調査軸 | 目的 | 主な手法 |
|--------|------|---------|
| **① Outlier検出** | 予想外にバズっている投稿の発見 | キーワード検索→アカウント平均比でフィルタ |
| **② 競合・参考アカウント分析** | 伸びているアカウントの勝ちパターン抽出 | 関連アカウント調査→人気投稿のパターン分析 |
| **③ 隣接ジャンル着想** | 障害系以外からのネタの種 | 育児・知育・癒し・キャラクター系の横断調査 |
| **④ 保護者の生の声・最新動向** | 悩み・ニーズ・制度変更の把握 | 掲示板・Q&A・ニュース検索 |

### Step 3: 並列リサーチ実行

**4つのサブエージェントを並列起動する：**

---

**Agent 1: Outlier検出**

以下のロジックで「予想外にバズっている投稿・動画」を探す：

1. 関連キーワードで各プラットフォームの投稿を検索
2. 各アカウントの平均再生数・エンゲージメントを推定
3. **平均の3倍以上**のパフォーマンスを出している投稿をOutlierとして特定
4. **登録者/フォロワー5万人以下の小規模アカウント**を優先
5. 大手は参考として記録するがOutlier検出の主戦場は小規模アカウント

検索キーワード例：
- 日本語：「発達障害 子育て」「ADHD あるある」「療育 ママ」「感覚過敏」「グレーゾーン 育児」
- 英語：「autism parenting」「ADHD mom hack」「sensory play kids」「neurodivergent child」

API制限がある場合はWeb検索ベースの手動調査で代替し、推定根拠を明記すること。

---

**Agent 2: 競合・参考アカウント分析**

1. 関連ジャンルで伸びているアカウントを各プラットフォームから5〜10個特定
2. 各アカウントの人気投稿トップ3〜5を調査
3. **刺さっているパターン**を抽出（フォーマット・トーン・フック・投稿頻度）
4. Nollaが参考にできる要素を整理

---

**Agent 3: 隣接ジャンル着想**

障害・発達系に限定せず、以下の領域を横断調査：

- 子ども向けアニメーション / キッズ動画チャンネル
- 育児・子育て全般のバズコンテンツ
- 感覚遊び / 知育 / モンテッソーリ系
- メンタルヘルス・セルフケア（親向け）
- ペット・癒し系（「言葉を持たない存在への共感」構造の類似性）
- キャラクターコンテンツ（例：ちいかわ — 発語がない・不器用・マイペースなキャラが共感を呼ぶ構造）

特に重点的に探すもの：
- 言葉を使わない / 発語が少ないキャラクターが人気のコンテンツ
- 不器用・マイペース・空気が読めないキャラが愛されている構造
- 「障害」と明示せずにニューロダイバーシティ的価値観を自然に体現しているコンテンツ
- 子ども向けアニメーション生成系アカウント（YouTube等で伸びているもの）

---

**Agent 4: 保護者の生の声・最新動向**

1. 掲示板・Q&A（知恵袋、発言小町、Reddit等）で保護者の悩み・関心を収集
2. 直近3ヶ月のニュース・研究・制度変更を調査
3. 「障害」を直接テーマにしなくても親が共感するコンテンツの種を探す

---

### Step 4: 統合・ネタ生成

各エージェントの結果を統合し、以下を生成する：

**採用基準（3つ以上該当で採用）：**
- [ ] 保護者の共感を得られるか
- [ ] 「知らなかった」と思わせる新しさがあるか
- [ ] シェア・保存したくなる実用性があるか
- [ ] 議論や反応を生む論点があるか
- [ ] Nollaのミッション（支援の空白を埋める）と繋がるか
- [ ] 直接「障害」をテーマにしなくても親が共感できるか

**多角性チェック：**
情報提供系（A）に偏っていないか自己チェックし、A〜Fの全カテゴリに最低5本を確保する。

### Step 5: アウトプット生成

以下の4つのセクションを含むレポートを `outputs/sns_research_[日付].md` に出力する。

---

#### ① Outlierコンテンツリスト（最低30件）

| # | プラットフォーム | アカウント名 | フォロワー数 | 該当投稿URL | 再生/いいね数 | チャンネル平均比 | なぜバズったか（仮説） | ネタカテゴリ（A〜F） | Nolla親和性（高/中/低） |
|---|--------------|------------|------------|------------|-------------|---------------|-------------------|-------------------|---------------------|

ルール：
- チャンネル平均比 = その投稿のパフォーマンス ÷ アカウントの直近投稿の平均値
- 「なぜバズったか」は必ず仮説を記述（「不明」は禁止。推測でよいので根拠とターゲットの心理的ニーズを含めること）
- 登録者5万人以下のアカウントを優先

---

#### ② ネタカテゴリ別 投稿アイデア（各カテゴリ最低5本、合計30本以上）

各アイデアに以下を記載：

| 項目 | 内容 |
|------|------|
| タイトル/フック | 最初の1秒で引く見出し・ワード |
| フォーマット | 静止画 / カルーセル / ショート動画 / アニメーション / テキスト |
| 想定プラットフォーム | X / Instagram / TikTok / YouTube Shorts（複数可） |
| コンテンツ概要 | 100字以内 |
| なぜ刺さるか | ターゲットのどの感情・ニーズに応えるか |
| 再現性 | AI量産可能性（高/中/低） |
| Outlierからの着想元 | ①のリストの何番から着想したか（該当あれば） |

---

#### ③ 隣接ジャンル発見レポート（最低10件）

障害・発達系の外から見つけた参考事例をリストアップし、Nollaの文脈でどう転用できるかを記述。

---

#### ④ プラットフォーム別 最適フォーマット分析

| プラットフォーム | 推奨フォーマット | 推奨尺 | 推奨投稿頻度 | 伸びやすいネタカテゴリ |
|--------------|--------------|-------|------------|-------------------|

---

### Step 6: 納品

1. フルレポートを `outputs/sns_research_[日付].md` に保存
2. チャットにはエグゼクティブサマリー（注目トレンド + 各セクションのハイライト3点ずつ）を出力

## 品質ルール

1. **数値グレード必須** — すべての数値に `[実数]` / `[推定]` / `[観察]` / `[仮説]` のいずれかを付与
2. **「〜と思われる」禁止** — 根拠を示すか `[仮説]` と明記
3. **ソース必須** — ネタの元になった情報源を必ず記載
4. **鮮度重視** — 直近3ヶ月以内の情報を優先
5. **クロスチェック** — 1ソースだけの情報は「未検証」とラベル付け
6. **事実と意見を分離** — 研究結果と個人の体験談は明確に区別
7. **Outlierの「なぜバズったか」にターゲットの心理的ニーズを必ず含める**
8. **投稿アイデアには「これを見て親はどう感じるか」を必ず記載**
9. **多角性チェック** — A〜Fの全カテゴリに最低5本あるか自己確認
10. **ギャップの明示** — 見つからなかった情報は「データ不足」と正直に書く

```
/sns-research 隣接ジャンルからの着想を重点的に
```
