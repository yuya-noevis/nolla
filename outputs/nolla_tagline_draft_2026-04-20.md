## Yuyaの要件

### 目的
「Nollaとは何のサービスか」を抽象的でも把握できる**サービス定義フレーズ**。キャッチコピーではない。

### 避ける表現
- 「AI xxx」（AIは当然の前提なので古臭い）
- 「xxxエージェント」（流行りに乗っかりすぎ）
- 「xxx Copilot」「xxx Companion」（一巡している）
- Duolingo比喩は使わない
- 「多様性」という言葉（表面的に聞こえる）

### 反映すべき思想
- 子どもの課題は一人ずつ違う → 一人ずつに合わせた支援
- 多機能を取りこぼさず包摂できる抽象度
- AIの進化を活かす姿勢

## Nolla 5案

### 【案1】型1 "The X for every Y"（最有力）
参考：Decagon "The AI concierge for every customer"

```
JP：すべての子どもに、家庭療育チームを
EN：The home therapy team for every child
```
**強み**：既存職業（療育チーム）を借用、"every child"で一人ずつ違う思想、ユニコーン多数採用の鉄板型

### 【案2】型2 身体拡張メタファー
参考：OpenEvidence "brain extender for doctors"

```
JP：家庭を、発達の専門家に変える
EN：Turning every home into a developmental expert
```
**強み**：家庭のエンパワメント表現、AIを一切使わない、療育士不足の社会問題にも刺さる

### 【案3】型3 抽象名詞×空間限定
参考：Abridge "Intelligence at the point of conversation"

```
JP：遊びの瞬間に、発達を。
EN：Development at the point of play
```
**強み**：Nollaのゲーム性を最も端的に表現、短く、抽象度が高くて陳腐化しにくい

### 【案4】型4 "AIを隠す"定義
参考：Suno "music company"／Granola "Notepad"

```
JP：子どもひとりひとりの、成長プログラム
EN：A growth program, built for every child
```
**強み**：AI疲れ回避、"プログラム"で機能包摂、親しみやすい日本語

### 【案5】型1+型4ハイブリッド（複合型、最強候補）
参考：Decagon + Suno

```
JP：すべての子の、家庭専属 発達プログラム
EN：The personalized developmental program for every child's home
```
**強み**：案1の「every child」×案4の「AI不使用」を両立、全要素（一人ずつ違う・家庭ベース・発達支援）を包摂

---

## 推奨：**案1 or 案5**

### 案1 を推す理由
- Decagon, Granola, Cognition Labsなど**複数のメガ級スタートアップが採用する最強テンプレ**
- 「家庭療育チーム」という既存職業借用で、非エンジニア投資家・保護者にもゼロ秒で刺さる
- 日本の療育給付事業・放課後デイの代替として具体的ポジション

### 案5 を推す理由
- 案1 + 案4のハイブリッドで、AI不使用＋every childを両立
- 「プログラム」がソフトウェアと療育プログラム両方を意味するダブルミーニング
- 最も長期的に陳腐化しない
