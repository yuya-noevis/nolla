# Crystal Cavern ゲーム設計 — 専門家壁打ち統合 v1

**日付**: 2026-04-02

---

## 3専門家の意見から見えた「現状の致命的な間違い」

| 間違い | なぜダメか |
|--------|----------|
| ソートゲーム / 落下パズル | Minecraftの「掘る→集める→作る」の中毒性ゼロ。単調作業 |
| 「ルールが変わりました」と通知 | ASD児にパニックを引き起こす。ルール変更は環境の変化で自然に気づかせる |
| 時間制限・タイムプレッシャー | ASD児の認知能力が大幅低下。Masterでも避けるべき |
| ランキング・比較 | ASD児は相対評価で離脱する。絶対評価（昨日の自分との比較）のみ |
| 失敗判定の可視化（× マーク） | 過度な感情反応→セッション中断 |

---

## 3つの視点の統合：最適なゲーム設計の方向性

### 認知科学者の知見
- **構造化タスク + 自由探索の層化モデル**が最も効果的
  - 層1: 明確なゴール付きミッション（認知的柔軟性を鍛える）
  - 層2: ミッション達成後の自由な建築・探索（動機維持）
- 正答率70-80%を維持する適応的難易度調整
- 週3回以上、1セッション20-30分、8-12週で効果

### ASD専門家の知見
- **コアループは予測可能、マイクロレベルで変動がある**のが飽きない秘訣
- ルール変更は「環境が変わる = ルールが変わる」形で自然に埋め込む
- ASD児は**帰納的にルールを発見する**（教えるのではなく気づかせる）
- 知的障害児は「採掘→配置」の繰り返しだけで十分に没頭できる

### ゲームデザイナーの知見
- ワールドビルダー型（島を設計、テーマが変わる）がMinecraftとの親和性最高
- 「構造の中の自由」がASD児に刺さる核
- 深掘り要素（効率追求、美学追求）が長期継続のカギ

## 統合コンセプト：「Crystal Cavern」再設計

### コアコンセプト
**洞窟を探索しながら、クリスタルを採掘し、自分だけの鉱山基地を建設する**

### コアループ（Minecraftの掘る→集める→作る を踏襲）

```
① 採掘する（タップで岩を掘る → クリスタルが出る）
    ↓
② 集める（クリスタルをインベントリに貯める）
    ↓
③ 配置する（集めたクリスタルで基地を装飾・拡張する）
    ↓
④ 新しいエリアへ進む（環境が変わる → 採掘ルールが変わる）
    ↓
① に戻る
```

### 認知的柔軟性はどこに埋め込まれるか

**「採掘ルール」がエリア（バイオーム）によって自然に変わる**

| エリア | 採掘ルール | 認知的柔軟性の訓練 |
|--------|----------|-----------------|
| 氷の洞窟 | 青いクリスタルだけが採掘可能 | 基準: 色 |
| 溶岩の深層 | ダイヤ型のクリスタルだけが採掘可能 | 基準が色→形に切り替わる |
| クリスタルの森 | 大きいクリスタルだけが採掘可能 | 基準が形→大きさに切り替わる |
| 闇の領域 | 光っているクリスタルだけが採掘可能 | 新しい基準の導入 |

**重要**: ルール変更は「告知」しない。エリアに入ると環境が変わり、子どもが**自分で試して、正解を発見する**。間違えても罰はない（「このクリスタルは固くて掘れないな」程度のフィードバック）。

### なぜこれがMinecraftの6つのメカニズムを活かしているか

| メカニズム | 本ゲームでの実現 |
|-----------|----------------|
| 予測可能性 | コアループ（掘る→集める→配置）は常に同じ |
| 構造内の創造的自由 | 集めたクリスタルで基地を自由に装飾 |
| 低ストレス | 間違えても「掘れないだけ」。失敗判定なし |
| 繰り返しの快感 | タップで岩を掘る反復アクション |
| 深掘り可能 | レアクリスタル収集、基地デザインの追求 |
| 自分のペース | 時間制限なし。好きなだけ同じエリアにいられる |

### 3段階の難易度（同じ世界・同じループ）

| | Explorer（中度ID） | Miner（軽度ID） | Master（IDなし） |
|---|---|---|---|
| 採掘操作 | タップのみ | タップ + 長押し | タップ + 長押し + スワイプ |
| エリア数 | 3 | 5 | 8+ |
| ルール変化 | 色のみ（2色） | 色→形（自然遷移） | 色→形→大きさ→複合 |
| 基地建設 | 自動配置（集めたら勝手に飾られる） | 選択配置（2択から選ぶ） | 自由配置（どこにでも置ける） |
| ルール発見 | 視覚ヒントあり（掘れるものが光る） | ヒント少なめ | ヒントなし（完全に自力発見） |

### 「飽きない」設計（ASD専門家の知見）

| 要素 | 実装 |
|------|------|
| コアループの予測可能性 | 毎回同じ「掘る→集める→配置」リズム |
| マイクロレベルの変動性 | クリスタルの位置・種類は毎回ランダム |
| セッション間の新しさ | 新エリア、レアクリスタル、基地拡張 |
| 深掘り要素 | クリスタル図鑑（コレクション）、基地スコア |

### 避けるべき要素（チェックリスト）

- [ ] 時間制限 → なし
- [ ] ランキング・他者比較 → なし（昨日の自分との比較のみ）
- [ ] 「失敗」表示 → なし（「掘れない」は失敗ではなく情報）
- [ ] 突然の音・フラッシュ → なし（全て予測可能に）
- [ ] 言語での説明 → 最小限（ビジュアルで伝える）
