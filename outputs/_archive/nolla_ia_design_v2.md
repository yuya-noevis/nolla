## 3. 未決定事項の解決

### 3A. ゲーム終了のトリガー

**判断: 試行数ベースの自動終了 + 任意離脱**

```
1セッション = 10-15試行（ゲームにより異なる）
  ├ 全試行完了 → 報酬まとめ画面へ自動遷移
  ├ 途中離脱 → 戻るボタン → 「やめる？」確認 → ホームへ
  │             ※途中までの分のスターは獲得済み
  └ 連続で「もう1回」→ 新セッション開始（セッション間に報酬まとめを挟む）
```

根拠:
- 仕様書: 「時間ではなく試行数で区切る」（アセスメントの設計原則と同じ）
- ASD児は「あとどれくらい？」がわからないと不安→試行数のプログレスバーで残りを可視化
- 1セッション10-15試行は研究ベースの適正量（CogMed: 15試行/セッション）
- ただし強制終了はしない。子どもが「もう1回」を選べば継続可能

### 3B. 将来のエリア内ナビ（20ゲーム時）

**判断: 縦スクロールのカード一覧（グリッドではない）**

```
建物に入る → エリア画面:
  ┌─────────────────────────┐
  │ ← [エリア名アイコン]     │
  │                         │
  │  ┌─────┐  ┌─────┐      │
  │  │Game1│  │Game2│      │  ← 2列グリッド
  │  │[img]│  │[img]│      │     各カード: アイコン+進捗ドット
  │  └─────┘  └─────┘      │     アンロック済み=カラー
  │  ┌─────┐  ┌─────┐      │     未アンロック=シルエット+鍵
  │  │Game3│  │Game4│      │
  │  │[img]│  │ 🔒 │      │
  │  └─────┘  └─────┘      │
  │       ↕ スクロール       │
  └─────────────────────────┘
```

根拠:
- 2列グリッド: 1画面に4-6個表示で5要素以下ルールを守る
- カードは大きく（最小64×64px）、アイコン+色で識別（文字不要）
- 縦スクロールはタップ操作と干渉しない（横スワイプはカルーセルで使用中）

### 3C. ゲームのアンロック方式

**判断: 最初のゲームはアンロック済み。以降は前のゲームをN回プレイで自動アンロック**

```
Game 1: 最初からプレイ可能
Game 2: Game 1を3セッション完了でアンロック
Game 3: Game 2を3セッション完了でアンロック
...
```

根拠:
- 全部最初から開放→選択麻痺（ASD児は選択肢が多いとパニック）
- 順番に1つずつ開放→予測可能性が高い（次に何がアンロックされるかわかる）
- 3セッションは「慣れ」と「飽きない」のバランス
- 親が設定で全アンロックに切替可能（自由度の確保）

### 3D. 名前収集のタイミング

**判断: 仕様書v3準拠。初回セッション完了後**

根拠:
- 仕様書v3: 「オンボーディングでは聞かない。初回セッション完了後」
- Zigpoll調査: 質問削除でサインアップ率+40%
- 初回は「遊ぶ楽しさ」を体験させることが最優先

### 3H. 複数子ども対応

**判断: MVPでは対応しない。将来対応として設計は考慮**

根拠: MVP段階で兄弟対応は開発コスト過大。ただしDB設計で子どもプロフィールを親アカウントに紐づける構造にしておけば、後から追加可能。

### 3I. プッシュ通知

**判断: MVPでは実装しない**

根拠: プッシュ通知は習慣化に有効だが、ASD児の親にとっては「また催促か」と負の印象リスク。まずアプリ自体の魅力で継続利用を実現し、通知は利用データを見てから判断。

### 3J. オフライン対応

**判断: ゲームプレイはオフライン対応。データ同期はオンライン復帰時**

根拠: 療育施設や移動中のWi-Fiなし環境での利用を想定。ゲームのコア体験がネットワーク依存では使い物にならない。

## 6. カルーセル詳細設計

### 6A. 建物の並び順（MVP）

```
位置1: なかまわけ（認知的柔軟性） — コテージ、オレンジ系背景
位置2: おぼえて（ワーキングメモリ） — 天文台、青紫系背景
位置3: みつけて（注意制御） — ツリーハウス、緑系背景
```

- 並び順はアセスメント結果で最適化（推奨ゲームが先頭）
- ただし一度決まった並び順は変更しない（予測可能性）
- 端でループしない（行き止まり = 予測可能）

### 6B. 各建物の識別要素（文字不要）

| 建物 | 建物タイプ | 背景色 | アイコン |
|------|-----------|--------|---------|
| なかまわけ | コテージ/納屋 | オレンジ〜暖色 | 果物/分類シンボル |
| おぼえて | 天文台 | 青紫〜夕暮れ | 星/望遠鏡 |
| みつけて | ツリーハウス | 緑〜晴天 | 虫眼鏡/双眼鏡 |

### 6C. インタラクション

- 矢印ボタン（64×64px、常時表示）で切替
- スワイプでも切替可能（補助操作）
- 建物タップ → scale 1.05拡大（200ms）→ ドアが開くアニメ → エリア画面へ
- 下部にドットインジケータ（現在位置）
