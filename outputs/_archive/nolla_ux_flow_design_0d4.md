IA設計v3で定義された21画面の基本フローを、0-D-1〜0-D-3/0-D-5の確定事項と整合させ、インタラクション詳細・エッジケース・トランジションを定義する。

## 1. 初回起動フロー（更新版）

IA v3のS3「お子さま情報入力」を0-D-3のアセスメント設計に合わせて再構成。

```
[S1] スプラッシュ（3秒自動遷移）
  ↓
[S2] アカウント作成
  メール or Apple/Google認証
  ↓
[S3-a] お子さまの基本情報
  ニックネーム + 生年月日
  ↓
[S3-b] 診断選択（複数選択可）
  [ASD] [ADHD] [知的障害] [LD] [ダウン症] [境界知能] [その他]
  ↓ 条件分岐
  知的障害選択時 → 程度選択: [重度] [中度] [軽度]
  LD選択時 → 種類選択: [読字] [書字] [算数] [その他]
  ↓
[S3-c] 発語レベル
  [発語なし] [Yes/Noのみ] [単語のみ] [話すが伝わりにくい] [会話可能]
  ↓
[S3-d] 認知スキル確認（Yes/No、3問）
  Q1: 色を区別できますか？
  Q2: 形を見分けられますか？
  Q3: 複数ステップの指示を実行できますか？
  ↓
[S4] 同意取得
  個人情報 + 行動データ取得の同意
  ↓
[S5] 端末引き渡し
  「お子さまにわたしてください」+ 横向き図解
  ↓ ── 操作者交代（保護者→子ども）──
[M1] ホーム画面
  ナビキャラが最初の建物へ誘導
  ↓
[M3] チュートリアル（初回のみ、文字なし）
  ↓
[M4] 初回ゲームプレイ
  ↓
[R1] 報酬演出
  ↓
[M1] ホーム
```

**S3-a〜S3-dの画面設計原則**:
- 1画面1質問（Otsimo方式）。複数質問を詰め込まない
- プログレスバー表示（上部に進捗を可視化）
- 戻るボタンあり（前の質問に戻れる）
- 条件分岐は自動（知的障害を選んだら程度選択が自動表示）
- 所要時間: 2-3分

## 2. 通常利用フロー（メインループ）

### 2.1 基本フロー

```
[M1] ホーム画面（カルーセル）
  │
  ├── 建物タップ ──→ [M4] ゲームプレイ
  │                    │
  │                    ├── ラウンド終了 → Staircase調整（0-D-2）
  │                    │   → 次ラウンド開始（背景色微変化のみ）
  │                    │
  │                    ├── セッション終了 → [R1] 報酬演出
  │                    │                    ├── [▶] もう1回 → [M4]
  │                    │                    └── [🏠] ホーム → [M1]
  │                    │
  │                    ├── 60分連続 → [U1] 休憩提案
  │                    │               ├── [▶] 続ける → [M4]
  │                    │               └── [🏠] おしまい → [M1]
  │                    │
  │                    └── [← 戻る] → 途中離脱確認 → [M1]
  │
  ├── [マイルーム] ──→ [R2] マイルーム統合画面
  │                    ├── おへやタブ: 配置+タッチ反応
  │                    ├── ショップタブ: スター交換
  │                    └── ずかんタブ: コレクション一覧
  │
  └── [🔒] ──→ [P1] PIN認証 → 親向け画面
```

### 2.2 前提能力による画面出し分け

0-D-3のアセスメント結果に基づき、カルーセルの建物を動的に制御する。

```
【ゲーム提供ルール】

  Q1（色）= No かつ Q2（形）= No
    → 分類ゲーム（ソーティング小屋）をカルーセルから非表示
    → カルーセルは3建物: 神経衰弱 / 視覚探索 / Corsi Block

  Q1 = No かつ Q2 = Yes
    → 分類ゲームは表示するが、色基準スキップ（形から開始）

  Q1 = Yes かつ Q2 = No
    → 分類ゲームは表示するが、形基準スキップ（色と大きさのみ）

  Q1 = Yes かつ Q2 = Yes
    → 全ゲーム通常提供

【非表示ゲームの復帰条件】
  保護者が設定画面（P5）からアセスメント回答を更新した場合
  → 即座にカルーセルに建物が追加される
  → チュートリアル（M3）から開始

【カルーセルの並び順】
  初回: アセスメント結果で推奨ゲームが先頭
  2回目以降: 初回と同じ順番を維持（予測可能性）
  順番は変更しない（ASD児の不安防止）
```

  │   ↓

  │   ├── Trial 1 → フィードバック（200ms以内）
  │   ├── Trial 2 → フィードバック

## 4. 報酬フロー

### 4.1 セッション完了 → 報酬演出（R1）

```
[M4] セッション完了
  ↓ 自動遷移（0.5秒のフェード）
[R1] 報酬演出画面
  │
  ├── Phase 1: スター獲得アニメーション（1.5秒）
  │   スターが飛んで集まる + 数字カウントアップ
  │   音: 柔らかいチャイム
  │   振動: 短い振動 15ms
  │
  ├── Phase 2: 新アイテム獲得判定（条件付き）
  │   累計スターが閾値に達した場合のみ:
  │   「?シルエット → アイテム出現」アニメーション（2秒）
  │   獲得しなかった場合: Phase 2スキップ
  │
  └── Phase 3: 選択肢表示（常時）
      [▶ アイコン] = もう1回（同じゲーム）
      [🏠 アイコン] = ホームへ
      文字なし。アイコンのみ。
```

### 4.2 スター獲得量

```
【要調整】
  具体的な獲得量は、ベンチマークゲーム（Minecraft、どうぶつの森、Roblox等）の
  報酬設計をリサーチしてから決定する。現時点では構造のみ定義。

【獲得タイミング（構造）】
  - 正解ごと: 基本スター
  - セッション完了: 完了ボーナス
  - デイリー初回: デイリーボーナス
  - 連続プレイ: コンボボーナス（日数に応じて増加）

【表示方法】
  子ども側: 獲得時に「+数字」が浮かんで消える（400ms）。内訳・計算過程は表示しない
  保護者側（P2）: 今日の合計獲得スター数のみ表示
```

## 7. エッジケース

### 7.1 途中離脱

```
[M4] ゲームプレイ中に [← 戻る] タップ
  ↓
確認画面（アイコンのみ、文字なし）:
  [▶ アイコン] = 続ける
  [🏠 アイコン] = おしまい
  ↓
おしまい選択:
  完了済み試行のスターは獲得済み（没収しない）
  セッションデータは保存（途中離脱フラグ付き）
  パラメータ状態も保存（次回復元用）
  [M1] ホームに戻る
```

### 7.2 ブランク復帰（14日以上）

```
アプリ起動
  ↓
前回プレイから14日以上経過?
  Yes → ナビキャラが「おかえり」演出（1.5秒）
       → [M1] ホーム（通常通り）
       → ゲーム開始時に反転カウンタリセット（0-D-2、大きなステップで再探索）
  No → 通常起動
```

### 7.3 全ゲーム非表示

```
アセスメントで Q1=No, Q2=No の場合:
  分類ゲームが非表示。残り3ゲームのみ。

全ゲームが非表示になることはない:
  神経衰弱: 前提能力なし → ペア数2、類似度0%で提供（最低難度）
  視覚探索: 前提能力チェック不要（適応型で対応）
  Corsi Block: Q3=Noでもシーケンス長2で提供（最低難度）
  → 最低でも3ゲームは常に提供される
```

### 7.4 ネットワーク切断

```
ゲームプレイ中にオフラインになった場合:
  ゲームプレイは継続可能（端末側で全処理）
  データはローカルに一時保存
  オンライン復帰時にSupabaseに同期
  同期完了まではP3（成長の記録）に「同期中」表示
```

### 7.5 アプリ強制終了

```
ゲームプレイ中にアプリが強制終了した場合:
  最後のラウンド完了時点までのデータは保存済み
  現ラウンドの未完了試行データは破棄
  次回起動時: [M1] ホームから通常開始
  パラメータは最後の保存値から復元
```

## 8. トランジション・アニメーション

### 8.1 画面遷移

| 遷移 | アニメーション | 時間 |
|------|-------------|------|
| M1 → M4（ゲーム開始） | 建物がズームイン → ゲーム画面にクロスフェード | 0.8秒 |
| M4 → R1（報酬演出） | フェードアウト → フェードイン | 0.5秒 |
| R1 → M1（ホームへ） | スライド右→左 | 0.4秒 |
| R1 → M4（もう1回） | フェード | 0.3秒 |
| M1 → R2（マイルーム） | スライド下→上 | 0.5秒 |
| どの画面 → M1（戻る） | スライド右→左 | 0.4秒 |
| カルーセル操作 | スワイプ or 矢印タップ → スライド | 0.3秒 |

### 8.2 ゲーム内アニメーション

| イベント | アニメーション | 時間 |
|---------|-------------|------|
| 正解フィードバック | 緑の光輪 + scale 1.1 + 星パーティクル3-5個 | 300ms + 500ms |
| エラーレス Stage 1 | 要素が元の位置に静かに戻る | 300ms |
| エラーレス Stage 2 | 正解選択肢が柔らかく脈動（pulse 1回） | 500ms |
| エラーレス Stage 3 | 正解に光の輪 + ナビキャラ指さし | 800ms |
| ラウンド間の難度変更 | 背景の色味が微妙に変化 | 500ms |
| スター獲得 | +1表示が浮かんで消える | 400ms |

### 8.3 禁止アニメーション

- 自動再生の連続アニメーション
- フラッシュ・点滅
- 画面全体の赤色変化
- 3秒以上のアニメーション（報酬演出のPhase 1除く）
- スクロールを要する画面遷移

## 9. カルーセル操作の詳細

```
【操作方法】
  矢印ボタン（←→）をタップ: 1つ隣の建物に移動
  スワイプ（左右）: 1つ隣の建物に移動（矢印と同じ動作）
  ※ゲーム内操作のスワイプは禁止（0-D-1）だが、カルーセルナビゲーションは別。両対応

【カルーセルの挙動】
  端でループしない（左端で←を押しても何も起きない）
  建物タップ: ゲーム開始（M3 or M4へ）
  ドットインジケータ: 現在位置を表示（下部中央）

【建物の視覚状態】
  プレイ可能: 建物が明るく表示
  非表示（前提能力不足）: カルーセルから完全に除外
  チュートリアル未完了: 建物に「?」アイコン（初回のみ）
```

  - N値（連続成功数）のみ新しいIQ帯に応じて変更

## 11. 0-D-1〜0-D-5との整合性チェック

| 確定事項 | 0-D-4での反映 |
|---------|-------------|
| タップのみ（0-D-1） | ゲーム内操作はタップのみ。カルーセルのみスワイプ+矢印の両対応 |
| 文字なし（0-D-1） | ゲーム内・報酬演出・休憩提案は全てアイコンのみ |
| エラーレス3段階（0-D-1） | M4内のフィードバックとして定義済み |
| 可変ステップサイズ（0-D-2） | ラウンド間の背景微変化でのみ視覚化 |
| 初期配置アルゴリズム（0-D-2） | 初回セッションで自動実行。UIへの影響なし |
| 前提能力チェック（0-D-3） | カルーセルからの建物非表示で反映 |
| プロフィール完了率（0-D-3） | P5設定画面上部に表示 |
