# Nolla 初回アセスメント設計 — リサーチレポート

**リサーチ実施日**: 2026年4月3日  
**対象**: ASD + 軽度〜重度知的障害児（3-18歳）向けアプリの初回アセスメント（オンボーディング）設計

---

## エグゼクティブサマリー

本リサーチは、グローバルなエビデンスと国内法制度を踏まえた初回アセスメント設計の基礎を提供する。以下が核となるインサイト：

1. **保護者質問は最小化が必須**: 一般的なオンボーディング離脱率は60-80%、フォーム長は離脱の27%を占める。タイムボックス目安は2分以内。
2. **実技アセスメント（ゲーム化）**: 1500ゲームを提供するJade NDや100ゲーム以上を持つOtsimoなど、既存アプリは認知データをリアルタイム収集・分析している。
3. **障害名を聞かない選択肢が有効**: 行動特性に基づく「Behavioral Phenotyping」アプローチ（Nature Medicine等で発表）により、診断名を避けて行動指標で分類可能。
4. **要配慮個人情報としての取扱**: 日本の個人情報保護法では障害情報は「要配慮個人情報」。原則として明示的な同意が必須。
5. **AAC利用経験の把握が重要**: 約25-30%の自閉症児は「minimally verbal」。既存AAC使用状況の把握がコンテンツ出し分けの鍵。

## 1. 保護者への質問設計（プロフィール設定）

### 1.1 必須項目と選択肢

#### A. 基本情報（3-5問、60秒で完了）
- 子どもの年齢（スライダーまたはドロップダウン）
- **診断名は「聞かない」選択肢を含める**
  - 「診断名がある場合のみ教えてください」（任意）
  - 代わりに「当てはまることはありますか？」形式で行動特性リスト提示
  - 例）「お子さんはよく同じ動作を繰り返しますか？」「音に敏感ですか？」

#### B. コミュニケーション能力（3-4問）
- **発語レベル**（単一選択）
  - 発語なし / 単語のみ / 二語文（2-3語） / 短い文章 / 会話できる
- **聞いて理解できることはありますか？**（複数選択）
  - 簡単な指示（「来て」）
  - 文で理解できる
  - 文脈で理解している様子
  - よくわからない
- **AAC（文字盤、タッチ音声など）の使用経験**
  - 使用していない / 試したことがある / 毎日使用している
  - 使用している場合、ツール名（任意記入）

#### C. 療育歴・支援実績（2-3問）
- **受けている支援の種類**（複数選択）
  - 特別支援学級 / 特別支援学校 / 療育施設 / ST（言語聴覚士）/ OT（作業療法士）/ ABA / その他 / 受けていない
- **学んでいる / 得意なことは？**（自由記入、オプション）
  - アプリ側で学習履歴より「既知スキル」を推測する際のヒント

#### D. 困りごと（生活における課題）（2問、パーソナライゼーション用）
- **日常で工夫していることはありますか？**（複数選択、最大3つ）
  - コミュニケーション補助 / 視覚支援（スケジュール等） / 感覚遊び / 身辺自立スキル / その他
- **アプリに期待することは？**（複数選択、最大3つ）
  - コミュニケーション練習 / 学習スキル支援 / 遊び・楽しむこと / 親の困りごと相談 / その他

**合計: 13-17問、想定所要時間: 3-5分**

### 1.2 個人情報保護法への対応

**要配慮個人情報の扱い**:
- 日本の改正個人情報保護法（2022年改正）では、障害情報は「要配慮個人情報」に分類
- 取得には原則として明示的な同意が必須（オプトアウト不可）
- **実装**: チェックボックス「このデータは専門家による学習支援をより正確にするために利用させていただきます。ご同意ください」を必須化

**GDPR対応（グローバル展開時）**:
- EU GDPR Art. 8: 16歳以上は本人、16歳未満は保護者同意が必須
- 日本は13歳以上が本人同意可だが、保護者同意を推奨

参照:
- [個人情報保護委員会 個人情報の保護に関する法律についてのガイドライン](https://www.ppc.go.jp/personalinfo/legal/guidelines_tsusoku/)
- [GDPR-K: Children's Data and Parental Consent](https://www.clarip.com/data-privacy/gdpr-child-consent/)

## 2. 子ども向けの実技アセスメント（ゲーム形式）

### 2.1 既存ソリューションの学習

#### Jade ND（中東発、グローバル160,000+ダウンロード）
- **仕組み**: 1,500個のゲームがプレイデータをリアルタイム収集
- **測定項目**: 認知スキル、注意力、動機づけ、衝動性、運動パターン
- **出力**: 教育的プログレスレポート、発達予測
- **強み**: AI駆動で個別化学習計画を自動生成

参照: [Jade ND Official](https://www.jadend.tech/en)

#### Otsimo（トルコ発、グローバル展開）
- **仕組み**: 100以上のゲームを17カテゴリに分類
- **初期評価**: アカウント開設直後に「developmental test」実施
- **パーソナライゼーション**: テスト結果に基づき自動的に「学習パス」を生成
- **柔軟性**: ユーザーが難度・ゲーム選択を手動調整可能

参照: [Otsimo Special Education User Guide](https://otsimo.com/en/user-guide-otsimo-special-education/)

### 2.2 測定対象スキルと実装案

**Nollaで実装すべき領域（認知負荷を考慮）**:

| スキル | 測定方法 | 実施時間 | 文字/音声指示 |
|--------|---------|---------|------------|
| **タップ反応速度** | ボタンをタップする速度 | 30秒 | 視覚指示のみ |
| **視覚的マッチング** | 絵・色・形のマッチング | 60秒 | 視覚指示のみ |
| **聴覚理解** | 簡単な指示（「触って」「押して」）への反応 | 45秒 | 音声指示 |
| **持続的注意** | パターン認識ゲーム | 60秒 | 視覚指示のみ |
| **報酬反応** | ゲーム内での喜び・モチベーション表現 | 全体を通じて観察 | なし |

**合計所要時間: 3-4分**

### 2.3 「テスト感」を出さない設計原則

参照: [Gamification in Mobile Apps for Children With Disabilities: Scoping Review](https://pmc.ncbi.nlm.nih.gov/articles/PMC11415723/)

- **ゲーム化要素**: デジタルペット・ファンタジーテーマ、複数のインセンティブレイヤー
- **達成感の可視化**: スター・ポイント・レベルアップを即座にフィードバック
- **中断可能性**: 「飽きたら後でもいいよ」メッセージで心理的負担を低減
- **マルチモーダルフィードバック**: 視覚 + 音声 + 振動

## 3. アセスメントのUX設計

### 3.1 フォーム長と離脱率

**グローバルデータ**:
- オンボーディング形式の全体離脱率: 60-80%
- フォーム長が27%の離脱を占める
- **タイムボックス**: 120秒を超えるとユーザーが放棄する傾向

参照: [150 Online Form Statistics](https://www.feathery.io/blog/online-form-statistics)

**Nolla実装案**:
- **Phase 1（保護者質問）**: 2-3分で完了
- **Phase 2（子ども実技）**: 3-4分で完了
- **合計**: 5-7分（タイムボックスは8分に設定）

### 3.2 保護者への結果フィードバック

**避けるべき表現**:
- 「発達遅滞」「できない」などのネガティブラベル
- 診断名の確定的な提示

**推奨フレーム**:
```
「お子さんの現在の得意なことと、さらに伸ばせることが見えました」

得意なこと:
- 視覚的な情報処理が得意
- タップ反応が速い

練習するといいこと:
- 音声指示の理解
- 注意の持続

↓
このプロフィールに基づき、お子さんに最適なゲームが自動選択されます
```

**公開すべき情報**:
- スキルプロフィール（レーダーチャート等で視覚化）
- 推奨難度レベル（初級 / 中級 / 上級）
- 個別化学習パスの内容

## 4. パーソナライゼーションの仕組み

### 4.1 アセスメント結果 → ゲーム難度マッピング

**実装例**:

```
発語レベル → AAC/ST コンテンツの出し分け
┌─────────────────┬──────────────────────────┐
│ 発語なし        │ AAC練習コンテンツ多重 50% │
│ 単語のみ        │ 単語拡張、二語文誘導 60%  │
│ 二語文～        │ 文法・会話練習 70%         │
└─────────────────┴──────────────────────────┘

年齢 + 認知スキル → ゲーム難度
┌──────┬───────┬──────┬──────┐
│年齢 3│スキル │難度  │説明   │
├──────┼───────┼──────┼──────┤
│   3-5│低     │星1   │単純選択│
│      │中     │星2-3 │複数選択│
│      │高     │星4-5 │推論   │
└──────┴───────┴──────┴──────┘

聴覚理解度 → インターフェース調整
┌──────────────────┬────────────────────┐
│ 音声指示NG       │ 完全ビジュアル指示  │
│ 音声+視覚OK      │ 両方提供            │
│ 聴覚優位         │ 音声 + テキスト表示 │
└──────────────────┴────────────────────┘
```

### 4.2 継続的な再評価

**タイミング**:
- **初回**: オンボーディング時
- **1ヶ月後**: 10ゲーム達成時
- **3ヶ月ごと**: 定期的な難度調整
- **親の申告**: 「うちの子が得意になったこと」の記入フォーム

**測定指標**:
- ゲーム完了率（完全達成 vs. 途中離脱）
- 応答時間の短縮化傾向
- 難度別成功率

参照: [Speech Therapy App Adaptive Difficulty](https://constanttherapyhealth.com/constant-therapy/)

## 5. 「行動による分類」アプローチ（診断名非依存）

### 5.1 Behavioral Phenotyping の科学的根拠

Nature Medicine等の最新研究で、**コンピュテーショナル行動分析**により診断名を用いずに個人を分類可能であることが示されている。

参照: [Early detection of autism using digital behavioral phenotyping - Nature Medicine](https://www.nature.com/articles/s41591-023-02574-3)

**強み**:
1. 主観性が低い（医師の経験に依存しない）
2. スケーラブル（大規模スクリーニング可能）
3. 個人への心理的負担が少ない（「診断」と感じさせない）

### 5.2 実装：質問紙の「行動リスト」化

**避けるべき質問**:
```
❌ 「お子さんはASDですか？」
❌ 「診断名は何ですか？」
```

**推奨する質問**:
```
✅ 「あてはまることを選んでください（複数選択OK）」
  - ○ よく同じ動作を繰り返す
  - ○ 音や光に敏感に反応する
  - ○ 新しい環境では不安そう
  - ○ 興味が限定的（好きなものに集中）
  - ○ 指示を聞いて理解しにくい時がある
```

**後段（レコメンデーション）**:
```
これらの特性は多くのお子さんに見られます。
お子さんに合わせて、このアプリが個別化された学習を提供します。
```

## 6. AAC（拡張代替コミュニケーション）との連携

### 6.1 現状と課題

**スコープ**:
- 自閉症児の25-30%が「minimally verbal」（最小限の音声言語）
- AAC導入により、さらなる音声言語発達を促進（「AAC = 音声言語発達を阻害する」は誤解）

参照:
- [The role of augmentative and alternative communication for children with autism - PMC](https://pmc.ncbi.nlm.nih.gov/articles/PMC5036660/)
- [Augmentative and Alternative Communication Assessment Protocol - ASHA/LSHSS](https://pubs.asha.org/doi/abs/10.1044/2024_LSHSS-23-00127)

### 6.2 オンボーディングでのAAC把握

**質問の入れ方**:

```
「コミュニケーション方法について」

お子さんが現在使用している、または試したことがあるのは？
（複数選択OK）

□ 音声言語のみ
□ 身振り・ジェスチャー
□ 文字盤・PEC（絵カード交換式）
□ タブレット音声アプリ（VOCA）
□ 手話
□ その他

「使用中のツール名」（あれば記入）:
例）PECS、JAcsille、Proloquo4Text等
```

### 6.3 Nolla側のコンテンツ出し分け

```
AAC使用 → AAC強化コンテンツ（50-70%）
  │├─ 語彙拡張（カテゴリ別）
  │├─ 文字盤操作スキル向上
  │└─ 音声出力フィードバック活用

使用経験あり → 復習 + 新規導入サポート（30-40%）
  │└─「使ってみようか」という優しい促進

未使用 → 聴覚理解・表現活動を通じた準備（20-30%）
  └─ 家庭で親がAAC導入検討時のガイド提供
```

## 7. UIの段階的調整（発語レベル・年齢別）

### 7.1 ボタンサイズとタップ領域

| 対象層 | 推奨 | 理由 |
|--------|------|------|
| 3-5歳（低認知） | 80-100px | 誤タップが少ない |
| 6-10歳（中程度） | 60-80px | 学習進行に対応 |
| 11-18歳 | 40-60px | 成人並みUI許容 |

### 7.2 画面の情報量

```
低認知・幼年層:
├─ 要素数 2-3個のみ
├─ アニメーション: 派手・即座フィードバック
└─ テキスト: なし（アイコン + 音声）

中程度:
├─ 要素数 4-6個
├─ テキスト: 1-2行の簡潔な説明
└─ 指示: 音声 + 視覚

高認知・思春期:
├─ 通常のUI許容
├─ 読む内容が増加可
└─ 自律的進行が可能
```

## 8. プライバシーとセンシティブ情報の配慮

### 8.1 保護者心理の尊重

**設計上のポイント**:

1. **フレーミングの工夫**
   - ❌ 「障害の程度は？」
   - ✅ 「お子さんの今の様子を教えてください」

2. **任意項目の明示**
   - 診断名、詳細な困りごとは「オプション」に
   - 「完全な個別化には詳細情報が役立ちますが、省略もできます」

3. **データ開示・同意の透明性**
   - 「このデータは何に使うのか」を親が理解できる言語で説明
   - 定期的に「データの活用方法」を通知

### 8.2 データ保護体制

**必須実装**:
- 要配慮個人情報の暗号化保存（エンドツーエンド）
- アクセスログの記録
- 定期的な第三者監査（セキュリティ診断）
- 利用者がデータ削除をリクエスト可能な仕組み

参照: [GDPR Recital 38 - Special Protection of Children's Personal Data](https://gdpr-info.eu/recitals/no-38/)
