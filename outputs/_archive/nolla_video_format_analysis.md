## 1. 子ども向けバズ動画フォーマット分析

### 1.1 構成パターン（秒数配分とフック）

#### 基本構造

| セクション | 秒数 | 役割 |
|-----------|------|------|
| フック（導入） | 0〜3秒 | 視聴継続を決定づける最重要パート |
| 展開 | 3〜20秒 | メインコンテンツ（学び・驚き・物語） |
| オチ/ループ | 20〜30秒 | 満足感 + リプレイ誘導 |

- **最適な動画長**: YouTube Shorts 15〜35秒、TikTok 15〜30秒、Instagram Reels 15〜45秒
- **完了率の目安**: 20秒以下のショートで90〜100%の視聴完了率を目指すのが理想
- **50〜60%の離脱が最初の3秒以内に発生**するため、フックの設計が最も重要

#### 子ども向けフックの鉄則

```
3秒ビジュアルフック + わかりやすい色 + 学びの要素
```

- 「見たことないもの」「不思議な状況」を最初のフレームに出す
- テキストフックは3〜7語（子ども向けはさらに短く、ほぼビジュアルのみ）
- **ループ構造**: 終わりが始まりにつながる設計で繰り返し再生を促す

#### 子ども向けで人気のコンテンツ構成

1. **「1つだけ教える」型**: 30秒以内に1つの明確な学びを完結させる
2. **スラップスティック型**: Oddbods等。セリフ少なめ・体を使ったユーモア
3. **歌・チャンツ型**: 繰り返しの歌＋アニメーションで記憶定着
4. **変身・ビフォーアフター型**: 色が変わる、形が変わる等の視覚的変化
5. **ダンスチャレンジ型**: 真似しやすい動きで参加感を促す

### 1.2 テンポ・リズム

#### カット割り

| コンテンツタイプ | カット間隔 | 備考 |
|----------------|----------|------|
| 一般ショート動画（バイラル） | 2〜4秒/カット | 脳が新規性を求めるため高速 |
| Cocomelon型（高刺激） | 1〜3秒/カット | 過刺激の指摘あり |
| Ms. Rachel型（低刺激） | 5〜10秒/カット | ゆっくり、間を取る |
| **Nolla推奨** | **4〜6秒/カット** | **高刺激と低刺激の中間** |

#### BGMのBPM

- 一般的な子ども向けコンテンツ: 100〜130 BPM（テンポよく、かつ速すぎない）
- 落ち着いた学習コンテンツ: 80〜100 BPM
- ダンス・アクティブ系: 120〜140 BPM
- **Nolla推奨**: 90〜110 BPM（やや落ち着いたテンポで予測しやすいリズム）

#### 繰り返しパターン

- Cocomelon/Pinkfong等の成功チャンネルは「繰り返し」を多用
- 同じメロディ・フレーズを2〜3回繰り返す構成が定番
- ASD児は繰り返しパターンを好む傾向が研究で確認されており、この設計は発達支援と相性が良い

### 1.3 色使い・ビジュアル

#### 主流の色使い傾向

| 要素 | バズ動画の傾向 | 備考 |
|------|-------------|------|
| 色数 | メイン3〜4色 | シンプルなパレットが主流 |
| 彩度 | 高彩度（鮮やか） | 乳幼児は彩度の高い色に反応 |
| コントラスト | 高コントラスト | 背景とキャラクターの明確な区別 |
| 背景 | シンプル・単色・グラデーション | 情報量を絞る |

#### 要注意ポイント

- Cocomelon等の超高彩度＋高速カットの組み合わせは「過刺激」との批判がある
- 2011年の研究: わずか9分の高速テンポ番組の視聴で、子どもの実行機能（集中力・自己制御）が一時的に低下
- **「鮮やかだがギラギラしない」バランスが重要**

### 1.4 音声・音楽

#### ナレーション

- **Ms. Rachel方式**: ゆっくり、はっきり、間を取る。子どもが言葉を繰り返す「間」を設ける
- **Cocomelon方式**: 歌中心、ナレーションは最小限
- 効果音: 動きに合わせた短い効果音（ポン、キラキラ等）が集中を維持

#### 音量設計

- 急激な音量変化を避ける（感覚過敏への配慮）
- BGMとナレーションの音量バランス: BGMはナレーションの50〜60%程度
- 研究結果: 音量とエコーを下げた環境でASD児の集中力が向上

### 1.5 キャラクターデザイン

#### バズ動画で成功しているキャラクターの共通点

| 要素 | 傾向 |
|------|------|
| 頭身比 | 2〜3頭身（頭が大きい） |
| 表情 | シンプルで大きい目・口 |
| 輪郭線 | 太くはっきり |
| 色数 | キャラ1体あたり3〜4色 |
| 動き | 全身を使った大きな動き |
| 感情表現 | 誇張された表情変化 |

## 2. 日本 vs US のデザイン・配色差分析

### 2.1 キャラクターデザインの違い

| 要素 | US（カートゥーン調） | 日本（かわいい系） |
|------|-------------------|-----------------|
| 頭身比 | 2〜4頭身（バラつきあり） | 2〜3頭身（丸みが強い） |
| 目のサイズ | 相対的に小さめ | 大きくキラキラ |
| 輪郭線 | 太く黒い線 | やや細め、色付き線も |
| 体型 | デフォルメが極端（手足が長い等） | 丸くころんとした体型 |
| 表情 | 大げさ・コミカル | 穏やか・柔らか |
| テクスチャ | フラットカラー | グラデーション・質感あり |

#### US の代表例

- **Cocomelon**: 3Dアニメ、高彩度、丸顔キャラ。全世界1.6億登録者
- **Blippi**: 実写＋アニメ、原色の衣装（青・オレンジ）、ハイテンション
- **Oddbods**: セリフなしスラップスティック、ビビッドカラーの丸いキャラ

#### 日本の代表例

- **アンパンマン**: 赤・黄・茶の原色系。乳幼児が認識しやすい色で設計
- **しまじろう**: 黄色ベース、穏やかな色調。教育的で保護者の信頼が厚い
- **Eテレキャラクター群（ワンワン・ガラピコ等）**: 明るいが落ち着いた色、NHK基準の丁寧な設計

### 2.2 配色の傾向

| 要素 | US | 日本 |
|------|------|------|
| 基本トーン | 原色・高彩度（赤・青・黄） | 中彩度〜やや高彩度 |
| 背景色 | 鮮やかなグラデーション | 白・パステル・自然色 |
| アクセント | ネオンカラーも使う | 暖色系のアクセント |
| 全体の印象 | 「派手・楽しい・エネルギッシュ」 | 「優しい・温かい・安心」 |
| 色の文化的意味 | 白 = 純粋、赤 = 危険/情熱 | 白 = 清浄（弔事にも）、赤 = 祝い/元気 |

#### 日本の伝統的な色彩感覚

- 日本の伝統色は自然・季節・風習に由来（桜色、若草色、山吹色など）
- 中間色・ニュアンスのある色が好まれる傾向
- 子ども向けでも「品の良さ」が重視される

#### ただし乳幼児向けは共通点も多い

- 乳児は赤・黄・オレンジ・白を好む（視覚発達の段階）
- 生後4ヶ月で色の区別が可能になり、彩度の高い色に惹かれる
- **0〜2歳向けは日米ともに原色寄りが有効。3歳以降で文化差が顕著になる**

### 2.3 ナレーション・声のトーン

## 3. 発達特性のある子どもへの視覚・聴覚設計ガイドライン

### 3.1 感覚過敏への配慮

#### 避けるべき要素

| カテゴリ | 避けるべき要素 | 理由 |
|---------|-------------|------|
| 視覚 | フラッシュ・点滅効果 | 光感受性てんかんのリスク、視覚過敏の悪化 |
| 視覚 | 急激な画面切り替え（1〜2秒/カット） | 情報処理が追いつかず過負荷に |
| 視覚 | ネオンカラー・蛍光色の多用 | 視覚過敏のある子に不快感 |
| 視覚 | 高コントラストの縞模様 | 視覚ストレスを引き起こす可能性 |
| 視覚 | 画面上の情報過多（テキスト+アイコン+アニメーション同時） | 処理負荷が高すぎる |
| 聴覚 | 急激な音量変化 | 聴覚過敏の子にパニックを引き起こす可能性 |
| 聴覚 | 甲高い効果音の連続 | 不快感・ストレス |
| 聴覚 | BGMとナレーションの音量が近い | 聞き分けが困難 |

#### 推奨する色の組み合わせ

- **安心感を与える色**: ペールブルー、ソフトグリーン、アースカラー（ベージュ・薄茶）
- **研究エビデンス**: ASD/ADHD児の30%がパステル調の背景を強く好み、視聴継続時間が15%向上（ProfileTree調査）
- **暖色系の使用**: 赤・オレンジは少量のアクセントに留め、大面積には使わない

### 3.2 注意維持のための設計（ASD/ADHD対応）

#### ASD児向け

- **予測可能な構造**: 毎回同じ「型」で進行する安心感
- **視覚的な予告**: 「これから○○をするよ」の前振り（研究で「内容プレビュー」の有効性が確認）
- **繰り返しパターンの活用**: ASD児は予測可能な動きを長く注視する傾向（Nature, 2025）
- **シンプルな背景**: 視覚情報を絞り、注目すべき対象を明確にする

#### ADHD児向け

- **短い単位での達成感**: 15〜30秒ごとに小さな「できた!」の瞬間
- **視覚的な進捗表示**: 「あと○ステップ」のような視覚的目安
- **適度な新規性**: 完全な繰り返しだと飽きるため、「同じ構造、少し違う中身」が有効
- **身体的な参加の促し**: 「一緒にやってみよう」で注意を再喚起

### 3.3 予測可能性と安心感

#### 研究エビデンス

- ASD児は予測可能な視覚刺激をより長く注視する（Scientific Reports, 2025）
- 予測可能な動きへの注目は、特に動画の後半で顕著に増加
- 一方、因果関係の学習（動きの軌道から結果を予測する）には困難を示す場合がある

#### 実践への応用

1. **オープニングの統一**: 毎回同じ音楽・同じキャラクターの登場シーンで始める
2. **構成の「型」を固定**: 例）あいさつ → 問題提示 → チャレンジ → 成功 → 振り返り
3. **トランジションの統一**: 場面転換は毎回同じアニメーション効果を使う
4. **エンディングの統一**: 同じフレーズ・同じ動きで終わる

### 3.4 自閉症の子どもが好む視覚パターン

| 研究知見 | 出典/背景 |
|---------|----------|
| 予測可能な繰り返し動きを好む | Scientific Reports (2025) |
| パターン認識能力が高い（パズル・視覚空間課題で強み） | Rainbow Therapy研究まとめ |
| 幾何学的パターンへの高い注目 | アイトラッキング研究 |
| 社会的刺激（顔）よりも物体・パターンに注目しやすい | PMC複数論文 |
| ソフトカラー+スムーズなテクスチャに安心感 | Nature Communications (2025) |

## 4. Nollaへの具体的提案

### 4.1 推奨デザインガイドライン

#### カラーパレット

**メインパレット（Nolla推奨）**

| 用途 | 色 | Hex例 | 備考 |
|------|------|------|------|
| プライマリ | やわらかいブルー | #6BAED6 | 安心感・信頼感。感覚過敏に配慮 |
| セカンダリ | やさしいグリーン | #7BC47F | 自然・成長のイメージ |
| アクセント | あたたかいオレンジ | #F5A623 | 注目ポイント・達成感に使用 |
| ベース | クリームホワイト | #FFF8F0 | 背景。真っ白より目に優しい |
| サブ | ソフトラベンダー | #B8A9C9 | 穏やかさ・特別感 |

**避けるべき色の使い方**
- ネオンカラー（#00FF00等）の大面積使用
- 赤（#FF0000）と緑（#00FF00）の直接隣接（色覚多様性への配慮）
- 白（#FFFFFF）背景に黒（#000000）文字のみの高コントラスト配色

#### テンポ・リズム設計

```
推奨BPM: 90〜110
カット間隔: 4〜6秒
1動画あたりの長さ: 20〜45秒
フック: 最初の2秒で視覚的インパクト（フラッシュではなく「動き」で）
```

#### 構成テンプレート（30秒版）

```
[0-2秒]  フック: キャラ登場＋問いかけ（「なにが出てくるかな？」）
[2-5秒]  問題提示: 今日のテーマを視覚的に提示
[5-15秒] チャレンジ: 一緒にやってみる（繰り返しを含む）
[15-22秒] 成功: 達成の喜び（キャラの笑顔＋効果音）
[22-27秒] 振り返り: 「できたね！」の確認
[27-30秒] エンディング: 統一のクロージング（ループ可能に設計）
```

#### 構成テンプレート（60秒版）

```
[0-3秒]   統一オープニング（毎回同じ音楽・キャラ登場）
[3-8秒]   問題提示（「今日は○○をやってみよう」）
[8-25秒]  メインチャレンジ（ステップごとに進行、繰り返しあり）
[25-35秒] 応用・バリエーション（少しだけ変化をつける）
[35-45秒] 一緒にやってみよう（参加を促す）
[45-52秒] 成功・達成（視覚的なごほうび演出）
[52-57秒] 振り返り（「今日は○○ができたね」）
[57-60秒] 統一エンディング（次回予告 or ループ接続）
```

### 4.2 避けるべき要素リスト

#### 絶対に避ける（Must Not）

1. **フラッシュ・点滅効果**（1秒間に3回以上の明滅）
2. **急激な音量変化**（10dB以上の急上昇）
3. **1〜2秒以下の超高速カット割り**
4. **ネオンカラー・蛍光色の大面積使用**
5. **複数の音声が同時に流れる状態**（BGM+ナレーション+効果音が全て大音量）
6. **予告なしの大きな音**（爆発音、叫び声等）

#### 極力避ける（Should Not）

7. **画面上の同時情報過多**（テキスト+アイコン+動くキャラ+装飾が同時に存在）
8. **背景の複雑なパターン**（チェック柄、細かい水玉等）
9. **赤と緑の直接隣接配色**
10. **不規則・予測不能なアニメーション**
11. **ハイテンションすぎるナレーション**（叫ぶような声）
12. **長い無音区間のあとの突然の音**

#### 注意して使う（Use with Care）

13. **赤色の大面積使用**（興奮・不安を誘発する可能性）
14. **高BPM（130以上）のBGM**
15. **画面全体の色変化**（背景色がガラッと変わる演出）

### 4.3 レベル別サムネイル色の提案

発達支援コンテンツのレベルを色で直感的に識別するシステム。

| レベル | 色 | Hex | 意味・根拠 |
|-------|------|------|-----------|

## まとめ: Nollaのショートコンテンツ戦略

### コアコンセプト

> **「安心できるテンポで、繰り返しの中に小さな発見がある」**

バズ動画の高速・高刺激フォーマットをそのまま採用するのではなく、**バズの仕組み（フック・ループ・完了率設計）は活かしつつ、テンポと刺激量を発達特性に配慮したレベルに調整する**のがNollaの差別化ポイント。

### 3つの設計原則

1. **予測可能で安心**: 毎回同じ構造・同じオープニング・同じエンディング
2. **やさしく鮮やか**: パステル寄りだが乳幼児にも認識できる適度な彩度
3. **参加できる間がある**: Ms. Rachel方式の「待つ設計」で、子どもが声を出したり動いたりする時間を確保

### 日本市場向けのポイント

- USの高刺激フォーマットではなく、**しまじろう＋Ms. Rachel的な「穏やかだけど引き込まれる」路線**
- 韓国（Pinkfong/Baby Bus）のデザインテイストは日本と相性が良く、参考になる
- 保護者が「安心して見せられる」ことが日本市場では特に重要な訴求点
