# Nolla 情報設計書 v1.0

**作成日**: 2026-04-04
**役割**: インフォメーションアーキテクト（IA）
**前提**: 設計仕様書v3.0 + ベンチマーク分析（Otsimo / Jade ND / Duolingo ABC）

---

## 0. ベンチマーク分析サマリー

| 項目 | Otsimo | Jade ND | Duolingo ABC |
|------|--------|---------|-------------|
| コンテンツ量 | 200+ゲーム | 3000+活動 | 150レッスン |
| 分類方法 | 16カテゴリ | テーマ別ワールド | 10レベル×15レッスン |
| ナビ方式 | パス+自由探索 | ハブ&スポーク | 建物型リニア |
| 子どもの見え方 | カテゴリ一覧 | テーマカード | 建物を順番に進む |
| 親子分離 | パスコード別画面 | 完全分離 | ジェスチャー認証 |
| 適応方法 | 親が手動調整 | 自動（成功率ベース） | 暗黙的ペーシング |

### 各社の強み・弱み（Nolla視点）

**Otsimo**: 200+ゲームを16カテゴリで整理。ただし「カテゴリ一覧」は文字が読めない子には厳しい。親がキュレーションする前提。
**Jade ND**: テーマ別ワールドで直感的。3000+活動を扱えるスケーラビリティ。ただしABA特化で「ゲーム感」は弱い。
**Duolingo ABC**: 建物メタファーが子どもに直感的。ただし完全リニア（自由選択なし）で、ASD児の「自分で選びたい」欲求に合わない場合がある。

## 1. Nollaの設計判断

### 1A. 利用者の特性（設計の大前提）

| 特性 | 設計への影響 |
|------|------------|
| 文字が読めない | テキストナビ不可。アイコン+音声+色で全て伝える |
| 選択肢が多いとパニック | 1画面5要素以下。段階的に選択肢を絞る |
| 予測可能性が最重要 | 画面構成・操作手順は毎回同じ。サプライズ禁止 |
| 自分の世界を作りたい | マイルーム+コレクション要素で満たす |
| ルーティンを好む | 毎日同じ流れで安心。「いつものやつ」が最優先表示 |
| 年齢幅が広い（3-18歳） | 知的レベルで分岐。暦年齢ではなく発達段階で適応 |
| 親は「効果」を求める | 親向け画面で成長の可視化。子どもの画面には見せない |

### 1B. コンテンツ分類: ゲーム vs ST を分けるか？

**結論: 子どもの画面では分けない。裏側で分ける。**

理由:
1. 子どもにとって「ゲーム」も「ST」も全部「あそび」に見えるべき（設計仕様書のコアコンセプト）
2. 「どっちで遊ぶ？」の選択は知的障害児には認知負荷が高い
3. STだけ別コーナーにすると「つまらない方」と認識され、使われなくなる
4. Otsimoも同一アプリ内でゲーム+ST+AACを統合している

**ただし親向け画面では明確に分類表示する**（「認知ゲーム」「ことばの練習」等）。

### 1C. ナビゲーション方式: ワールド型カルーセル

**結論: 「エリア（＝建物）」をカルーセルで選ぶ → エリア内のゲームを選ぶ 2階層構造**

```
ホーム画面（村の広場）
  ↓ カルーセルで建物を左右にスワイプ
エリア選択（建物をタップ）
  ↓ 建物の中に入る
ゲーム選択（エリア内のゲーム一覧）
  ↓ ゲームをタップ
ゲームプレイ
```

理由:
- MVP（3エリア）でも将来（15エリア）でも破綻しない
- カルーセルは1画面1建物なので情報過多にならない
- 建物メタファーは3歳でも理解できる（「おうちに入る」）
- エリア内のゲームが増えても、建物の中にゲームを追加するだけ

### 1D. ST系コンテンツの配置

STも「建物」として扱う。子どもからはゲームエリアと区別がつかない。

```
カルーセル上の建物:
  [なかまわけの家] [おぼえての天文台] [みつけてのツリーハウス]
  [ことばの図書館] [おはなしの劇場]
  [マイルーム] [ショップ]
```

- 「ことばの図書館」= AAC + 音声模倣
- 「おはなしの劇場」= 構音練習 + ビデオモデリング
- 子どもからは全部「建物に入って遊ぶ」に見える

**表示の出し分け:**
- アセスメント結果で「発語なし」→ AACの建物を優先表示（カルーセルの手前に配置）
- 「会話できる」→ 認知ゲームを優先、構音練習は奥に配置
- 親が設定で表示/非表示を切り替え可能

## 2. 画面フロー全体図

### 2A. 初回起動フロー

```
[アプリ起動]
    │
    ▼
[スプラッシュ画面]                    ← ロゴ + ナビキャラ登場アニメ（3秒）
    │
    ▼
[保護者セットアップ]                  ← ここは保護者が操作
    ├── アカウント作成（メール or Apple/Google）
    ├── お子さまの基本情報（年齢・性別・特性）  ← 3問
    ├── コミュニケーション能力               ← 4問
    ├── 療育と好み                          ← 3問
    ├── 同意取得（個人情報・眼球データ）
    └── 「お子さまにわたしてください」画面
            │
            ▼
[実技アセスメント]                    ← ここから子どもが操作
    ├── ナビキャラ登場「いっしょにあそぼう！」
    ├── タップチャレンジ（5-8試行）
    ├── おなじものどーれ（5-8試行）
    ├── おとをきいて（5-8試行）
    └── おぼえてね（5-8試行）
            │
            ▼
[アセスメント完了]
    ├── ナビキャラ「すごい！」+ 報酬（初回スター）
    └── パーソナライゼーション計算（裏側）
            │
            ▼
[ホーム画面（村の広場）]              ← 通常利用開始
```

### 2B. 通常利用フロー（毎回の起動）

```
[アプリ起動]
    │
    ▼
[ホーム画面（村の広場）]
    │
    ├── ナビキャラ「〇〇ちゃん、きょうもあそぼう！」
    ├── カルーセル: 建物が並ぶ
    ├── 右上: スター数
    ├── 左上: ナビキャラ + 気分表示
    └── 右下: ペアレンタルロック（大人専用入口）
    │
    │  左右スワイプ or 矢印タップ
    ▼
[建物選択]                           ← カルーセルで建物を1つ選ぶ
    │  建物をタップ
    ▼
[エリア画面（建物の中）]              ← エリア内のゲーム一覧
    │
    ├── MVPの場合: ゲーム1つだけ表示
    │   → 直接ゲーム開始（選択不要）
    │
    ├── 将来: ゲーム複数表示（カード型一覧）
    │   → ゲームをタップして開始
    │
    ▼
[チュートリアル]                      ← 初回のみ（文字なし、アニメ主導）
    │
    ▼
[ゲームプレイ]
    │
    ├── 適応型難度調整（成功率75-80%維持）
    ├── 即座フィードバック（200ms以内）
    ├── エラーレス学習
    ├── スター獲得
    │
    ├── 連続60分 → 「きゅうけいする？」（提案のみ）
    │
    ▼
[ゲーム終了]
    │
    ▼
[報酬まとめ画面]
    ├── 獲得スター表示
    ├── 「ショップにいく？」→ [ショップ]
    ├── 「おへやにいく？」→ [マイルーム]
    └── 「もっとあそぶ？」→ [ホーム画面]
```

### 2C. 報酬系フロー

```
[ショップ]
    ├── 棚にアイテムが並ぶ（木造店内の世界観）
    ├── 購入可能アイテムが柔らかく光る
    ├── タップ → スター消費 → アイテム獲得アニメ
    └── 「おへやにおく？」→ [マイルーム]

[マイルーム]
    ├── 俯瞰視点の自分の部屋
    ├── アイテムをドラッグで配置
    ├── 壁紙・床の変更
    └── 図鑑で所持アイテム一覧
```

### 2D. 親向けフロー

```
[ホーム画面]
    │  右下ロックアイコン → PIN入力 or 生体認証
    ▼
[親向けダッシュボード]
    │
    ├── [今日の記録]
    │   ├── プレイ時間・回数
    │   └── プレイしたゲーム + 評価
    │
    ├── [成長の記録]
    │   ├── 認知的柔軟性の推移グラフ
    │   ├── ワーキングメモリの推移グラフ
    │   ├── 注意制御の推移グラフ
    │   └── 各スキルの[?]で説明表示
    │
    ├── [設定]
    │   ├── セッション時間制限
    │   ├── 休憩提案タイミング
    │   ├── 眼球データ ON/OFF
    │   ├── 音声認識 ON/OFF
    │   ├── コンテンツ表示/非表示
    │   └── 子どものプロフィール編集
    │
    └── [戻る] → ホーム画面
```

## 3. カルーセルの設計詳細

### 3A. 建物の並び順

```
デフォルト順（左から）:
  位置1: アルゴリズム推奨（今日やるべきコンテンツ）
  位置2: 前回プレイしたゲームのエリア（ルーティン尊重）
  位置3-N: 残りのエリア（アセスメント結果に基づく優先度順）
  最後尾: マイルーム、ショップ

MVP時（5+2建物）:
  [推奨] [前回] [認知的柔軟性] [WM] [注意制御] [ことば] [おはなし] [マイルーム] [ショップ]
  ※実質的にはアセスメント結果で3-5個だけ表示（全部は出さない）

表示ルール:
  - 発語なし → ことばの建物を位置1-2に（AAC優先）
  - 会話できる → 認知ゲームを位置1-2に
  - 親が非表示にしたエリアはスキップ
```

### 3B. 建物の視覚的区別

```
各建物は以下で識別（文字不要）:

  なかまわけ:  コテージ/納屋     + オレンジ系屋根 + 果物アイコン
  おぼえて:    天文台           + 青緑系ドーム  + 星アイコン
  みつけて:    ツリーハウス      + 緑系木       + 虫眼鏡アイコン
  ことば:      図書館           + 桜ピンク     + 吹き出しアイコン
  おはなし:    劇場             + パステル紫   + 劇場カーテンアイコン
  マイルーム:   自分の家         + 温かい木造   + 家アイコン
  ショップ:    お店             + 看板つき     + 星/カートアイコン
```

### 3C. カルーセルのインタラクション

```
操作方法（2種類、切替不要で両方使える）:
  1. 左右矢印ボタンタップ（矢印は常時表示、64×64px）
  2. 左右スワイプ（ただしメインはタップ。スワイプは補助）
  ※ダブルタップ禁止（設計ルール）

建物の選択:
  建物をタップ → 建物が少し拡大（scale 1.05、200ms）
  → ドアが開くアニメーション（300ms）
  → エリア画面に遷移（フェード、300-500ms）

カルーセルの予測可能性:
  - 建物の並び順は毎回同じ（アルゴリズムが裏で変えても、位置は固定）
  - 端に来たらループしない（行き止まり = 予測可能）
  - 現在位置インジケータ: 画面下部にドット（今いる位置がわかる）
```

## 4. MVP時のエリア内画面

MVPでは各エリアにゲーム1つしかないため、建物に入る＝即ゲーム開始。

```
[建物タップ]
    │
    ├── ゲーム1つしかない場合:
    │   → チュートリアル（初回）→ ゲーム開始
    │   （エリア画面をスキップ、無駄な画面遷移を減らす）
    │
    ├── 将来ゲーム複数の場合:
    │   → エリア画面（ゲームカード一覧）
    │   → ゲームをタップ → チュートリアル → ゲーム開始
```
