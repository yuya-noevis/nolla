## 競合分析サマリ：Nollaの差別化軸

### Nollaが勝っている ✓

| 軸 | 詳細 |
|---|---|
| **IQ層の特化度** | IQ21-110（知的障害+境界知能）に完全特化。他アプリは定型発達または汎用 |
| **NCI型認知測定** | 実行機能・視覚空間・ワーキングメモリの3軸。直接的に療育効果が測定可能 |
| **長期スコアリング** | Day1から全データ永続保存で個別進捗と群特性の両方を追跡（プレーヤー＆クリニカル双方の見地） |
| **適応型難度の特化** | 75-80%成功率の自動維持（ASD児のパニック防止 + エラーレス学習） |
| **親貨幣統合** | ゲーム内報酬と親の現物報酬の連携（Goallyと同等レベル、ゲーム性はNolla優） |
| **ゲーム性** | Minecraft/どうぶつの森型の「蒐集＋マイルーム」。Toca Bocaの楽しさ + 療育機能 |
| **非競争設計** | ASD児のストレスを最小化（タイマーなし、減点なし、エラー音なし） |
| **文字なしナビゲーション** | 知的障害児の読字困難を考慮した設計（業界標準外） |

|---|---|---|
| **知的障害特化型NCI** | IQ21-110層に最適化された3軸認知測定 | **他アプリなし** ← ここが唯一のポジション |
| **長期個別化スコアリング** | Day1データの永続化で個別曲線追跡 | CogniFit も類似だが、知的障害最適化でない |
| **エラーレス学習の自動実装** | 75-80%成功率の自動維持 | ほぼ競合なし（設計レベルでの考慮） |
| **ASD児向けゲーム設計** | Minecraft/どうぶつの森+療育 | Toca Bocaはゲーム性のみ。Otsimoはゲーム性欠 |

---

## 競合分析から導き出す戦略的示唆

### 1. 差別化戦略：「認知科学 ✕ ゲーム設計」

**Nollaの独自価値提案（UVP）:**
- CogniFitの「認知測定の科学」 + Toca Bocaの「ゲーム設計の卓越性」 + Goallyの「親介入の実装度」を結合

**市場ポジショニング:**
- 「療育アプリ」ではなく「ASD児がハマるゲーム（療育効果付き）」
- 親向けスローガン: 「毎日のゲームが、認知機能の個別化トレーニングに」

### 2. 認知測定の深化（RCT実施）

**現在の課題**: CogniFitに学術的エビデンス量で負けている

**対抗策**:
- MVP実装後、半年以内にRCT設計着手
- 測定軸: 成功率、習熟速度、NCI推移の個別化効果
- 論文公表で「知的障害特化型の臨床効果」を実証

### 3. 親連携機能の段階化

**Phase 1（MVP）**: NCI型スコアリング + ダッシュボード表示  
**Phase 2**: トークン報酬の親への通知 + 現物報酬連携オプション  
**Phase 3**: 施設向けダッシュボード（CoDMON等との連携）

### 4. AAC機能の除外判断

**決定**: MVP段階では言語療法機能を除外（正解）

**理由**:
- Speech Blubsがこの領域で強い
- AAC は SLP 資格や医療管理が必要（スコープ外）
- NCI + ゲーム性に注力すべき

**後期対応**: v2以降、ロゴペディア連携で検討（自社実装でない）

### 5. マーケット拡張方向

**国内市場**:
- LITALICO との連携（情報→ゲーム流入）
- 放課後等デイサービス向けパッケージ（CoDMON連携）

**国際市場**:
- 英語版でグローバル展開（2-3年後）
- Otsimoより**学術的に優位な立場**で攻撃

## 最終結論

### Nollaの市場ポジション

```
従来型療育アプリ（たすく等）    ← 単機能、ゲーム性欠
                ↓
         Nolla（MVP）← 認知測定＋ゲーム性
                ↓
認知トレーニングアプリ（CogniFit） ← 科学的だが障害特化でない
```

**Nollaが占める空白**: 「知的障害+ASD特化 ✕ 認知測定 ✕ ゲーム性」

**成功要件**:
1. MVP 実装品質（Toca Bocaレベルの楽しさ）
2. NCI スコアリングの実装検証（CogniFitレベルの信頼性）
3. RCT による臨床エビデンス
4. 親向けダッシュボードの UX（Goallyレベルの実用性）

全てが揃えば、**独占市場（IQ21-110の療育ゲーム）** を形成可能。
