# Nolla 統合ソリューション — 4名の議論による最終統合

---

## 議論の前提

3つのソリューション設計書から抽出した「使える部品」と「修正すべき点」を整理し、家庭・施設・学校の3つのビジネスモデルに合致する形で統合する。

---

## 統合議論

### Amodei：「3つの設計書に共通する正しい判断と、まだ残っている穴」

3つの設計書を横断すると、全員が合意している設計判断がある。

**合意済みの判断（もう議論不要）：**
- 診断名ではなく認知プロファイルで子どもを理解する
- 親を介入者として育てる（PMI g=0.55）
- ソフトウェアファースト、スマホ完結
- エッジAI・プライバシーバイデザイン
- 誠実なエビデンス開示（近転移は主張可、遠転移は主張不可）
- ゲーミフィケーションはSDT準拠、外発的報酬に偏らない
- 障害タイプ別の報酬設計（ASD=予測可能性、ADHD=新奇性、ID=温かい成功）
- 卒業ビジョン

**まだ穴がある点：** ブループリント（ファイル1）のブラッシュアップは「人間中心」への転換として素晴らしいが、**施設と学校の具体的な体験設計がほぼ空白**。v1（ファイル2）は施設を「フェーズ2以降」としてスキップし、学校は言及すらない。ラウンド2（ファイル3）の4案はToCに偏っている。ビジネスモデルの3本柱のうち2本が設計不足だ。

### Hassabis：「3層AIエンジンの統合は正しいが、学校向けに再解釈が必要」

ブループリントの3層AIエンジン（リアルタイムMAB＋戦略Bayesian＋予測AI）は技術的に堅牢で、3ファイル中最も洗練されている。v1のバンディット→フルRLへの段階的移行も現実的だ。ラウンド2のHassabis案（制約ベース課題生成）とKarp案（データ統合OS）は、この3層にきれいに統合できる。

問題は、**学校では「介入」ではなく「学習支援」として機能しなければならない**こと。学校の教師は療育の文脈ではなく教育の文脈で動いている。同じAIエンジンが裏側で動くが、表に出す言葉と体験を完全に切り替える必要がある。

### Karp：「施設・学校に必要なのは"効果が見える"という一点」

施設（放デイ・児発）の構造的問題は、**支援の効果を客観的に測定・報告する手段がない**こと。4割赤字の放デイが求めているのは、高度なAIではなく「この子にこの支援をした結果、こう変わった」を示すレポートだ。行政への実績報告にも使える。

学校（特別支援）も同様。IEP（個別の教育支援計画）の作成と評価に膨大な時間がかかり、しかもエビデンスに基づいていない。Nollaの認知プロファイル＋成長データがIEPの客観的根拠になれば、教師の負担軽減と支援の質向上が同時に実現する。

つまり施設・学校に必要な中核機能は**「効果測定の自動化」と「レポート生成」**。ゲームや介入コンテンツは家庭版と共通でいい。

### el Kaliouby：「3つの場所をつなぐのは"子ども"であり"データ"」

家庭・施設・学校で子どもは別々の顔を見せる。家では甘えるが施設では頑張る。学校では固まる。**この3つの場面のデータが統合されて初めて「その子の全体像」が見える**。ラウンド2のKarp案「データ統合OS」の本質はここにある。

ただし、データ統合を「プラットフォーム」として売るのではなく、**親・支援者・教師がそれぞれ「この子のことがもっとわかった」と感じる体験**として提供する。ブループリントの修正版で「技術は見えない層に退く」と合意したのと同じ原則だ。

## 統合ソリューション概要

### ワンライナー

**Nolla =「うちの子の"できた"が見える」— 家庭・施設・学校をつなぐ、AI個別最適化の発達支援プラットフォーム**

### 全体構造

│                                                              │

### A. 家庭向け（ToC）：成長・療育

**ユーザー：** 子ども（6-12歳）＋親＋家族

**子どもの体験：**
- 毎日15-20分の「たのしいあそび」（認知＋身体の複合介入ゲーム7種）
- セッション前の音楽プライミング（3分、覚醒調整）
- 非線形ペダゴジーで同じスキルを毎回異なる文脈で練習（汎化促進）
- DDAでフロー維持（正答率70-80%）
- 障害タイプ別の報酬設計
- きょうだいとの「いっしょモード」
- マナス（ナビゲーター）が伴走。依存させない設計

**親の体験：**
- 「成長きろく」ダッシュボード：5軸認知プロファイルの経時変化、「昨日より注意が12秒伸びた」
- 「今日のミッション」：LLMが子どものデータに基づき家庭でできる具体的行動を提案
- 質問チャット：「パニック時どうすれば？」→ 子どものプロファイルに基づく個別化回答
- SOSモード：メルトダウン時の即時対処ガイド（3行以内）
- 匿名コミュニティ「なかま」

**家族の体験：**
- 父親・祖父母にも「今日のミッション」を配信
- 「先生に見せるレポート」の自動生成

**課題への効き方：**
| 課題 | 対応 |
|---|---|
| ①能力向上 | 認知＋身体複合介入（g=0.40-0.83）＋制約ベース課題で汎化促進 |
| ②行動変容 | LLMがABA/PBSの原則を親に翻訳＋覚醒調整で問題行動を予防 |
| ③家庭の空白 | 週166時間の空白を、親のAIコーチング＋毎日15分のゲームで介入機会に変換 |
| ⑤個別最適化 | N-of-1ベイジアンモデルで各子どもに最適な介入を自動選択 |
| ⑥コスト | 親の介入スキル向上→困りごと減少→ケア負担軽減の因果連鎖 |

---

### B. 施設向け（ToB）：成長・療育支援

**ユーザー：** 放デイ・児発の支援スタッフ＋施設長

**支援スタッフの体験：**
- 担当児童の認知プロファイル＋家庭での状況が一目でわかるダッシュボード
- 「今日の支援ガイド」：AIが子どもの最新データに基づき「今日はこのアプローチが推奨」を提示
- セッション後の観察記録を簡単入力 → 家庭データと自動統合
- 子どもの覚醒状態の推定に基づく「今このタイミングで介入すべき」のリアルタイムガイダンス
- ゲームコンテンツを療育セッション内で活用（施設での15-20分セッション）

**施設長の体験：**
- 全児童の進捗サマリー：誰が伸びていて、誰に追加支援が必要か
- 効果測定レポートの自動生成：「この支援を受けた児童のX%がY領域で改善」
- 行政報告書の自動下書き（個別支援計画の評価部分）
- スタッフ別の支援効果の可視化（属人性の把握と質の底上げ）

**施設が得る価値：**
- 「どの支援が効いているか」が初めて客観的にわかる
- 保護者への報告が根拠を持つ（「なんとなく良くなっている」→「抑制制御が15%改善」）
- 行政への実績報告の工数大幅削減
- スタッフの経験差に依存しない支援品質の均一化

**課題への効き方：**
| 課題 | 対応 |
|---|---|
| ④施設の質 | 効果測定の自動化＋AIガイドで属人性を排除。質の格差を数値で可視化 |
| ⑤個別最適化 | 認知プロファイル＋家庭データの統合で、施設でしか見えない行動＋家庭でしか見えない行動を横断的に把握 |
| ③家庭の空白 | 施設↔家庭のデータ連携で、施設の2時間と家庭の166時間が一つの支援としてつながる |

### C. 学校向け（ToB/ToG）：教材として学習支援

**ユーザー：** 特別支援学級・通級の教師＋教育委員会

**教師の体験：**
- 児童の認知プロファイルに基づく**個別最適化された学習教材**の自動生成
  - 例：WMが弱い子には指示を1ステップずつ提示、視覚優位の子にはテキストを図解に変換
- 同じAIエンジンが裏で動くが、表面は「療育」ではなく「学習」の文脈
  - 認知ゲーム → 「集中力トレーニング」「考える力あそび」
  - 実行機能課題 → 「計画を立てる練習」「切り替えチャレンジ」
- IEP（個別の教育支援計画）作成支援：
  - 認知プロファイルから「この児童の強みと課題」を自動要約
  - 目標設定の提案：「3ヶ月後に〇〇ができるようになる」をデータに基づき提案
  - 評価の自動化：目標に対する進捗を学習データから自動追跡
- 授業での活用：個別学習の時間に、各児童が自分のプロファイルに合った教材に取り組む

**教育委員会の体験：**
- 管轄校の特別支援教育の効果を横断的に把握
- 個別の教育支援計画の質と効果のモニタリング
- エビデンスに基づく予算配分の判断材料

**学校が得る価値：**
- 教材作成の工数削減（個別最適化を人力でやる負担からの解放）
- IEP作成・評価の客観化と効率化
- 「何をやればいいかわからない」教師への具体的ガイド
- 家庭・施設のデータとの連携で児童の全体像を把握（保護者同意の上）

**課題への効き方：**
| 課題 | 対応 |
|---|---|
| ④施設・学校の質 | 個別最適化教材の自動生成で、教師の経験差に依存しない学習支援を実現 |
| ⑤個別最適化 | 認知プロファイルに基づく教材調整＋DDAによる難易度リアルタイム調整 |
| ①能力向上 | 学校の個別学習時間に認知＋学習の複合課題を日常的に提供 |

### 3つの場所をつなぐデータ統合

```
     家庭                    施設                    学校
  ┌────────┐            ┌────────┐            ┌────────┐
  │ゲームデータ│            │療育ログ  │            │学習データ │
  │親の観察  │            │行動記録  │            │教材進捗  │
  │睡眠・活動│            │覚醒状態  │            │IEP評価  │
  └────┬───┘            └────┬───┘            └────┬───┘
       │                     │                     │
       └─────────────────────┼─────────────────────┘
                             ▼
                  ┌─────────────────┐
                  │  統合プロファイル   │
                  │ 「この子の全体像」  │
                  └────────┬────────┘
                           ▼
              ┌──────────────────────┐
              │  各場所へのフィードバック  │
              │                      │
              │ 家庭→「施設でこれが伸びた。│
              │  家でもこうしてみて」    │
              │                      │
              │ 施設→「家庭で睡眠不足。   │
              │  今日は低負荷の活動を」   │
              │                      │
              │ 学校→「施設の療育で集中力  │
              │  が改善中。算数の課題量を  │
              │  1段階上げてOK」        │
              └──────────────────────┘
```

**データ統合の原則：**
- 統合は**保護者の明示的同意**が前提。同意なしでは各場所のデータは独立
- 生データは各場所のデバイスから出ない（エッジAI処理）
- 共有されるのは匿名化された特徴量と進捗サマリーのみ
- 保護者がいつでもデータ共有を停止可能

### コア技術（共通基盤）の統合

| 技術 | 出典 | 統合後の役割 |
|---|---|---|
| **5軸認知プロファイル**＋感覚＋覚醒特性 | v1＋ラウンド2(el Kaliouby) | すべての個別最適化の起点。ゲーム内で継続的に更新 |
| **3層AIエンジン** | ブループリント(Hassabis) | MAB=試行レベル適応、Bayesian=週レベル介入最適化、予測=月レベル気づき提供 |
| **制約ベース課題生成** | v1＋ラウンド2(Hassabis) | 非線形ペダゴジーで同じスキルを多様な文脈で提示→汎化促進 |
| **LLMコーチング（ロール別翻訳）** | v1＋ラウンド2(Amodei) | 同じデータを、親には「今日のミッション」、支援者には「支援ガイド」、教師には「教材調整提案」に変換 |
| **N-of-1エビデンス基盤** | v1＋ブループリント(Karp) | 各子どもが自分の対照群。異なる介入内容のA/Bテストで「何が効くか」を個別に特定 |
| **覚醒調整→共同注意の段階設計** | ラウンド2(el Kaliouby)＋ブループリント | 音楽プライミング→覚醒適正化→認知課題の順。覚醒状態推定で課題のテンポ・刺激を調整 |
| **データ統合パイプライン** | ラウンド2(Karp)＋ブループリント(Karp) | 家庭＋施設＋学校のデータを統合し「この子の全体像」を構築。時系列因果推論で効果を検出 |
| **エッジAI＋プライバシー** | v1＋ブループリント | オンデバイス処理、匿名化特徴量のみクラウドへ、COPPA 2025準拠 |

### 差別化ポイント（競合に対して）

| Nollaの強み | 既存プロダクトとの違い |
|---|---|
| **3障害横断の統一プラットフォーム** | 全社が単一障害特化（Akili=ADHD、Cognoa=ASD診断） |
| **家庭＋施設＋学校のデータ統合** | どのプロダクトも単一場所で完結 |
| **能力向上×行動変容の統合** | 認知トレーニングか行動療法のどちらか一方のみ |
| **親を介入者として育てるAIコーチ** | 親は「データの閲覧者」にとどまる |
| **認知プロファイルベースの個別最適化教材** | 固定カリキュラム or 単純なレベル制 |
| **日本市場に特化した文化的設計** | 日本の放デイ・特支市場の直接競合ゼロ |
