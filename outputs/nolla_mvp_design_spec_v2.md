## 0. 設計原則（全機能共通）

### 絶対ルール
1. **時間制限なし** — self-pacedが基本。カウントダウンタイマー禁止
2. **予測可能性の確保** — ルールは変わらない。UI配置は常に同じ。「次に何が起こるか」が常に明白
3. **文字を使わない** — ナビゲーション・ゲーム内は全てアイコン+色+形+アニメーションで意味を伝える
4. **感覚オーバーロード防止** — 1画面の情報要素 5個以下、自動再生アニメ禁止、フラッシュ/点滅禁止
5. **エラーを罰しない** — 大げさなエラー音禁止、スコア減点禁止、「間違い！」禁止。サイレント修正（正解への再提示）
6. **即座フィードバック** — 200ms以内にビジュアル+音声+振動で応答

### ビジュアルデザイン方針（Minecraft/どうぶつの森の成功要因を踏襲）

ASD児に人気のゲームに共通する6つの設計要因を本アプリに適用:

| 成功要因 | Minecraft | どうぶつの森 | Nollaでの実装 |
|---------|-----------|------------|-------------|
| 予測可能性 | ルール不変、物理法則一定 | 日課固定、季節予測可能 | ゲームルール固定、毎回同じ画面構造 |
| 視覚的シンプルさ | ピクセルアート・ブロック | パステル・丸み | 幾何学的デフォルメ + くすんだ配色 |
| 自分の世界を作る | 建築自由 | 島カスタマイズ | 報酬ショップ + マイルーム |
| コレクション | アイテム・ブロック収集 | 家具・虫・魚コレクション | アイテムコレクション + 図鑑 |
| 即座フィードバック | ブロック配置=即結果 | 釣り=即結果 | 200ms以内の複合フィードバック |
| 非競争 | シングルプレイ可 | 勝ち負けなし | 他者との比較なし |

根拠: ASD児の85%が色彩を高い強度で知覚（PMC 2022）。幾何学的刺激に14ヶ月から親和性（JAMA Psychiatry）。コレクション行動はASD児の25%に見られる健全な特別関心（Springer 2024）。

### カラーパレット
```
背景:       #F5F5F5 (ライトグレー)
プライマリ:  #4A7BA7 (カームブルー)
セカンダリ:  #6BA47A (ソフトグリーン)
アクセント:  #D4A574 (ウォームベージュ)
テキスト:    #333333 (ダークグレー)
成功:       #6BA47A (グリーン) + チェックマーク
ゲーム世界:  Minecraftの抑制型ブロック色 + どうぶつの森のパステル を参考に、
            くすんだ（低彩度の）カラフルさを実現。退屈にならないよう
            アクセントとしてソフトなカラーバリエーションを使用
```
赤・黄・蛍光色・高彩度色は使用禁止（感覚過負荷を誘発）

### タッチターゲット
- 最小: 48×48px（3-8歳は64×64px推奨）
- アイコン間余白: 最低16px
- スワイプ操作は使わない（タップ or 大きなドラッグのみ）
- ダブルタップ禁止

### キャラクターデザイン方針
- **幾何学的シンプルさ**: Minecraftのブロック型キャラのように、対称的でシンプルな形状。ASD児は幾何学的刺激に対する親和性が高い（JAMA Psychiatry）
- **デフォルメ動物**: どうぶつの森のように丸みを帯びた柔らかいシルエット
- リアルすぎる顔は避ける（不気味の谷）
- 感情表現は明確かつ単純（笑顔、中立、困り顔の3段階）
- ユーザーの「好きなもの」（電車、数字、回転するもの等）をマイルーム・報酬に組み込む

## 1. 初回アセスメント（オンボーディング）

### 1A. 保護者への質問（2分以内、10-12問に削減）

#### Step 1: 基本情報（3問）
- お子さまの年齢（スライダー: 3-18歳）
- 性別（男/女/回答しない）
- 診断名は「聞かない」を基本にし、任意入力
  - 「あてはまることを選んでください（複数選択OK）」形式
  - よく同じ動作を繰り返す / 音や光に敏感 / 新しい環境で不安 / 興味が限定的 / 指示の理解が難しい時がある

#### Step 2: コミュニケーション能力（4問）
- **発語レベル**（単一選択）: 発語なし / 単語のみ / 二語文 / 短い文章 / 会話できる
- **理解力**: 簡単な指示（「来て」）/ 文で理解 / 文脈で理解 / よくわからない
- **AAC使用経験**: 使用していない / 試したことがある / 毎日使用
- 使用中のツール名（任意: PECS、Proloquo等）

#### Step 3: 療育と好み（3問）
- 受けている支援（複数選択）: 特別支援学級・学校 / 療育施設 / ST / OT / ABA / なし
- 日常の困りごと（最大3つ）: コミュニケーション / 生活スキル / 感覚の過敏さ / 切り替えの難しさ / 学習
- 好きなもの・興味があるもの（複数選択）: 電車 / 数字 / 動物 / 音楽 / 回転するもの / その他

※ v1.0にあった「アプリに期待すること」は削除（質問削除でサインアップ率+40%のエビデンス）

#### 法的対応
- 障害情報 = 要配慮個人情報 → 明示的同意チェックボックス必須
- 「このデータはお子さまに最適な学習を提供するために利用します」
- オプトアウト不可（個人情報保護法）

### 1B. 子ども向け実技アセスメント（ゲーム形式）

**コンセプト: 「テスト」ではなく「はじめての冒険」**

| ミニゲーム | 測定項目 | 時間 | 指示方法 |
|-----------|---------|------|---------|
| **タップチャレンジ** | タップ反応速度・精度 | 制限なし（子どものペース） | 光るボタンをタップ（視覚のみ） |
| **おなじものどーれ** | 視覚的マッチング能力 | 制限なし | 同じ絵を見つけてタップ（視覚のみ） |
| **おとをきいて** | 聴覚理解力 | 制限なし | 「りんごはどれ？」→ 絵を選ぶ（音声指示） |
| **おぼえてね** | ワーキングメモリ | 制限なし | 光った順番にタップ（視覚のみ） |

- **時間制限なし**: ASD児は精度重視で応答時間が長い。制限は真の能力を過小評価する（PMC Sensory Processing研究）
- 各ミニゲームは5-8試行で自動完了（時間ではなく試行数で区切る）
- 全ミニゲームに即座報酬（スター、音声「すごい！」）
- 中断OK（「あとでもいいよ」メッセージ）
- 間違えてもネガティブフィードバックなし
- 結果は裏で自動記録 → 難度設定に使用

### 1C. 名前の収集（初回セッション後）

初回セッション完了後に表示:
```
「たのしかった？ もっとたのしくするために、
 おなまえをおしえてね！」

 [ニックネーム入力欄]
 [あとで]
```
- 任意（スキップ可）
- 入力された場合、ホーム画面で「〇〇ちゃん、きょうもあそぼう！」と表示
- 根拠: 名前の個人化は読解理解と感情的絆を向上させる（European Journal of Psychology of Education）

### 1D. パーソナライゼーションマッピング

```
発語レベル → コンテンツ比率
├ 発語なし      → AAC 50% / 認知ゲーム 40% / 音声模倣導入 10%
├ 単語のみ      → AAC 30% / 認知ゲーム 40% / 語彙拡張 30%
├ 二語文〜      → AAC 10% / 認知ゲーム 40% / 構音練習 50%
└ 会話できる    → 認知ゲーム 50% / 構音練習 30% / 応用会話 20%

実技結果 → ゲーム難度初期値
├ 成功率 0-30%   → Level 1（最も簡単、選択肢2つ）
├ 成功率 31-60%  → Level 2（選択肢3つ）
├ 成功率 61-80%  → Level 3（選択肢4つ）
└ 成功率 81%+    → Level 4（複合タスク）
```

### 1E. 保護者への結果フィードバック

```
「お子さまのプロフィールができました！」

得意なこと:
  - 視覚的な情報の処理が得意です
  - タップの反応がとても速いです

これから一緒に練習すること:
  - 音を聞いて選ぶ力
  - 順番を覚える力

→ お子さまに合わせたゲームが自動で選ばれました
```
※ 「できない」「遅れ」等のネガティブ表現は絶対に使わない

## 2. 認知機能強化ゲーム（3種類）

### エビデンスに基づく選定理由

| 順位 | 認知機能 | Effect Size | 根拠 |
|------|---------|------------|------|
| 1位 | 認知的柔軟性（実行機能） | 中〜大 (g>0.50) | メタ分析59研究n=2,122 |
| 2位 | ワーキングメモリ | 中 (g=0.63-0.64) | CogMed RCT、Nature Scientific Reports |
| 3位 | 注意制御 + 処理速度 | 小〜中 | CPAT研究、アクションゲーム研究 |

### 共通仕様

- **セッション時間制限なし**: 保護者が設定画面で上限を任意設定可能（デフォルト: 制限なし）。連続60分経過時に「休憩しませんか？」と提案（強制ではない）
- **適応型難度調整**: 成功率75-80%を自動維持
- **強化スケジュール**: 初期CRF（全正解に報酬）→ 習得後VR3-5（平均3-5正解ごとに報酬）
- **フィードバック**: 200ms以内、ビジュアル+音声+振動の複合
- **チュートリアル**: アニメーション主導、4段階プロンプティング削減法

### 適応型難度調整アルゴリズム

```
入力:
  - 直近10試行の成功率 (weight: 50%)
  - 反応時間 (weight: 25%)
  - エラーパターン (weight: 15%)
  - セッション継続時間 = 疲労度 (weight: 10%)

判定:
  IF 成功率 < 75% → 難度 -1（選択肢を減らす、ヒントを増やす）
  IF 成功率 > 85% → 難度 +1（選択肢を増やす、複合タスク化）
  ELSE → 難度維持

  IF 反応時間が前5試行比 +50%以上 → 疲労と判定 → 休憩を提案
```

### 報酬システム（v2.0で追加: 報酬ショップ + マイルーム）

```
ゲームプレイ → スター獲得
  ↓
報酬ショップ（スターとアイテムを交換）
  ├ 小アイテム: スター3個（机、椅子、花等）
  ├ 中アイテム: スター10個（ベッド、テレビ、ペット等）
  └ 大アイテム: スター30個（特別な家具、乗り物等）
  ↓
マイルーム（集めたアイテムで自分の部屋を飾る）
  ├ アイテムを自由に配置
  ├ 壁紙・床を変更
  └ 集めたアイテムを図鑑で一覧表示（コレクション要素）
```

**設計根拠**:
- トークンエコノミー（ABA）: スターを中間強化子、アイテムをバックアップ強化子として使用。等価交換の概念学習にもなる
- コレクション要素: ASD児の25%が収集行動を示す（Springer 2024）。どうぶつの森の家具コレクション、Minecraftのアイテム収集と同じ原理
- 自分の世界構築: Minecraft/どうぶつの森がASD児に人気な最大の理由（GamesLearningSociety研究）

### Game 1: 「なかまわけ」（認知的柔軟性）

**鍛えられる力**: 「ルールを理解して切り替える力」
→ 日常例: 遊園地で順番を忍耐強く待てるようになる。お着替えの手順を覚えて場面に応じてできるようになる

#### ゲームフロー
```
1. 画面上部にルールカード表示（例: 色アイコン = 「いろでわける」）
2. 画面中央にアイテムが1つ表示される（例: 赤いりんご）
3. 画面下部に2-3つの箱（分類先）
4. 子どもが正しい箱にドラッグ or タップ
5. 正解 → 緑チェック + 「ピンポン♪」 + スター獲得
6. 不正解 → サイレント修正（正解の箱が光って誘導）
```

#### 難度レベル

| Level | ルール | 選択肢 | 対象 | ルール切替 |
|-------|--------|--------|------|-----------|
| 1 | 色だけ（赤/青） | 2つの箱 | 重度ID | なし |
| 2 | 色（赤/青/黄） | 3つの箱 | 中度ID | なし |
| 3 | 形だけ（丸/四角/三角） | 3つの箱 | 中度ID | なし |
| 4 | 色→形に切り替え（予告あり） | 3つの箱 | 軽度ID | あり（5-10秒予告） |
| 5 | カテゴリ（食べ物/動物/乗り物） | 3つの箱 | 軽度ID | あり（5-10秒予告） |

#### ルール切り替えの設計（Level 4以上のみ）
- **Level 1-3ではルール切り替えなし**。ゲーム難度は選択肢の数・複雑さのみで調整
- Level 4以上でのルール変更:
  - **5-10秒の静的予告画面**（アニメーションではなく静止画で明確に提示）
  - **変更頻度は固定**: 10試行ごと（予測可能）
  - **変更前に練習ラウンド**: 新ルールで2試行の練習（エラーレス）
- 根拠: ASD児は予測可能性に高い感度。事前通知で問題行動が有意に減少（PMC Advance Notice研究）。3秒は短すぎ、5-10秒の明確な予告が推奨

#### チュートリアル（文字なし）
```
Step 1: 画面に赤いりんごが表示される（2秒待機）
Step 2: ハンドアニメーションが赤い箱を指す
Step 3: りんごが赤い箱に吸い込まれる → スター + 「ピンポン♪」
Step 4: 「やってみよう！」（ボタンが光る）
  → 子どもがタップ → 成功 → フル報酬
  → 失敗 → 正解の箱が光って誘導（エラーレス学習）
```

### Game 2: 「おぼえてタッチ」（ワーキングメモリ）

**鍛えられる力**: 「さっき見たものを覚えておく力」
→ 日常例: お着替えの順番を覚えて自分でできるようになる。「りんごとパンを持ってきて」と言われて両方覚えていられる

#### ゲームフロー
```
1. 画面にカード（動物の絵）が2-6枚並ぶ
2. カードが順番に1枚ずつ光る（1枚あたり1秒）
3. 全部光り終わったら「やってみよう！」
4. 子どもが光った順番にカードをタップ
5. 正解 → 動物が動く + 音声「すごい！」 + スター
6. 不正解 → もう一度光る（ヒント）→ 再挑戦
```

#### 難度レベル

| Level | 記憶数 | カード枚数 | 光る速度 | 対象 |
|-------|--------|-----------|---------|------|
| 1 | 1つ | 2枚 | 2秒/枚 | 重度ID |
| 2 | 2つ | 3枚 | 1.5秒/枚 | 重度〜中度ID |
| 3 | 3つ | 4枚 | 1秒/枚 | 中度ID |
| 4 | 4つ | 5枚 | 1秒/枚 | 軽度ID |
| 5 | 5つ | 6枚 | 0.8秒/枚 | 軽度ID |

#### エラー処理（エラーレス学習）
```
1回目の不正解: 正解のカードだけが薄く光る（ヒント）
2回目の不正解: 正解のカードが明確に光る + ハンドアニメーション
3回目: 自動的に正解を実行（子どもは見るだけ）→ 次の試行へ
※ エラー回数はスコアに影響しない
```

### Game 3: 「みつけてタッチ」（注意制御 + 処理速度）

**鍛えられる力**: 「たくさんの中から目的のものを見つける力」
→ 日常例: おもちゃ箱の中から欲しいものを見つけられる。スーパーでお気に入りのお菓子を棚から見つけられる

#### ゲームフロー
```
1. 画面上部にターゲット表示（例: 赤い星のアイコン）
2. 画面にアイテムが複数表示される（ターゲット + ディストラクタ）
3. 子どもがターゲットをタップ
4. 正解 → ターゲットがキラキラ + 音声 + スター
5. 不正解 → ディストラクタが薄くなる（ヒント）
```

#### 難度レベル

| Level | ターゲット数 | ディストラクタ数 | 類似度 | 対象 |
|-------|------------|----------------|--------|------|
| 1 | 1つ | 2つ | 全く違う（色も形も違う） | 重度ID |
| 2 | 1つ | 3つ | 色が同じ、形が違う | 中度ID |
| 3 | 2つ | 4つ | 色が同じ、形が似ている | 中度ID |
| 4 | 2つ | 6つ | 色・形が似ている | 軽度ID |
| 5 | 3つ | 8つ | 微細な違いのみ | 軽度ID |

#### 時間制限の代替設計
```
制限時間を設けない
「見つけた数」で報酬を与える
「ゆっくりでいいよ」のビジュアルメッセージ
10秒以上操作がない場合 → ターゲットが少し揺れる（ヒント）
```

## 3. ST系コンテンツ

### 3A. AAC（発語なし〜単語レベルの子ども向け）

**設計根拠**: ASD児の25-30%がminimally verbal。AAC導入は音声言語発達を阻害しない（むしろ促進する）— PMC/ASHA

#### 基本画面構成
```
┌─────────────────────────────────┐
│  メッセージバー（選択したシンボルが並ぶ）    │
│  [りんご] [ほしい] → 「りんごがほしい」     │
├─────────────────────────────────┤
│                                  │
│  ┌──────┐ ┌──────┐ ┌──────┐    │
│  │たべもの│ │きもち│ │どうさ│    │
│  └──────┘ └──────┘ └──────┘    │
│                                  │
│  ┌──────┐ ┌──────┐ ┌──────┐    │
│  │ばしょ│ │ひと │ │もっと│    │
│  └──────┘ └──────┘ └──────┘    │
│                                  │
└─────────────────────────────────┘
```

#### シンボル設計
- **形式**: イラスト（写真よりもASD児に理解しやすいエビデンスあり）
- **コア語彙 50語**（日常の80%をカバー）: ほしい、いや、もっと、おわり、たべる、のむ、いく、みる、あそぶ、てつだって、トイレ、ママ、パパ、せんせい、ともだち、りんご、パン、みず、ジュース、うれしい、かなしい、いたい、こわい、つかれた、ここ、あっち...
- **フリンジ語彙**: カテゴリ別に拡張（食べ物30語、気持ち15語、動作20語、場所10語...）
- **カスタマイズ**: 保護者がシンボル追加・非表示設定可能

#### 音声出力
- AI TTS（子どもの声）で自然な発声
- タップ → 即座に音声再生（100ms以内）
- 複数シンボル選択 → 文として読み上げ（「りんご」+「ほしい」→「りんごがほしい」）

#### 段階的学習設計
```
Stage 1（導入）: 1シンボルタップ → 音声再生（「りんご」）
Stage 2（選択）: 2つから1つを選ぶ（「りんご」or「バナナ」、ほしいのはどっち？）
Stage 3（組み合わせ）: 2シンボルで文を作る（「りんご」+「ほしい」）
Stage 4（拡張）: 3シンボル以上の文構築
```

### 3B. 音声模倣・発語促進（発語なし〜単語レベル）

**設計根拠**: 発語なし→音声模倣→単語の段階的アプローチ（ABA/VB原則）

#### ビデオモデリング + 音声模倣

```
ゲームフロー:
1. AI生成キャラクター（同年齢風）が画面に表示
2. キャラクターが口を動かして音を出す（リップシンク）
   例: 「あー」（母音から開始）
3. マイクアイコンが光る → 子どもが真似して声を出す
4. 音声認識ON: AIが判定 → 「すごい！」
   音声認識OFF: 親が「できた」ボタン → 「すごい！」
5. スター + キャラクターが拍手
```

#### 発声促進の段階設計
```
Level 1: 母音（あ、い、う、え、お）
Level 2: 子音+母音（ま、ぱ、ば = 口唇音から）
Level 3: 単語（ママ、パパ、バナナ等、反復音を含む語）
Level 4: 二語文（「ママ、きて」「りんご、ちょうだい」）
```

#### 音声認識の仕様
- API: SpeechSuper（日本語対応、音素レベル判定）
- ON/OFF切替: 設定画面にトグル
- OFF時: 親タップ方式に自動切替
- 初回表示:
```
「音声認識機能について」
お子さまの声を聞いて、発音をチェックする機能です。
発話の特性によっては正しく認識できない場合があります。
うまく機能しない場合は、設定からOFFにできます。
       [OK、使ってみる]    [OFFにする]
```

---

### 3C. 構音練習（発語あり、発音改善が必要な子ども向け）

#### ビデオモデリング + 発音チェック

```
ゲームフロー:
1. ターゲット語の絵が表示（例: 「りんご」の絵）
2. AI生成キャラクターがお手本を言う（リップシンク付き）
3. 「いっしょに言ってみよう！」（マイクアイコン光る）
4. 子どもが発声
5. 音声認識ON → スコア表示（星1-3、パーセントは見せない）
   音声認識OFF → 親が判定
6. 報酬 + 次の語へ
```

#### 対象音（日本語の構音発達順に配置）

| Level | 対象音 | 例語 | 発達目安年齢 |
|-------|--------|------|------------|
| 1 | 母音（あいうえお） | あめ、いぬ | 2-3歳 |
| 2 | 口唇音（ま、ぱ、ば） | ママ、パパ、バナナ | 2-3歳 |
| 3 | 歯茎音（た、だ、な） | たまご、なす | 3-4歳 |
| 4 | 軟口蓋音（か、が） | かめ、がっこう | 3-4歳 |
| 5 | 摩擦音（さ、し、す） | さかな、しか | 4-6歳 |
| 6 | 弾き音（ら行） | りんご、ライオン | 5-7歳 |

## 7. 反省点と対策

| 問題（Crystal Cavern v1.0） | 本設計での対策 |
|---------------------------|-------------|
| ルールが予告なく急変 | Level 1-3はルール切替なし。Level 4+で5-10秒の静的予告 |
| スピードが連続加速 | 速度変化なし。難度は選択肢の数・複雑さで調整 |
| チュートリアルが文字ベース | 完全アニメーション主導、4段階プロンプティング削減法 |
| キャッチゾーンが不明確 | タップ対象は明確にハイライト、64×64px以上 |
| 時間制限でストレス | 時間制限なし、self-paced |
| エラー時の否定的フィードバック | サイレント修正（エラーレス学習）、スコア減点なし |
| 感覚過負荷のリスク | Minecraft/どうぶつの森を参考にしたくすんだカラーパレット |
