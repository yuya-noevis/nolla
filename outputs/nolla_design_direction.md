# Nolla デザイン方向性・ベンチマーク指示書

**目的**: 開発者が別ターミナルで作業する際に、デザインの意図と方向性を正確に引き継ぐためのドキュメント
**注意**: このドキュメントの内容は全て科学的根拠に基づいている。個人の好みではない。
**確定日**: 2026-04-10（Mario Galaxy Movie風 宇宙テーマに確定）

---

## 1. 画面の向き

**横向き（ランドスケープ）固定**

- 子ども向けゲーム・療育コンテンツは横型が業界標準（Otsimo、Jade ND、市販知育アプリ全て横型）
- ゲーム画面が広く使える
- タブレットスタンドとの相性が良い
- フロントカメラで眼球データを取得するため、カメラ位置が安定する横持ちが適切

## 2. デザインのベンチマーク

### 世界観: Mario Galaxy Movie風の宇宙テーマ

**コンセプト**: 「明るく鮮やかな宇宙の中に、ゲームごとのテーマ惑星が浮かんでいる」

Super Mario Galaxy Movieのビジュアル言語を基盤とする。暗い宇宙背景の中に宝石のような発光要素が浮かぶ構造。背景が暗いからこそ発光要素が映え、「怖くない夜空」「夜のお祭り」感覚を実現する。

### なぜこの方向性がASD児に適するのか

| 成功要因 | Mario Galaxy的表現 | ASD児への効果 |
|---------|-------------------|-------------|
| **予測可能性** | 宇宙空間は常に同じ構造。星の配置は固定 | ルール不変、安心感 |
| **視覚的シンプルさ** | 暗い背景に浮かぶ明確な要素。主役が際立つ | 情報の優先順位が明確 |
| **自分の世界を作る** | 惑星を巡る冒険感 | 報酬ショップ + マイルーム |
| **コレクション** | 星を集める、惑星を巡る | アイテムコレクション + 図鑑 |
| **即座フィードバック** | 発光・パーティクル・リムライト | 200ms以内の複合フィードバック |
| **非競争** | 一人で宇宙を探検 | 他者との比較なし |

根拠:
- ASD児の85%が色彩を高い強度で知覚（PMC 2022）
- ASD児は14ヶ月から幾何学的刺激に親和性（JAMA Psychiatry）
- コレクション行動はASD児の25%に見られる（Springer 2024）
- 暗い背景 + 多数の小光源 = 好奇心喚起。恐怖ではない（Galaxy Movie分析）

### 旧ベンチマーク（参考のみ）

Minecraft・どうぶつの森・Robloxの成功要因（予測可能性、コレクション、非競争等）はそのまま維持。ビジュアルスタイルのみMC+AC風からGalaxy風に変更。

## 3. やってはいけないこと

| NG | なぜダメか | 代わりにやること |
|----|----------|-------------|
| **蛍光色** | 視覚的攻撃性が高い | グロウ（拡散光）で発光を表現 |
| **点滅・フラッシュ** | 光過敏+てんかんリスク | 星は静止 or ゆっくりフェード |
| **暗すぎるシーン** | 恐怖につながる | ネイビーの最暗値を#0A0A2Eに制限。暖色光源を必ず配置 |
| **縦向きUI** | 子ども向けゲームは全て横型 | 横向き固定 |
| **リアルな人間キャラ** | 不気味の谷。ASD児は幾何学的シンプルさを好む | デフォルメキャラ |
| **文字だらけ** | 知的障害があると文字認識ができない | アイコン+色+形+アニメーションで意味を伝える |
| **フラットUI** | 療育アプリ感が出る | グロウ・リムライト・被写界深度で奥行き |
| **小さいボタン** | 運動制御の困難がある子が押せない | 最小48x48px、3-8歳は64x64px推奨 |

## 5. 世界観: 球体惑星（星）

### コンセプト: 「宇宙を旅して星を巡る」

建物を廃止し、各ゲームを象徴する**テーマ球体惑星（星）**に変更。Mario Galaxyの小惑星を参照。子どもは「星を選ぶ」＝「ゲームを選ぶ」。

**重要な方針**: 星の**上にオブジェを載せる**のではなく、**球体自体の形状・テクスチャ・色・光り方でそのゲームの本質を表現する**。

### 4つの惑星

| 星の名前 | ゲーム | 球体の表現 | ネビュラ主色 | アクセント光 |
|---------|--------|-----------|-------------|-------------|
| カード星 | 神経衰弱 | ラベンダー〜ディープパープルのベルベット質感。スート模様の浮き彫り。金色の結晶が生えている | ディープパープル→マゼンタ | ゴールドの結晶光 |
| いろわけ星 | 分類ソーティング | パッチワーク状に3-4色に色分け（オレンジ/ライムグリーン/スカイブルー/ピンク）。各セクション異質感 | ティール→エメラルド | アンバーの境界光 |
| さがし星 | 視覚探索 | 深いティール〜エメラルドの密生した草地。小さな色違いのモノが隠れている。レンズ状の結晶 | シアン→ディープブルー | ホワイトの結晶レンズ光 |
| ひかり星 | Corsi Block | ダークネイビーの岩質。グリッド状の光の脈（シアン〜コバルト）。キューブ状結晶 | インディゴ→コバルト | シアンのグリッド脈光 |

### 球体惑星の共通ルール

- 直径: 画面高さの50-60%（ホーム画面表示時）
- 球体の周囲に薄いアトモスフィアリング（大気圏のグロウ。テーマ色。10-15% opacity）
- 球体の低高度に2-3個の小さな雲が漂う（ピンク〜白、半透明）
- 完全な真球でなくてよい。微妙な歪み・凸凹が個性
- 小さな装飾（結晶、キノコ、草、光の粒）は球面から自然に存在する表現ならOK
- 重力は星ごと（全ての表面要素が球体中心に向かって生えている）
- 星の周囲に1-2個の小さな衛星岩が浮遊

### 報酬・コレクション

```
ゲームプレイ → スター獲得
  |
報酬ショップ（スターとアイテムを交換）
  -- 小アイテム: スター3個（机、椅子、花等）
  -- 中アイテム: スター10個（ベッド、テレビ、ペット等）
  -- 大アイテム: スター30個（特別な家具、乗り物等）
  |
マイルーム（集めたアイテムで自分の部屋を飾る）
  -- アイテムを自由に配置
  -- 壁紙・床を変更
  -- 図鑑で一覧表示（コレクション要素）
```

## 6. ライティング

ライティングが「怖くない夜空」を成立させる最大の要因。

| 技法 | 詳細 | 再現指示 |
|------|------|----------|
| **リムライト** | 全オブジェクトの輪郭にシアン〜ホワイトの縁取り光 | 建物・UIパネルの外周に1-2pxのグロウ |
| **ポイントライト散在** | 暖色+寒色の小光源が5-8個点在 | 「夜のお祭り」感覚 |
| **ボリュメトリック雲** | ピンク〜ラベンダーの半透明雲が光を透過 | 画面端や下部に柔らかいグロウ雲 |
| **流れ星/光跡** | 斜め45度前後の細い光線が2-4本 | シアン〜ホワイト。動きの暗示 |
| **星のボケ** | 大小の星が被写界深度でボケ | 手前に大きいボケ星、奥に小さいシャープ星 |

**感情効果**: 暗い背景 + 多数の小光源 = 「夜祭り」「星を見に来た」感覚。恐怖ではなく好奇心を喚起。

## 7. UI構造

### ホーム画面（カルーセル）

```
+--------------------------------------------------+
|  [star count]                        [parent lock]|
|                                                   |
|           (小さいボケ惑星)                          |
|                                                   |
|     (ボケ)    ★ メイン惑星 ★    (ボケ)              |
|              (画面中央、大きく)                      |
|                                                   |
|              < o o o o >                          |
|              (ドットインジケータ)                    |
|                                                   |
+--------------------------------------------------+
```

- 中央にフォーカス中の惑星（大きく、シャープ）
- 左右に隣の惑星（小さく、ボケ）
- 下部にドットインジケータ（テキストなし）
- 左右にスワイプ用シェブロン

### ゲーム画面

- 惑星の上に「着陸」した視点
- 地面は球体の表面（各惑星のテーマ色・テクスチャ）
- 地平線が湾曲して見える（小さい惑星上にいる感覚）
- 上部75%は宇宙の空（ネビュラ、星、遠くの惑星）
- ゲーム要素（カード、アイテム等）は惑星表面に配置
- 宇宙背景はホーム画面より控えめ（ゲームに集中させるため彩度-30%）

### 画面遷移

- ゲーム選択→ゲーム開始: **ワープアニメーション**（黒背景+星が放射状に高速で流れる、1.2秒）
- ゲーム終了→ホーム: 逆ワープ or フェード

## 8. キャラクターデザイン

### 方向性: 幾何学的デフォルメ

- 丸みを帯びた柔らかいシルエット
- 感情表現: 笑顔、中立、困り顔の3段階のみ
- リアル禁止: 写実的な人間の顔はNG

### ナビキャラ
- アプリ全体を案内するメインキャラクターを1体設計
- ゲーム中の褒めフィードバックにも登場

## 9. 感覚オーバーロード防止ルール

| 要素 | NG | OK |
|------|-----|-----|
| アニメーション | 自動再生、ストロボ、フラッシュ | ユーザートリガー、200ms以上のスロー |
| 音声 | 突然の大きな音、BGMの自動再生 | 調整可能な音量、穏やかな効果音 |
| 情報密度 | 画面いっぱいの要素 | 十分な余白、主役が明確 |
| 形状 | シャープなコーナー、複雑な形 | 丸角、シンプル形状 |
| 色 | 蛍光色、高コントラストの急変 | グラデーション遷移、グロウ表現 |
| 宇宙背景 | ネビュラ+星の密度が高すぎる | ゲーム中は控えめ。ホーム画面のみフル |

## 10. 開発者への注意事項

1. **「療育アプリ」を作ろうとするな。「ASD児がハマるゲーム」を作れ。** 療育効果はゲーム設計の裏側で実現する
2. **Mario Galaxy Movieのスクリーンショットを見てからデザインしろ。** 発光、リムライト、ネビュラグラデーション、球体惑星の表現を体に染み込ませる
3. **安っぽいフラットUIは絶対NG。** グロウ、リムライト、被写界深度、ネビュラ背景で奥行きを出す
4. **横向き固定。縦にするな**
5. **文字を使うな。** ゲーム内のナビゲーション、チュートリアル、フィードバックは全てアイコン+アニメーション+音声で
6. **暗い≠怖い。** 暖色の小光源を必ず配置して「夜祭り」感覚を作る
