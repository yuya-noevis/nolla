# Nolla デザイン方向性・ベンチマーク指示書

**目的**: 開発者が別ターミナルで作業する際に、デザインの意図と方向性を正確に引き継ぐためのドキュメント
**注意**: このドキュメントの内容は全て科学的根拠に基づいている。個人の好みではない。

---

## 1. 画面の向き

**横向き（ランドスケープ）固定**

- 子ども向けゲーム・療育コンテンツは横型が業界標準（Otsimo、Jade ND、市販知育アプリ全て横型）
- ゲーム画面が広く使える
- タブレットスタンドとの相性が良い
- フロントカメラで眼球データを取得するため、カメラ位置が安定する横持ちが適切（特にiPadは横持ち時にカメラが上辺中央に来る）

## 2. デザインのベンチマーク

### なぜMinecraft・どうぶつの森・Robloxがベンチマークなのか

この3つはASD児に世界的に最も人気のあるゲーム。人気の理由は科学的に分析されている:

| 成功要因 | Minecraft | どうぶつの森 | Nollaに適用すべきこと |
|---------|-----------|------------|-------------------|
| **予測可能性** | ルール不変、物理法則一定 | 日課固定、同じキャラが同じ行動 | ゲームルール固定、毎回同じ画面構造 |
| **視覚的シンプルさ** | ピクセルアート・ブロック型 | パステル・丸みを帯びたデフォルメ | 幾何学的デフォルメ + くすんだ配色 |
| **自分の世界を作る** | 建築自由 | 島カスタマイズ | 報酬ショップ + マイルーム |
| **コレクション** | アイテム・ブロック収集 | 家具・虫・魚コレクション | アイテムコレクション + 図鑑 |
| **即座フィードバック** | ブロック配置=即結果 | 釣り=即結果 | 200ms以内の複合フィードバック |
| **非競争** | シングルプレイ可 | 勝ち負けなし | 他者との比較なし |

根拠:
- [Why autists like Minecraft - GamesLearningSociety](https://www.gameslearningsociety.org/why-do-autists-like-minecraft/)
- [Autistic ethnographic case study of Animal Crossing](https://systems.enpress-publisher.com/index.php/JGS/article/view/11616)
- ASD児の85%が色彩を高い強度で知覚（PMC 2022）
- ASD児は14ヶ月から幾何学的刺激に親和性（JAMA Psychiatry）
- コレクション行動はASD児の25%に見られる（Springer 2024）

## 3. やってはいけないこと（安っぽく見える原因）

| NG | なぜダメか | 代わりにやること |
|----|----------|-------------|
| **白背景にフラットなボタン** | 療育アプリ感が出て退屈。ASD児がハマらない | Minecraftのブロック感、どうぶつの森の温かみのある世界観を参考にした背景・UI |
| **原色（赤・黄・蛍光色）** | ASD児の85%が色彩を高強度で知覚。過刺激になる | くすんだ（低彩度の）カラフルさ。パステル寄り |
| **縦向きUI** | 子ども向けゲームは全て横型。画面が狭くてゲームにならない | 横向き固定 |
| **リアルな人間キャラ** | 不気味の谷。ASD児は幾何学的シンプルさを好む | デフォルメキャラ（どうぶつの森的な丸み or Minecraft的なブロック感） |
| **文字だらけ** | 知的障害があると文字認識ができない | アイコン+色+形+アニメーションで意味を伝える |
| **複雑なUI** | 1画面に情報が多いと感覚オーバーロード | 1画面の情報要素5個以下 |
| **小さいボタン** | 運動制御の困難がある子が押せない | 最小48×48px、3-8歳は64×64px推奨 |

## 4. カラーパレット

```
背景:       #F5F5F5 (ライトグレー) — 白すぎず目に優しい
プライマリ:  #4A7BA7 (カームブルー) — どうぶつの森の空・海を意識
セカンダリ:  #6BA47A (ソフトグリーン) — どうぶつの森の草原を意識
アクセント:  #D4A574 (ウォームベージュ) — 木の温かみ
テキスト:    #333333 (ダークグレー) — 親向け画面のみ
成功:       #6BA47A (グリーン) + チェックマーク
```

**重要**: これはベースカラー。ゲーム世界内では、このベースの上にMinecraft的なブロック色やどうぶつの森的なパステルのバリエーションを使って「くすんだカラフルさ」を実現する。単色の寂しい画面にしない。

## 5. キャラクターデザイン

### 方向性: 幾何学的デフォルメ

- **Minecraftの影響**: 対称的、シンプル、ブロック的。複雑な表情筋がなく、単純な目と口。認知負荷が低い
- **どうぶつの森の影響**: 丸みを帯びた柔らかいシルエット。愛らしいがミニマルな表情
- **感情表現**: 笑顔、中立、困り顔の3段階のみ。複雑な感情は避ける
- **リアル禁止**: 写実的な人間の顔はNG。不気味の谷でASD児が拒否反応を示す

### ナビキャラ
- アプリ全体を案内するメインキャラクターを1体設計
- ホーム画面で「〇〇ちゃん、きょうもあそぼう！」と表示
- ゲーム中の褒めフィードバックにも登場

## 6. 世界観

### コンセプト: 「自分だけのやさしい世界」

Minecraftとどうぶつの森が愛される最大の理由は「自分の世界を作れること」。Nollaでもこれを実現する:

```
ゲームプレイ → スター獲得
  ↓
報酬ショップ（スターとアイテムを交換）
  ├ 小アイテム: スター3個（机、椅子、花等）
  ├ 中アイテム: スター10個（ベッド、テレビ、ペット等）
  └ 大アイテム: スター30個（特別な家具、乗り物等）
  ↓
マイルーム（集めたアイテムで自分の部屋を飾る）
  ├ アイテムを自由に配置
  ├ 壁紙・床を変更
  └ 集めたアイテムを図鑑で一覧表示（コレクション要素）
```

マイルームのビジュアルも、どうぶつの森の部屋のような温かみのある空間にする。

### ゲーム画面の雰囲気

- 背景は単色ではなく、世界観のある背景（草原、空、木のテクスチャ等）をくすんだトーンで
- ゲーム要素（箱、カード等）にもテクスチャ・影・奥行きを持たせる
- 「療育ツール」ではなく「楽しいゲーム」に見えることが最重要
- 参考: Otsimoのゲーム画面（シンプルだが世界観がある）

## 8. 感覚オーバーロード防止ルール

| 要素 | NG | OK |
|------|-----|-----|
| アニメーション | 自動再生、ストロボ、フラッシュ | ユーザートリガー、200ms以上のスロー |
| 音声 | 突然の大きな音、BGMの自動再生 | 調整可能な音量、穏やかな効果音 |
| 情報密度 | 画面いっぱいの要素 | 1画面5要素以下、十分な余白 |
| 形状 | シャープなコーナー、複雑な形 | 丸角、シンプル形状 |
| 色 | 赤、黄、蛍光色、高コントラストの組み合わせ | くすんだパステル、低彩度 |

## 9. 開発者への注意事項

1. **「療育アプリ」を作ろうとするな。「ASD児がハマるゲーム」を作れ。** 療育効果はゲーム設計の裏側で実現する。子どもが見る画面は「楽しいゲーム」でなければならない
2. **Minecraft、どうぶつの森のスクリーンショットを見てからデザインしろ。** くすんだカラフルさ、幾何学的シンプルさ、温かみのある世界観を体に染み込ませてから作業する
3. **安っぽいフラットUIは絶対NG。** テクスチャ、影、奥行き、世界観のある背景を使う。ただし情報量は抑える
4. **横向き固定。縦にするな**
5. **文字を使うな。** ゲーム内のナビゲーション、チュートリアル、フィードバックは全てアイコン+アニメーション+音声で
