## 1. 確定済み設計判断

| # | 項目 | 判断 | 根拠 |
|---|------|------|------|
| 1 | コンテンツ分離 | 子ども画面では認知ゲームとSTを区別しない。全て「建物」 | 認知負荷軽減 |
| 2 | ナビゲーション | カルーセル型。全画面1建物表示 | Duo ABC参考 |
| 3 | コンテンツ階層 | エリア（建物）→ ゲーム一覧 の2階層 | 将来300種対応 |
| 4 | 報酬獲得 | スター通貨 → ショップ交換 | ABAトークンエコノミー |
| 5 | コレクション | 図鑑+マイルーム（グリッド配置+タッチ反応） | 能動的配置が必要 |
| 6 | 強化スケジュール | CRF → VR3-5移行 | ABA研究 |
| 7 | アイテム出し分け | 限定的興味ベースのパーソナライズ | fMRI研究 |
| 8 | 親子分離 | PIN/生体認証で完全分離 | 業界標準 |
| 9 | 初期プロファイル | 保護者質問で取得（必須） | 眼球データのラベリング精度に直結 |
| 10 | AI差別化 | 親向けAI療育サポートエージェント | 競合との最大の差別化 |
| 11 | ホーム画面 | 要素5以下。ボタン最小化 | 認知負荷削減 |
| 12 | ゲーム内文字 | 一切使わない。アイコン+視覚のみ | 知的障害児は文字認識不可 |

## 3. ホーム画面レイアウト（M1）

### 修正方針: 要素を最小限に

```
ホーム画面（横向き 1194×834px）
┌──────────────────────────────────────────────────┐
│                                                    │
│  [ナビキャラ]                         ★128  [🔒]  │
│                                                    │
│                                                    │
│                                                    │
│  [←]     全画面: 建物の背景画像          [→]       │
│          （V3 MCスタイル）                          │
│                                                    │
│                                                    │
│                                                    │
│              （建物名の看板）                        │
│          ● ○ ○ ○                                  │
│                                        [マイルーム] │
└──────────────────────────────────────────────────┘

要素数: 5
  1. ナビキャラ（左上）
  2. スター数（右上）
  3. カルーセル建物（中央全画面）+ 矢印
  4. ドットインジケータ（下部中央）
  5. マイルーム（右下）
```

### v2からの変更点

| 削除 | 理由 |
|------|------|
| 上部[ゲーム][ことば]切替アイコン | STも建物としてカルーセルに統合 |
| 下部[図鑑][ショップ] | マイルーム内にタブとして統合 |
| 気分アイコン | 認知負荷に寄与しない |

## 4. 報酬演出画面（R1）

### 修正方針: 文字ゼロ、ボタン2つ

```
┌──────────────────────────────────────────────────┐
│                                                    │
│                                                    │
│          ★ ★ ★ （スターが飛ぶアニメ）              │
│                                                    │
│     [★] ──→ [?シルエット] ──→ [ショップアイコン]    │
│     （視覚ガイド: アイコンのみ、文字なし）           │
│                                                    │
│        [▶ アイコン]          [🏠 アイコン]          │
│        （もう1回）           （ホームへ）            │
│                                                    │
└──────────────────────────────────────────────────┘

要素:
  - スター獲得アニメーション（自動再生、200ms以内に開始）
  - 視覚ガイド: スター→?→ショップ（アイコン3つを矢印でつなぐ）
  - ボタン2つのみ:
    [▶] = もう1回遊ぶ（再生アイコン）
    [🏠] = ホームに戻る（家アイコン）
  - ショップ/マイルームへはホーム経由
  - 文字は一切なし
```

## 6. AI戦略

### 6A. データ取得アーキテクチャ

```
デバイス上のみ（サーバーに送信しない）:
  ├ カメラ生映像/画像
  └ 眼球追跡の生座標（リアルタイム処理用）

Nollaサーバーに送信（取得する）:
  ├ ゲームプレイデータ
  │  ├ 正答率、反応時間、エラーパターン
  │  ├ 試行ごとの詳細ログ
  │  └ 学習曲線データ
  ├ 行動メトリクス
  │  ├ 注意持続時間
  │  ├ タップ精度（位置ズレ、圧力）
  │  ├ セッション長・頻度
  │  └ 操作パターン（速度、間隔）
  ├ 眼球データ（処理済み統計値）
  │  ├ 注視パターン
  │  ├ サッケード特性
  │  └ 固視時間分布
  ├ 保護者質問の回答（初期プロファイル）
  └ AIエージェントとのやりとりログ
```

### 6B. AI療育サポートエージェント（親向け）

**Nollaの最大の差別化要素。**

```
入力:
  全取得データ + 子どもの初期プロファイル + 学術的知見

出力（4種類）:

  1. 認知特性レポート（週次更新）
     子どもの強み・課題・傾向を専門的に分析
     例: 視覚的マッチングの処理速度、ルール切替時の認知負荷パターン

  2. 成長の変化検出（プロアクティブ通知）
     データの変化を検出して自動通知
     例: 注意持続時間の向上傾向、新しいスキルの獲得兆候

  3. 家庭での療育アドバイス
     ゲームで鍛えているスキルを日常に般化させる具体的方法
     例: 認知的柔軟性の成長に合わせた家庭での実践法

  4. 危険信号の検出
     パフォーマンス低下・利用パターンの異変を検出
     例: 急激な成績低下、セッション回避パターン

技術: ゲームプレイデータ → 構造化KPI → LLM API → 親向け平易な日本語
```

### 6C. ゲームプレイからの継続的プロファイリング

```
保護者質問（初期プロファイル）
    ↓
ゲームプレイデータ蓄積
    ↓
MLモデルで継続的精緻化:
  ├ 認知スキルレベルの推定精度向上
  ├ 学習速度の個人差モデリング
  ├ 最適難度の精密調整
  └ 発達軌跡の予測（4週間分のデータ蓄積後）
```

### 6D. 将来のAI拡張（MVP後）

| 機能 | 時期 | 内容 |
|------|------|------|
| パーソナライズドコンテンツ | Phase 2 | テーマセット出し分け→動的素材差替 |
| 予測的介入提案 | Phase 2 | 発達軌跡予測に基づく先回りアドバイス |
| 動的コンテンツ生成 | Phase 3 | AI生成ゲーム（技術成熟後） |
| 研究データAPI | Phase 3 | 匿名化データの研究機関提供 |

## 7. 画面フロー

### 7A. 初回起動

```
[S1] スプラッシュ (3秒)
  ↓
[S2] アカウント作成
  ↓
[S3] お子さま情報入力 (2分)
  ↓
[S4] 同意取得
  ↓
[S5] 端末引き渡し
  ↓ ── 操作者交代 ──
[M1] ホーム画面
  ↓
  ナビキャラがチュートリアル的に最初の建物へ誘導
  ↓
[M3] チュートリアル → [M4] 初回ゲームプレイ
  ↓
[R1] 報酬演出 → [M1] ホーム
```

### 7B. 通常利用（メインループ）

```
[M1] ホーム画面
  │
  ├── 建物タップ ──→ [M4] ゲームプレイ (10-15試行)
  │                     ├ 60分連続 → [U1] 休憩提案
  │                     └ セッション完了 → [R1] 報酬演出
  │                                          ├ [▶] → [M4] もう1回
  │                                          └ [🏠] → [M1] ホーム
  │
  ├── [マイルーム] ──→ [R2] マイルーム統合画面
  │                     ├ おへやタブ: 配置+タッチ反応
  │                     ├ ショップタブ: スター交換
  │                     └ ずかんタブ: コレクション一覧
  │
  └── [🔒] ──→ [P1] PIN認証 → 親向け画面

途中離脱: [← 戻る] → 確認アイコン → [M1] ホーム（途中分スター獲得済み）
```

### 7C. 親向けフロー

```
[P1] PIN/生体認証
  ↓
[親向けダッシュボード]
  ├ [P2] 今日の記録: プレイ時間・ゲーム別評価
  ├ [P3] 成長の記録: 認知スキル推移+AI分析コメント
  ├ [P4] AI療育サポート: AIエージェント対話画面
  │   ├ プロアクティブ通知一覧
  │   ├ 家庭療育アドバイス
  │   ├ 認知特性レポート
  │   └ 質問・相談（チャット形式）
  └ [P5] 設定
      ├ セッション時間制限
      ├ 休憩提案（提案/強制切替）
      ├ 眼球データ ON/OFF
      ├ 音声認識 ON/OFF
      ├ コンテンツ表示/非表示
      ├ ゲームアンロック（段階的/全開放）
      ├ 実世界報酬連携（将来）
      └ プロフィール編集
```

## 10. MVP開発フェーズ

### Phase 1: コア体験
1. ホーム画面（カルーセル、4建物）
2. 4ゲーム実装（神経衰弱/分類ソーティング/視覚探索/Corsi Block）
3. 報酬演出画面
4. マイルーム統合（おへや+ショップ+ずかん）
5. 保護者質問+同意取得
6. 親向けPIN認証+簡易設定

### Phase 2: コンテンツ+AI
7. ST系コンテンツ（ことば、おはなし）
9. AI療育サポートエージェント（親向け）
10. ゲームプレイデータ分析+認知プロファイリング
11. 親ダッシュボード（成長記録+AI分析）

### Phase 3: データ+磨き込み
12. 眼球データ取得
13. 行動データ取得（カメラAI）
14. 適応型難度調整の精密化
15. 音声認識
16. パーソナライズドテーマ出し分け
