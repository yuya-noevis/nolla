# Nolla × Bluey 2.5D スタイルガイド

**作成日**: 2026-04-09  
**適用範囲**: Nollaゲーム世界内の全キャラクター、背景、UI演出  
**コンセプト**: 「オーストラリア郊外のパステルカラー家族世界。手描き風、温かみ、予測可能性」

---

## 1. 世界観コンセプト

ASD児に世界的に最も支持されているBlueyの視覚言語（手描き風パステル、幾何学的で丸い人物、穏やかなアニメーション）を採用。Nollaの療育機能を損なわないビジュアル階層と予測可能性を保証する。

**キーワード**: 温かみ / 予測可能性 / パステル / 手描き感 / 親しみやすさ / オーストラリア郊外

## 3. キャラクター設計ルール

### 3A. 基本プロポーション

- **頭身**: 2.5-3頭身（Blueyと同等）
- **頭部**: 円形〜楕円形。直線的な角を避ける
- **体**: 楕円形で構成（矩形でない）
- **四肢**: 丸みのある円柱。関節は丸いドット
- **足**: 簡略化（靴または肉球状）

### 3B. 輪郭線・線の太さ

- **外輪郭**: 2-3px黒線（#1D1D1D）
- **内部ディテール**: 1px黒線（目、口、服飾）
- **マテリアル分割**: 色の切り替えで表現、線不要
- **手描き風**: ベクターは完全直線ではなく、わずかなブレ（±0.5px振動）を不規則に入れる

### 3C. 目

- **形**: 大きな円形または楕円形（全体の15-20%）
- **瞳**: 黒丸。反射は小さい白点（1-2px）
- **まつげ**: 上部のみ、3-4本の短い線
- **目の輝き**: 予測可能性を損なわないため、常に同じ位置

### 3D. 口・表情（3段階のみ）

| 感情 | 表現方法 | 使用箇所 |
|------|---------|---------|
| 笑顔 | 口が弧形で上向き。目が細くなる（目尻が上がる） | 正解、報酬、友好的交互作用 |
| 中立 | 口が直線。目は通常。眉は水平 | 待機、説明、ニュートラル |
| 困り顔 | 口が弧形で下向き。眉は内側上がり（八の字）。涙の玉 | 不正解時（サイレント修正前）、考え中 |

**重要**: 感情は厳密に3種のみ。中間状態や複雑な表情は使わない（予測可能性）。

### 3E. 衣装・特徴付け

- **色数**: 1キャラ最大3色（肌色 + 衣装2色）
- **パターン**: ドット、ストライプOK。複雑な幾何学模様は避ける
- **アイデンティティ**: 帽子、スカーフ、バッジで識別（色より形）

## 4. 背景・ステージ設計ルール

### 4A. 構成原則

| レイヤー | 役割 | 詳細 |
|---------|------|------|
| 背景（奥） | 空 | 単色（#87CEEB） or グラデーション（わずか） |
| 遠景 | 丘、遠い建物 | 10-15%の透明度で淡色化 |
| 中景 | メイン建物、主要オブジェクト | 完全不透明、中心位置 |
| 前景 | 地面、手前オブジェクト | 装飾、フレーミング |

### 4B. 色彩・彩度

- **全体彩度**: 60-75%（明るく温かいが派手でない）
- **遠景**: さらに淡く（彩度-10%、明度+10%）
- **地面**: 草緑（#7CBA5D）または土色（#8B6B4A）
- **手描き感**: SVG/Lottieでも完全直線を避け、わずかなぎざぎざ感を入れる

### 4C. パラレックス（控えめに）

- **有効な場合**: 画面スクロール時のみ。通常遊戲中は**使わない**
- **実装**: 遠景は1/3スクロール速度、中景は1/1
- **目的**: 深度感、退屈さ軽減。ASDの予測可能性を損なわない程度

### 4D. オーストラリア郊外設定の要素

- **ユーカリの木**: リファレンスに従い、渦巻き状の幹
- **ポストボックス**: 柱の上に小さな箱（赤または青）
- **フェンス**: ピケットフェンス（白、シンプル）
- **光**: 暖色系の午後の光（影は淡い紫/青）

## 5. 2.5D実装ガイド

### 5A. 定義

「2.5D」= 2D資産を複数レイヤーに分割し、疑似3D空間に配置。パラレックス効果で奥行き表現。Bluey（CelAction2D）と異なり、SVG/CSS/Rive/Lottieで実装可能。

### 5B. 推奨テクノロジー

| ツール | 用途 | 理由 |
|--------|------|------|
| **Rive** | キャラクター骨組みアニメーション | 状態機械、軽量、Web最適化 |
| **Lottie** | 短い演出（パーティクル、フィードバック） | JSON形式、軽量、Bodymovin互換 |
| **SVG + CSS** | 背景レイヤー、静的オブジェクト | パラレックス制御容易、ベクター品質 |
| **WebGL（Three.js）** | 高度な3D効果が必要な場合 | MVP段階では不要 |

### 5C. レイヤー構成（具体例：コテージゲーム画面）

```
Layer 5 (前景)   : 地面、装飾品 (SVG, static)
Layer 4 (中景)   : コテージ建物、柱 (SVG, parallax 1/1)
Layer 3 (中~遠景): キャラクター(Rive 骨組み + Lottie 反応)
Layer 2 (遠景)   : 木、丘 (SVG, parallax 1/3)
Layer 1 (背景)   : 空 (CSS gradient or SVG)
```

### 5D. アニメーション仕様

| 場面 | 動作内容 | 期間 | 推奨ツール |
|------|---------|------|------------|
| **待機** | キャラ呼吸アニメ（わずかな拡縮） | ∞ loop, 2-3s周期 | Rive |
| **正解反応** | ジャンプ + 回転 + スター飛び出し | 0.5s動き → 1-2s静止 | Rive + Lottie |
| **不正解修正** | サイレント位置調整 + 柔らかい光 | 0.3s | CSS + Lottie |
| **ラウンド完了** | Bluey式クロージング（2-3秒の静画） | 2-3s | Lottie/SVG |

### 5E. フレームレート・タイミング

- **フレームレート**: 24fps（Blueyと同等、なめらか + 計算軽量）
- **アニメーション曲線**: Ease-in-out（予測可能性を損なわない）
- **フィードバック遅延**: 最大200ms（nolla_game_mechanics_design.md）

## 9. よくある質問

**Q. Blueyのカラーパレットをそのまま使えば十分では？**  
A. ビジュアル言語は採用するが、色はNollaの既存パレット（nolla_color_regulation.md）と統合する必要がある。蛍光色禁止、点滅禁止はBlueyより厳格。

**Q. CelAction2Dを使わなくても大丈夫？**  
A. MVP段階ではRive/Lottie/SVGで十分。CelAction2Dは放送テレビ向けで、Web最適化観点からは過剰スペック。

**Q. パラレックスは完全に禁止？**  
A. 通常遊戲中は不要。スクロール画面（ステージセレクト等）に限定して使用可能。予測可能性を損なわない範囲で。

**Q. キャラクター動きの期間（0.5s→1-2s→2-3s）の根拠は？**  
A. 心理学：「Blueyのクロージング」（2-3秒）と「ゲーム内フィードバック」（0.5s動き＋1-2s静止）のバランス。過度なアニメーション疲労を避ける。
