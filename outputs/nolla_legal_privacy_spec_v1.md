# Nolla 法務・プライバシー仕様 v1

## 0. 重要な前提

**本書は弁護士監修前のドラフト**。Phase 0 完了前に**児童プライバシー法に詳しい弁護士の最終レビュー**が必須(C4-CRITICAL)。
特にCOPPA(米国)/GDPR(EU)/個人情報保護法(日本)の同時適合は法域の解釈差があり、専門家確認なしに公開してはいけない。

## 1. データ取得の整理

### 1.1 Nollaが扱うデータ種別

| カテゴリ | データ | 法的分類 |
|---|---|---|
| **アカウント** | 親のメール、パスワード(ハッシュ)、OAuth ID | 個人情報 |
| **児童プロフィール** | 年齢、性別、診断有無、IQ帯、ニックネーム | 児童個人情報(=要保護) |
| **プレイデータ** | 試行ごとの正誤・反応時間・難度パラメータ | 行動データ |
| **生体・センサーデータ** | タッチ圧力/半径/軌跡、デバイスモーション(ジャイロ/加速度) | **要配慮個人情報相当** |
| **音声(将来)** | 音声録音(SpeechSuper連携時) | **要配慮個人情報** |
| **画像(将来)** | カメラ映像(MediaPipe Face Mesh) | **要配慮個人情報** |
| **NCIスコア** | 認知機能推定値 | **準医療情報相当** |

### 1.2 リスクレベル分類

- **Critical**: 児童の生体データ・音声・画像・NCIスコア → 最高水準の保護必須
- **High**: 児童プロフィール・プレイデータ → 暗号化必須、利用目的明示
- **Medium**: 親アカウント情報 → 標準的な個人情報保護

## 2. 法域別対応チェックリスト

### 2.1 日本: 個人情報保護法(APPI)

- [ ] プライバシーポリシー作成・公開
- [ ] 個人情報取扱事業者としての届出(必要なら)
- [ ] 利用目的の明示(取得時)
- [ ] **本人同意の取得**(児童の場合は親権者)
- [ ] **要配慮個人情報の取得には明示同意**(生体データ/医療情報相当)
- [ ] 安全管理措置(技術的・組織的)
- [ ] 第三者提供時の同意取得
- [ ] 開示・訂正・利用停止請求への対応窓口設置
- [ ] 漏洩時の本人通知・個人情報保護委員会への報告体制
- [ ] 海外移転時の同意・移転先国の保護水準確認(SupabaseはAWSリージョン依存)
- [ ] **改正APPI(2022)対応**: 個人関連情報、Cookie等の取扱

### 2.2 米国: COPPA (Children's Online Privacy Protection Act)

13歳未満の児童データ取扱に厳格な規制。Nollaは3-18歳対象のため**適用必須**。

- [ ] **検証可能な親同意**(Verifiable Parental Consent, VPC)の取得
  - メール+クレジットカード or 政府ID確認 or 電話 or PDFサイン
  - 単純なチェックボックスはNG
- [ ] プライバシーポリシーに以下を明記:
  - 何のデータを取るか
  - どう使うか
  - 第三者にどう渡すか
  - 親が削除/開示請求できる方法
- [ ] **データ最小化原則**: ゲーム機能に必要なデータのみ取得
- [ ] **広告ターゲティング禁止**: 児童データを広告目的で使用してはいけない
- [ ] 第三者(SDK/分析ツール)もCOPPA準拠のものだけ使用
  - Supabase: COPPA準拠確認必要
  - Vercel: 分析機能オフ推奨
  - Google Analytics: 児童データには使用禁止
- [ ] Safe Harbor認定取得検討(kidSAFE Seal等)

### 2.3 EU: GDPR (General Data Protection Regulation)

EU圏ユーザーがいる場合適用。グローバル展開予定なら必須準拠。

- [ ] **児童(16歳未満、加盟国により13-16歳)の同意は親権者経由**
- [ ] 法的根拠の明示(同意/契約/正当な利益等)
- [ ] **データ保護責任者(DPO)選任**(児童データの大規模処理の場合)
- [ ] **データ保護影響評価(DPIA)実施**(生体データ大規模処理に該当)
- [ ] データポータビリティ権(エクスポート機能)
- [ ] 忘れられる権利(削除機能)
- [ ] アクセス権、訂正権、処理制限権
- [ ] **データ侵害の72時間以内通知義務**
- [ ] EU代理人選任(EU圏内に拠点がない場合)
- [ ] Cookie同意バナー(EU圏ユーザー対象)

### 2.4 法域横断: その他

- [ ] **学校導入時のFERPA(米国教育記録法)対応**(B2B展開時)
- [ ] **HIPAA非該当の確認**(医療データ扱いにならないようNCIの位置づけ整理)
- [ ] 中国・韓国等、進出予定国の児童保護法調査(Phase 2)

## 3. プライバシーポリシー雛形(日本語版)

> 以下は弁護士監修前の**テンプレート**。実際の公開には専門家レビュー必須。

```markdown
# Nolla プライバシーポリシー

最終更新日: YYYY-MM-DD
事業者: 株式会社○○(住所/連絡先)
個人情報保護管理責任者: ○○○○

## 1. 取得する情報

Nollaは以下の情報を取得します:

### 1.1 親アカウント情報
- メールアドレス、パスワード(暗号化保存)
- 認証用OAuth ID(Apple/Googleでログインの場合)

### 1.2 お子様に関する情報
- 年齢、性別、診断の有無(任意)、ニックネーム
- ゲームプレイの記録(正誤、反応時間、操作データ)
- タッチ操作の特徴(力の強さ、軌跡、タイミング)
- デバイスの動き(タブレットの傾き、加速度)
- お子様の認知機能スコア(NCI)

### 1.3 デバイス情報
- OS、ブラウザ、画面サイズ、IPアドレス(セッション管理目的)

## 2. 利用目的

取得した情報は以下の目的のみに使用します:

1. お子様に最適な難度のゲームを提供するため
2. 認知機能の発達を測定し、保護者の方にお伝えするため
3. アプリの品質改善と研究のため(個人を特定できない形で)
4. 不正利用の検知と防止のため

**広告配信には一切使用しません。**
**第三者への販売は行いません。**

## 3. 第三者への提供

以下の場合を除き、個人情報を第三者に提供しません:

1. 法令に基づく場合
2. 人の生命・身体・財産の保護のために必要な場合
3. 利用者の同意がある場合
4. 業務委託先(サーバー運用、決済処理等)に必要な範囲で提供する場合
   委託先一覧:
   - データ保管: Supabase(AWS東京リージョン)
   - 決済処理: ○○
   - エラー監視: ○○

## 4. 保管期間

- 親アカウント情報: アカウント削除まで
- お子様の学習データ: 親権者の同意のもと、削除請求まで保管
- 操作データ・センサーデータ: 同上(研究目的での保管)
- 削除請求された場合: 30日以内に完全削除

## 5. お子様の権利と保護者の方の権利

保護者の方は、お子様の情報について以下の権利を行使できます:

- **開示請求**: 保存されているデータの確認
- **訂正請求**: 誤った情報の修正
- **削除請求**: 全データの削除
- **利用停止請求**: 一時的な利用停止
- **データのエクスポート**: 全データのダウンロード

請求方法: 設定画面「データ管理」または support@nolla.example.com まで

## 6. 安全管理措置

- 通信の暗号化(TLS 1.3)
- データベースの暗号化(AES-256)
- アクセス権限の最小化
- 定期的なセキュリティ監査
- 従業員教育

## 7. お子様のための特別な配慮

Nollaは子ども向けアプリです。以下の特別な保護を行います:

- 13歳未満のお子様のデータは、保護者の方の同意なしに取得しません
- お子様に直接マーケティング情報を送信しません
- お子様のデータを広告ターゲティングに使用しません
- お子様の個人情報を商業目的で第三者に共有しません

## 8. 国際的なデータ移転

データは原則として日本国内(東京リージョン)に保管します。
分析等で他地域のサーバーを使用する場合、適切な保護水準を確保します。

## 9. ポリシーの変更

本ポリシーは予告なく変更される場合があります。
重要な変更時はアプリ内通知でお知らせします。

## 10. お問い合わせ

[連絡先]
```

## 4. 同意フロー設計(MVP実装)

### 4.1 初回起動時の同意取得画面(S4)

現在の S4 同意取得画面を以下の構造に拡張する必要あり:

```
[画面 1] プライバシーポリシー全文
- スクロール必須
- 「確認しました」ボタンは最下部までスクロールするまで無効

[画面 2] 取得データの個別同意
☐ ゲームプレイの記録(必須)
☐ タッチ操作の特徴データ(必須)
☐ デバイスの動きデータ(必須)
☐ 研究目的での匿名利用(任意)
☐ 開発改善のためのエラーレポート(任意)

[画面 3] 親権者確認
□ 私はこのお子様の親権者または法的保護者です
□ 上記内容を理解し同意します
署名(タイプ): ___________
```

### 4.2 COPPA対応の検証可能な同意(米国展開時)

13歳未満ユーザー向けに以下のいずれかを実装:
- クレジットカード認証(課金時に確認)
- 電話認証
- 政府発行ID画像のアップロード(運転免許等)
- 親のメールに送られるリンクをクリック+追加質問

MVPは日本のみなら不要。**日本→英語(米国)展開時に必須実装**。

## 6. データ漏洩時の対応プロセス(雛形)

### 6.1 検知

- 自動アラート: Supabase監視+Vercel監視
- 通報: support@nolla.example.com への通報
- 第三者通知: セキュリティ研究者からの責任ある開示

### 6.2 初動(発生から24時間以内)

1. インシデント対応チーム招集
2. 影響範囲の特定(影響件数、データ種別)
3. 被害拡大の防止(該当アカウント停止、トークン無効化)
4. 証拠保全

### 6.3 通知(発生から72時間以内)

- 個人情報保護委員会(改正APPI)
- GDPR該当時はEU監督当局
- 影響を受けた本人(親権者)
- 必要に応じて報道発表

### 6.4 事後対応

- 根本原因分析
- 再発防止策の実施
- セキュリティ監査
- 法的責任の確認
