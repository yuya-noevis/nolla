# Nolla ユーザーインタビューガイド v5 — 重心児カスタム版

**ベース**: nolla_interview_guide_v5.md
**作成日**: 2026-04-20
**用途**: 本日実施予定。重症心身障害児（重心児）の保護者1名向けにカスタマイズ

---

## 対象者の属性（事前情報）

| 項目 | 内容 |
|------|------|
| お子さんの年齢 | 4歳 |
| 診断・特性 | 知的障害・ダウン症ほか染色体/遺伝関連・身体障害（重症心身障害児） |
| 日常の負担場面 | 食事・入浴・就寝など生活リズム / 学校・療育・医療の調整（連絡・書類・送迎）/ 相談できる人が少ない・情報が見つからない / 小1の壁・18歳の壁への不安 |
| AI利用 | 仕事での利用 / 障害福祉や子育て関連の法規制確認 / 献立の相談 |

## このインタビューの位置づけ

### 初期メインターゲットではない
重心児はMVPの初期ターゲット（ASD + 知的障害の3-18歳）からは外れる可能性が高い。
ただし **プロダクト設計の参考情報として以下を深く知りたい**：

1. **診断前〜診断直後（出生前後）の情報・支援アクセスの空白**
   → 早期介入プロダクトの価値検証（H4）
2. **相談者不在・情報アクセス困難の実態**
   → AI相談・情報集約機能の需要の手触り
3. **学校・療育・医療の調整負担**
   → 家庭外連携・記録共有の潜在ニーズ
4. **将来不安（小1の壁・18歳の壁）の具体性**
   → 長期伴走型プロダクトの設計示唆
5. **AI利用の実態（仕事・法規制・献立）**
   → AIに求められている役割の具体像

### このガイドで検証しない仮説
- **H2（成長の可視化ニーズ）**: 重心児では意味合いが異なる。質問は最小限
- **H3（子ども直接利用型の需要）**: 身体障害により操作困難の前提。ルートBで進める

### Mom Test の徹底
- **プロダクトの話は最後まで一切しない**。「Nollaを作っています」「AIで〜」とは言わない
- 仮定の未来ではなく、**過去の事実**を聞く
- 感情への共感は最小限。データの精度を優先

## S0. オープニング（2分）

> 「今日はお子さんの日々の生活や、これまでの歩みについてお話を聞かせてください。大変なこと、ご家族のこと、率直なところを教えていただけると助かります。正解や不正解はありませんし、言いにくいことは答えていただかなくて大丈夫です。メモと録音をさせていただいてもよろしいですか？」

**※ この時点でプロダクト名・機能は一切言わない。**

## S0.5. 背景確認（3分）

**目的：重心児特有の事実関係を確認する。深掘りはS2で行う。**

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 0-1 | 「お子さんは今4歳とのことですが、普段どんなところに通っておられますか？」 | 療育・医療ケアの入り口 |
| 0-2 | 「ご家族の構成を教えてください。きょうだいはいらっしゃいますか？」 | きょうだい児の有無・家族の手 |
| 0-3 | 「お子さんのケアで、ご家族の中で主に担われているのはどなたですか？」 | ケア分担の事実 |
| 0-4 | 「医療的ケア（経管栄養・吸引・呼吸器など）はありますか？」 | 医療ケアの必要度。直接聞いてOK |

**※ 運用注意:** 「大変ですね」と言わない。事実だけ記録。

## S1. 日常と生活リズム（8分）

**目的：食事・入浴・就寝の負担を具体エピソードで把握する。**

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 1-1 | 「お子さんの平日は、朝起きてから寝るまで、どんなふうに一日が流れますか？」 | 時間配分の事実。ケアタスクの連続性 |
| 1-2 | 「食事・入浴・就寝の中で、最近一番大変だったのはどれでしたか？具体的にどんなことがありましたか？」 | 具体的エピソード。5W1H |
| 1-3 | 「その時、どう対処しましたか？結果どうでしたか？」 | 対処行動の事実 |
| 1-4 | 「ご自身やご家族が、睡眠を十分に取れていると感じますか？最後にまとまって眠れたのはいつですか？」 | 親の消耗度の客観指標 |
| 1-5 | 「きょうだいがいる場合、その方との時間はどうなっていますか？」 | きょうだい児への影響 |

**→ フォローアップ:**
- 「それはどのくらいの頻度で起きますか？」
- 「他に大変なことはありますか？」（3回まで聞く）

## S2. 診断〜ここまでの歩み（10分） ← 核心

**目的：出生前〜現在の情報・支援アクセスの空白を時系列で明らかにする。**

**※ 設計意図:** ダウン症の場合、出生前または出生直後の診断が多い。ASDとは異なる診断タイミング。早期からの情報アクセス・支援アクセスの経路が核心的に重要。

### Part 1: 診断のタイミングと直後（4分）

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 2-1 | 「お子さんの診断について、最初にわかったのはいつ、どんなタイミングでしたか？」 | 出生前／出生時／NICU／乳児期等の事実 |
| 2-2 | 「その時、どんなお気持ちでしたか？」 | 感情の事実。沈黙を待つ |
| 2-3 | 「その時、どんな情報が伝えられましたか？誰から、どんな形で？」 | 情報伝達の実態。医療側の対応 |
| 2-4 | 「診断直後の数日〜数週間で、具体的に何をしましたか？」 | 行動の時系列 |

### Part 2: 診断直後の情報・支援アクセス（4分）

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 2-5 | 「その時期、『これが知りたかった』『これがあれば良かった』と思うものはありましたか？」 | 未充足ニーズの過去事実 |
| 2-6 | 「誰かに相談しましたか？相談できる人はいましたか？」 | 相談資源の実態 |
| 2-7 | 「同じような状況の親御さんと話す機会はありましたか？どうやって見つけましたか？」 | ピアサポートへのアクセス |

### Part 3: そこから現在まで（2分）

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 2-8 | 「診断直後から今までの4年間で、転機になった出来事や人はありましたか？」 | 支援ルートが開けた瞬間の事実 |
| 2-9 | 「振り返って、『この時こういう支援があれば違った』と思う時期はありますか？」 | 空白の時期の具体化 |

**※ S2運用注意:**
- 感情の質問では5秒以上の沈黙を許す。埋めない
- 「そうですよね」「つらかったですね」は最小限
- exact quote で記録する発言を見逃さない
- プロダクトの話は絶対に出さない

## S3. 学校・療育・医療の調整（7分）

**目的：家庭外連携の実務負担を事実で把握する。**

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 3-1 | 「現在、お子さんに関わっている支援者・機関は具体的にどこですか？（療育・病院・保健センター・地域の相談員など）」 | 関係者マップ |
| 3-2 | 「それぞれの連絡・書類・送迎を、どうやってやりくりしていますか？」 | 調整の実態 |
| 3-3 | 「最近、連絡・書類・送迎で『これはきつい』と感じた場面を具体的に教えてください」 | 具体的エピソード |
| 3-4 | 「機関どうしで情報が共有されていないと感じたことはありますか？最後にそう感じたのはいつ？」 | 情報分断の事実 |
| 3-5 | 「書類やスケジュールをどうやって管理していますか？（紙／アプリ／スプレッドシート等）」 | 現在の管理手段 |

## S4. 情報探索とAI活用（8分） ← この属性特有の強み

**目的：AI利用が事実として確認されている。何に・どう・どこまで使えているかを具体的に引き出す。**

### Part 1: 情報探索全般（3分）

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 4-1 | 「お子さんのことで情報を調べたり相談したりする時、最近何を使いましたか？（ネット検索／SNS／本／人に聞く／AI等）」 | 使っている情報源の実態 |
| 4-2 | 「その中で、一番役に立ったものは何ですか？なぜ？」 | 現在のエース情報源 |
| 4-3 | 「逆に、調べても答えが見つからなかった／ピンとこなかったことは最近ありましたか？」 | 情報アクセス失敗の事実 |

### Part 2: AI利用の実態（5分）

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 4-4 | 「AIを使われているとのことですが、具体的にどんなことで使っていますか？例えば今週使ったものを教えてください」 | 実例。仕事・法規制・献立を深掘り |
| 4-5 | 「障害福祉や子育て関連の法規制を確認するのに使っているとのことですが、具体的にどんな場面でしたか？」 | 法規制確認の具体例 |
| 4-6 | 「その時、AIの答えはどのくらい役に立ちましたか？何か物足りなかった・不安だったことはありましたか？」 | 満足度と未充足 |
| 4-7 | 「献立の相談は、AIのどこが便利で、どこが物足りないですか？」 | 子どものケア関連のAI活用実態 |
| 4-8 | 「障害福祉・子育ての領域で、今AIに聞いても答えがもらえない、もしくは答えが信用できないと感じることはありますか？」 | AI未充足領域の特定 |

**※ この属性はAI利用者なので、この領域で最も具体的なデータが取れる。時間配分を厚めに。**

**目的：将来不安の中身を抽象語（「不安です」）で終わらせず具体化する。**

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 5-1 | 「『小1の壁』について不安があるとのことですが、具体的にどんなことが心配ですか？」 | 不安の中身の具体化 |
| 5-2 | 「その不安に対して、今何か情報を集めたり相談したりしていますか？」 | 行動の有無 |
| 5-3 | 「『18歳の壁』についても同じく、具体的にどんなことが気になりますか？」 | 長期不安の具体化 |

**※ 5-4は仮定の未来質問だが、自由回答で選択肢を示さない形なら許容。プロダクトを示唆しない。**

## 絶対に聞いてはいけない質問・言ってはいけないこと

| ❌ 禁止 | 代替 |
|---------|------|
| 「このアプリ使いたいですか？」 | 「実際に試したアプリはありますか？どうでしたか？」 |
| 「月額〜円なら払いますか？」 | 「今、どのくらい使っていますか？」 |
| 「こういうものがあったら便利だと思いますか？」 | 「最近〇〇をしたのはいつですか？どんな状況でしたか？」 |
| 「Nollaというアプリを作っています／AIで〜」 | **ソリューションの話は出さない** |
| 障害名・程度について断定的な言い方 | 相手の使う言葉をそのまま使う |
| 「大変でしたね」の連発 | 事実を短く返す／沈黙を使う |

## インタビュー中の行動規範（重心児カスタム）

| ルール | 理由 |
|--------|------|
| 話す量は全体の20%以下 | 80%は相手に話させる |
| 感情の質問では5秒以上の沈黙を許す | 重いテーマ。先回りしない |
| 褒め言葉・共感連発は避ける | 社交辞令はデータではない |
| 「他に何かありますか？」を3回聞く | 1回目は社交辞令。3回目に本音 |
| プロダクト説明は最後まで一切しない | バイアス混入防止 |
| 過去の事実で質問する | 「その時どうしましたか？」「最後に〇〇したのはいつ？」 |
| **医療ケア・きょうだい児・家族関係はデリケート。踏み込みすぎない** | 拒否サインが出たら次へ |
| **涙が出たら水を勧め、少し待つ** | 急がない |

## このインタビューから特に拾いたいシグナル

1. **診断タイミング×情報伝達の質**: ダウン症は出生前後診断が多い。「どう伝えられたか」のクオリティは医療従事者依存で、ここに大きな空白があれば早期介入プロダクトの強い示唆になる
2. **相談資源の砂漠度合い**: 「相談できる人が少ない」が事前情報。誰に・何を・どう相談したかの事実で、未充足の密度を測る
3. **AI利用の具体像**: すでにAIユーザー。何に使って、何に物足りないかが、プロダクト設計の直接的な示唆になる
4. **小1の壁・18歳の壁の中身**: 抽象語で終わらせず、具体化する。ここが長期伴走型プロダクトの価値提案に直結
5. **調整負担の内訳**: どの作業がどれだけ時間を食っているか。デジタル化/集約できる余地の在処

## 本日のインタビュー後の振り返り観点

- H4空白スコア（2以上か）
- 情報アクセス失敗の具体事例が3件以上出たか
- AI未充足領域が特定できたか
- 5-4（「あったら違うもの」）で示唆された形はどんなプロダクト類型か
- MVP協力者として連絡先提供があったか
- インタビュアーのソリューション説明発生有無
