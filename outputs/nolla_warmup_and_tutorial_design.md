# ウォームアップ + チュートリアル + 感覚フィードバック設計書

## 1. 目的と背景

Phase 1-J までで Galaxy 惑星テーマの視覚統合は完了した。しかし実機テストを前提とした
ユーザー体験としては以下の課題が残っている。

- **ログイン直後の10回タップ（motor baseline）が単調**でホーム画面上に埋め込まれており
  集中できない。科学計測（DDM の RT ベースライン）であることが体験上も伝わらない。
- **初見の子どもがゲームルールを理解する導入が存在しない**。4つのゲームはいきなり開始
  してしまい、ASD＋知的障害児にとっては「何をすればいいか分からない」状態になる。
- **効果音・バイブレーションが未実装**。タップ時の即時フィードバック（200ms 以内の
  視覚+音声+振動）という MVP 必須要件が満たせていない。

本設計書は上記3点を統合的に解決するための仕様を定義する。実装は Phase 1-K として
ロードマップに追加する。

## 2. 設計原則（本ドキュメントに限定）

1. **科学的妥当性を崩さない** — Motor baseline の RT 計測仕様（ランダム遅延 800〜
   2500ms、単一刺激、正確な RT 捕捉）は変更しない。演出は純粋な見た目と触覚のみ。
2. **文字を一切使わない** — チュートリアル・イントロは全てアニメーション駆動。
3. **スキップ＝スタート** — チュートリアル中央下部に大きな「Startボタン（光る星）」を
   常時表示する。このボタンをタップするとチュートリアルが消えて本番が開始する。
   「スキップ」と「スタート」は同じ行為。ボタンは1つだけ。
4. **共通部品化** — ウォームアップ・4ゲーム・将来の追加ミニゲーム全てで同じ
   `IntroOverlay` を使い回す。実装の重複を避ける。
5. **BGM は入れない** — ウォームアップは集中計測のため無音。ゲーム本編の BGM 検討は
   別フェーズで行う。効果音とバイブのみを本設計書のスコープとする。

## 3. 共通 IntroOverlay コンポーネント

### 3.1 役割

5つの画面（`/warmup` + 4ゲーム）で共通して使う「文字なしチュートリアル層」。

### 3.2 API

```ts
type IntroOverlayProps = {
  demoAnimation: "warmup" | "memory-match" | "sorting" | "visual-search" | "corsi";
  onDismiss: () => void;  // タップされたら呼ばれる
};
```

### 3.3 振る舞い

1. マウント時に半透明の暗幕（`rgba(0,0,0,0.55)`）をフェードイン（300ms）
2. 中央上部〜中央に **デモアニメーション** をループ再生（各バリアント固有の内容。
   下記 4/5 節参照）
3. 画面下部中央に **Startボタン**（大きな光る星、直径 120px、ゆっくり脈動）を常時表示
4. Startボタンのみが有効タップターゲット。タップすると `onDismiss()` が呼ばれる
5. 消える時は暗幕と全要素を 200ms でフェードアウト
6. **毎セッション必ず表示** する（localStorage で抑制しない）。スキップ＝スタートなので
   ストレスはゼロ

**重要**: Startボタンタップは開始合図であり、その後に始まる本番プレイのタップには
含めない。ウォームアップなら Start → ready → 刺激1 → タップ1 → ... → 刺激10 → タップ10
の順で全10タップが RT 計測対象になる。

### 3.4 レイアウト

- z-index: 50（通常のゲームUIが 0〜40 を使用）
- 暗幕は pointer-events: auto、内部のアニメ要素は pointer-events: none
- タップターゲットはオーバーレイ全面（`absolute inset-0`）

### 3.5 アクセシビリティ

- `role="button"` / `aria-label="タップしてはじめる"`
- キーボード Enter / Space でも発火（PC 開発時用）

## 4. ウォームアップ新仕様

### 4.1 配置変更

- **旧**: `src/app/home/carousel.tsx` 内に `MotorBaselineMeasurement` を直接埋め込み
- **新**: 専用ルート `src/app/warmup/page.tsx` を新設。ホームに入る前に必ず経由

### 4.2 フロー

```
ログイン完了
   ↓
/home にアクセス
   ↓
needsMotorBaseline() チェック（localStorage）
   ↓ true
router.push("/warmup")
   ↓
/warmup でウォームアップ実施
   ↓
完了 → saveMotorBaselineFromHome() → router.replace("/home")
```

既存の `needsMotorBaseline()` と `saveMotorBaselineFromHome()` はそのまま流用。
ただし宣言場所を `lib/session/motor-baseline-client.ts`（新規）に切り出して両画面で
import できるようにする。

### 4.3 画面構成（/warmup）

```
┌──────────────────────────────────────┐
│    [星座プレート: 10個の薄い位置マーカー]     │  ← 上部
│                                        │
│                                        │
│              [刺激が出る領域]              │  ← 中央
│         (ready: 何も見えない)             │
│         (target: 光る星がワープ出現)        │
│                                        │
│                                        │
│      [漆黒 + 流れ星パーティクル]            │  ← 背景
└──────────────────────────────────────┘
```

- 背景: `#000000` + 小さな流れ星パーティクル（CSS animation、10〜15個）
- カルーセル・惑星・スター残高・ヘッダー・タブバー全て非表示
- 星座プレートは画面上部中央、横並びの 10 スロット（薄い破線円 32px）

### 4.4 刺激仕様

- 1種類のみ: **光る星**（SVG、直径 128px、グラデーション: 中心 `#FFF8D6` → 外周 `#FFCE5C`）
- 出現演出: 中心から半径0→128px にスケール＋透明度0→1（300ms）
- 出現位置: 画面中央を中心とした半径 180px の円内からランダム選出（刺激が毎回同じ場所だと
  予測可能になってRTが短縮するため少しだけ散らす）
- ready 状態中は何も見えない（暗黒）

### 4.5 RT 計測仕様（変更なし）

- ランダム遅延: 800〜2500ms（既存通り）
- 刺激表示完了時点を計測開始とする（既存通り）
- タップ時刻との差分が RT
- 10試行分の中央値を `calculateMotorBaseline()` に渡す（既存通り）

### 4.6 タップ成功時の演出

1. タップ位置に白いフラッシュ円（半径 8→64px、200ms、透明度1→0）
2. 星が現在位置から **星座プレートの次の空きスロット** まで二次ベジェ曲線で飛翔（500ms）
3. 飛翔中にパーティクル軌跡（小さな白点 5〜8個）
4. スロット到達時に軽いスケールバウンス（1.0→1.3→1.0、200ms）
5. 効果音「キンッ」再生（SFX-01、下記 6節）
6. バイブ 30ms（下記 7節）
7. 次の trial の ready 状態へ遷移

### 4.7 10個完成時の演出（新規）

1. 10個目の星がスロットに入った瞬間、**全スロットが同時に強く発光**（500ms）
2. 隣接する星の間に **白い線** が順番に描画される（stroke-dashoffset アニメ、総計 800ms）
3. 全線が描画完了した瞬間に **星座絵** が完成（線＋星の全体がひとつの絵として認識される）
4. 効果音「キラーン」再生（SFX-02）
5. バイブパターン再生（50-100-50ms）
6. **3秒間** 星座を表示し、その後 `router.replace("/home")` でホームへ遷移
7. 遷移時は通常のワープアニメ（既存の carousel.tsx で定義済み）

### 4.8 チュートリアル（毎回表示）

- `IntroOverlay` を `/warmup` マウント時に常に表示
- デモアニメ: 中央に星がふわっと出現 → ハンドアイコンがスライドしてタップ → 星が上部
  スロットに吸い込まれる → 2秒空けて再ループ
- 画面下部中央の **Startボタン（光る星）** をタップすると `onDismiss()` が発火、
  オーバーレイが消えて本番の ready 状態（ランダム遅延カウント）が開始
- Startボタンのタップ自体は計測対象外。その後に出現する刺激1〜10への10タップが
  RT 計測対象になる

## 5. 4ゲームのイントロチュートリアル

### 5.1 共通仕様

- 各ゲームの `game-session.tsx` マウント直後に `IntroOverlay` を表示
- 毎プレイ必ず表示（localStorage 抑制なし）
- 子どもが画面をタップ → オーバーレイ消失 → 通常のゲーム開始

### 5.2 ゲーム別デモ内容

#### 5.2.1 神経衰弱（memory-match）

- デモ: 裏向きカード2枚 → 片方がめくれる → もう片方がめくれる → 絵柄が一致 → 両方光る
- ループ: 2.5秒 + 1秒休止

#### 5.2.2 分類ソーティング（sorting）

- デモ: アイテム1個が上部に出現 → ハンドが3つの箱のうち正しい箱にタップ → アイテムが
  その箱に飛んでいく → 箱が光る
- ループ: 2.5秒 + 1秒休止

#### 5.2.3 視覚探索（visual-search）

- デモ: 複数のアイテムが並んだ盤面 → 1つだけ違う色のアイテムが拡大強調 → ハンドが
  それをタップ → タップしたアイテムが光る
- ループ: 3秒 + 1秒休止

#### 5.2.4 Corsi Block

- デモ: 3つのブロックが順番に光る（1→2→3）→ ハンドが同じ順にタップ → 全ブロックが
  成功光
- ループ: 3.5秒 + 1秒休止

### 5.3 既存 M3 チュートリアルとの関係

既存の `tutorial-overlay.tsx`（4段階プロンプティング削減法）は **ゲーム中の段階的補助**
として別機能。本設計書の `IntroOverlay` はその前段（ルール説明）を担う。

フロー:
```
ゲーム開始
  ↓
IntroOverlay （新規・文字なしデモ、タップで消える）
  ↓
通常のゲームプレイ + M3 tutorial-overlay（ゲーム中の補助、既存機能）
```

## 6. SFX（効果音）仕様

### 6.1 必要な効果音リスト

| ID | 用途 | 秒数目安 | 音の性質 |
|---|---|---|---|
| SFX-01 | ウォームアップ・タップ成功 | 0.2-0.3s | 「キンッ」高音の澄んだ鈴音 |
| SFX-02 | ウォームアップ・星座完成 | 1.0-1.5s | 「キラーン」上昇系のきらめき音 |
| SFX-03 | ゲーム・正解 | 0.3-0.4s | 「ピコッ」丸く柔らかい肯定音 |
| SFX-04 | ゲーム・星獲得 | 0.4-0.6s | 「チャリン」コインが触れ合う音 |
| SFX-05 | UIタップ（ボタン全般） | 0.1-0.2s | 「ポッ」ごく控えめなタップ音 |
| SFX-06 | イントロのハンドタップ演出 | 0.1s | SFX-05 と同じ |

### 6.2 素材調達

- **Freesound.org**（CC0 / CC-BY）から選定
- **Zapsplat**（無料会員・クレジット表記必要）を次点
- 候補音を `app/public/sfx/` に配置
- ファイル命名: `sfx-01-warmup-tap.mp3` 等
- クレジット一覧を `docs/credits.md`（新規）に記録

### 6.3 再生基盤

`src/lib/feedback/sfx.ts`（新規）:

```ts
export type SfxId = "warmup-tap" | "warmup-complete" | "game-correct"
                  | "star-gain" | "ui-tap" | "hand-demo";

export function playSfx(id: SfxId): void;
export function setSfxEnabled(enabled: boolean): void;
export function isSfxEnabled(): boolean;
```

- 実装は HTMLAudioElement ベース。同一効果音の連続再生に対応するため pool 方式
- 設定は localStorage `nolla_sfx_enabled` で ON/OFF 切替（保護者画面 P5 で切替可能に）
- テスト: Mock Audio を使用した単体テストで再生回数を検証

## 8. 星座プリセット（10種）

### 8.1 データフォーマット

```ts
type ConstellationPreset = {
  id: string;
  name: string;  // 内部管理用、UIには表示しない
  slots: ReadonlyArray<{ x: number; y: number }>;  // 10個固定、正規化座標 0-100
  lines: ReadonlyArray<[number, number]>;  // slotsのインデックスペア
};
```

座標系: 横幅 100 × 高さ 60 の正規化空間。実画面では viewport 幅 60〜80% に収まるよう
スケール。`y=0` が画面上部、`y=60` が下端。

### 8.2 プリセット初期12種

デザイン担当は Claude 側。抽象化された「輪郭が伝わる」絵柄を12種選定した。
男の子が好みやすい乗り物（車・飛行機）とロケットを含み、動物・自然・道具・幾何を
バランス良く配置している。

1. **cat** （ネコの顔）
2. **rocket** （ロケット）
3. **fish** （魚）
4. **flower** （花）
5. **sword** （剣）
6. **heart** （ハート）
7. **crown** （王冠）
8. **star** （五芒星）
9. **butterfly** （蝶）
10. **house** （家）
11. **car** （車）
12. **airplane** （飛行機）

座標と線の具体的な JSON は `app/src/lib/warmup/constellations.ts` に実装する。
初期値は本設計書 9節の JSON を参照のこと。実装時に実画面で見栄えを確認し微調整する。

### 8.3 選出ロジック

- セッションごとにランダム選出
- 直近3回に出した星座は除外（localStorage に履歴保持）
- 実装: `src/lib/warmup/constellation-picker.ts`（純粋関数、テスト必須）
