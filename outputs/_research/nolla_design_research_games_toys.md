## 各コンテンツ詳細分析

### ランクS（世界的デファクト＆最適化最高）

#### 1. Minecraft

**ASD児評価の根拠**
- 親コミュニティ・専門家の一致した評価：「ASD児の最高傑作」[^1][^2][^3]
- 研究機関の推奨：Games Learning Society, LearningWorks for Kids が「7つの理由」で推奨[^4]

**なぜASD児にハマるのか — メカニズム**

1. **予測可能性（CRITICAL）**
   - ブロックの振る舞いが100%一貫。ルール変更なし
   - クラフトレシピが固定。試行錯誤の不確実性がない
   - 「ゲームの裏切り」がゼロ → ASD児の不安軽減

2. **カスタマイズ性の無限性**
   - 音声ON/OFF、明度調整、視覚効果削除可能
   - 各児の感覚過敏に合わせた遊び方が可能
   - 強制フロー・時間制限なし

3. **視覚的シンプルさ＋ピクセルアート美学**
   - ブロック単位の構造。不気味の谷ゼロ
   - キャラクターが幾何学的デフォルメ
   - 色彩が「自然+人工」のバランス

4. **世界構築の自由度**
   - 「正解」がない。全プレイスタイルが肯定される
   - 建築・採掘・農業・探索と多様な目標設定
   - サバイバル/クリエイティブ/ピースフル選択可

5. **社会性の柔軟な入口**
   - マルチプレイでも「高速社会交渉」不要
   - ルールベース・構造化された相互作用
   - 感覚過敏者と通常児が同じ世界で遊べる

**ビジュアル言語**
- **色調**：土色（#8B7355）、青（#4A90E2）、赤レンガ（#C85A54）、木色（#8B6239）のミュートトーン
- **形状**：立方体 + 直方体。円形・有機形状なし
- **質感**：ブロック感 + テクスチャーの細かさ（サンプリング感）
- **キャラ**：Steve（ピクセル人間）、モブ（牛・豚・羊）。顔は4px級のドット
- **世界観**：「採掘→構築→生存」のサイクル。時間経過（昼夜）の可視化

**UX/ゲームデザイン**
- **進行構造**：完全自由度。クエスト・目標なし
- **フィードバック**：ブロック破壊時の粒子効果。サウンド可視化（OFFにも対応）
- **報酬**：素材獲得（即物的）+ 建築の達成感
- **難度調整**：ゲームモード選択（ピースフル→サバイバル）。敵出現ONOFFで調整

**Nollaへの適用可能性**
- ブロック調への固執を解く許可 ✅
- 予測可能性＆カスタマイズ性は必須要素 ✅
- 世界観は「農業＋建築」より「遊園地＋自宅」へ ～

#### 2. Roblox (Adopt Me! / Brookhaven RP)

**ASD児評価の根拠**
- 企業研究：Autism Society of America × Internet Matters による Neurodiversity in Gaming Survey[^5]
- ユースケース：「自分の世界を持つ」「社会的相互作用の低圧化」[^6]

**なぜASD児にハマるのか — メカニズム**

1. **世界観の完全な所有**
   - 自宅・部屋・庭をカスタマイズして「自分の場所」を作れる
   - Minecraft同様、ルール不変 + 建築自由度
   - 採用Me/Brookhavenは「ロールプレイ」の枠組みを提供（目標が曖昧でない）

2. **社会的相互作用の柔軟性**
   - マルチプレイ必須だが、同じ場所に共存するだけでOK
   - 高速なチャットやトレード圧力はない（ローカル設定可）
   - ASD児同士で自動マッチング（オタク文化が受け入れられる）

3. **報酬ループの視覚的明晰さ**
   - ペット購入 → 育成 → インベントリに表示（コレクション達成感）
   - 家具レイアウト → 即座に3D反映
   - 経済ループ（ゲーム内通貨獲得→購入）が単純

4. **予測可能な日常シミュレーション**
   - 現実社会の「家＋ペット＋友人」を再現
   - 失敗がないRP環境。「間違い」概念なし
   - 何度でもリセット可能（心理的安全）

**ビジュアル言語**
- **色調**：パステルトーン。薄いピンク（#FFB6D9）、薄い青（#ADD8E6）、薄い黄（#FFFFE0）
- **形状**：Minecraftより丸み。家具が有機形状（椅子・ベッド）
- **質感**：ポリゴン低解像度だが「キュート」。アニメ寄り
- **キャラ**：人型アバター（顔が描かれていない、デフォルメ）+ かわいいペット
- **世界観**：「日常生活シミュレーション」。ファンタジー要素は弱い

**UX/ゲームデザイン**
- **進行構造**：完全に開放。Adopt Meは「ペット育成」が主軸だが強制ではない
- **フィードバック**：UI操作時の即座な画面反映。社会的報酬（友人との相互作用視認）
- **報酬**：ペット・家具・通貨
- **難度調整**：なし。同一レベルプレイ

**懸念点**
- オンライン社会的危害リスク（Internet Matters が指摘）[^5]
- 他プレイヤーとのトラブル可能性
- チャット依存の可能性

#### 3. どうぶつの森

**ASD児評価の根拠**
- 親・研究の強い推奨：「ASD向けゲーム筆頭」という位置付け[^1][^7]
- Switch版普及で、子ども向けデバイスの標準的選択肢

**なぜASD児にハマるのか — メカニズム**

1. **完全な非競争性**
   - ゲームオーバーなし。失敗概念なし
   - 島の発展に「正解」なし。全プレイスタイル肯定
   - NPCが常に優しい

2. **無限のルーチン性**
   - 毎日の日課（化石掘り、魚釣り、草刈り、会話）
   - 四季の変化は予測可能（カレンダー）
   - イベント日程が固定・事前告知

3. **パステル美学 + カラフルさ**
   - 単色ではなく、「くすんだカラフルさ」
   - 人物キャラは目玉焼きのような目（愛らしい）
   - 背景が「自然」「マイホーム」のハイブリッド

4. **島全体が「自分の世界」**
   - 家のレイアウト、地形、橋、建物すべてカスタマイズ
   - Minecraftより「完成度」が高い（素人でも見栄えが良い）
   - 自分の島以外に「他島訪問」で社会的確認

**ビジュアル言語**
- **色調**：パステル全色使い。黄色（#FFDF5D）、ピンク（#FFB6D9）、緑（#A8D8A8）、水色（#87CEEB）
- **形状**：有機形状。丸みの美学。直線なし
- **質感**：セル画風。毛並みディテール
- **キャラ**：かわいい動物。人間キャラは目が大きく、表現豊か（喜怒哀楽）
- **世界観**：「自然+人間生活」の融和。ファンタジーだが地に足がついている

**UX/ゲームデザイン**
- **進行構造**：完全に自由。ストーリーはない（たぬきちの融資は自動返済可能）
- **フィードバック**：操作時の音声 + 粒子効果。非常に優しい
- **報酬**：家具・服・素材・ベル（通貨）
- **難度調整**：なし

### ランクA（高い配慮＋実証済み効果）

#### 4. LEGO（物理玩具）

**ASD児評価の根拠**
- 治療的効果が実証済み：Dr. Daniel LeGoff が 2003年に「LEGO Therapy」を開発[^8]
- LEGO Foundation による sensory inclusion 公式プログラム[^9]

**なぜASD児にハマるのか — メカニズム**

1. **予測可能な組立構造**
   - 全ブリック（レンガ）が同じシステムで接続
   - 説明書が視覚的でステップバイステップ
   - 「間違い」が物理的にわかる（合わない）

2. **マルチモーダル感覚フィードバック**
   - 視覚：色彩 + 形状組み合わせ
   - 触覚：ブリック接合時のクリック感 + 圧力感
   - 聴覚：接合音

3. **実在性のある完成感**
   - 仮想ではなく、実物が手に残る
   - 親/兄弟と共有可能
   - 飾る・遊ぶの両用

4. **社会的協力の自然な入口（LEGO Therapy）**
   - 3人グループで Builder/Designer/Supplier ロール分担
   - 完全にはマニュアル化されていない柔軟性
   - 社会的スキル習得の「足掛かり」

**ビジュアル言語**
- **色調**：基本色。赤（#E40C02）、黄（#FFEC00）、青（#0052CC）、白、黒、肌色
- **形状**：立方体ベース。角ばった品質
- **質感**：プラスチック + 意図的な「おもちゃ感」
- **パッケージ**：色分けされた説明書。年齢別セット

**適用への配慮**
- 物理的なため、デジタルゲームではない
- 但し「ビジュアル言語」として参考にする価値高

---

#### 5. Pokemon（ゲーム＆カード）

**ASD児評価の根拠**
- Creator Satoshi Tajiri 自身が ASD[^10]
- 親・研究による強い推奨：「特殊興味の最大事例」[^11][^12]

**なぜASD児にハマるのか — メカニズム**

1. **無限の分類・コレクション構造**
   - 図鑑システム：全ポケモン分類 + 捕獲済み/未捕獲の可視化
   - タイプ・能力値・進化段階という構造化された情報
   - 「全て揃える」という目標が明確かつ現実的

2. **予測可能なゲームループ**
   - ジム・ストーリーが固定
   - ポケモンの技が固定化（予測可能）
   - 敵 AI が明確で単純

3. **特殊興味の最優適合**
   - ASD児の「hyperfocus」が Pokemon を対象に
   - 知識深掘りが長期的に可能（世代交代で新ポケモン追加）
   - コミュニティ内での知識共有が価値（社会的接点）

4. **カード化による物質化**
   - ゲーム版より、カード版が ASD児に強い
   - 視覚的コレクション + 物理的な希少性（レア度）
   - 売買・交換の社会経験

**ビジュアル言語**
- **色調**：彩度高め。黄（Pikachu #FFD700）、赤（Fire タイプ #FF0000）、青（Water タイプ #0000FF）、紫（Psychic）
- **形状**：動物をモチーフ。但し「デフォルメ」は優しい
- **質感**：アニメーション風。2D ドット絵 → 3D ポリゴンへの進化
- **キャラ**：ポケモン図鑑の多様性 + パートナー感（Pikachu）

**Nolla との距離**
- Pokemon は「特殊興味」枠。Nolla ゲームそのものにはなりにくい
- 但し「コレクション心理」と「図鑑システム」は参考価値高

### ランクB（有効だが限定的）

#### 6. Stardew Valley

**理由**：ルーチン＆非競争性は ASD 向きだが、8歳以上向け + 大人が主ユースケース

**なぜASD児にハマるのか**
- 毎日の農業ルーチンが固定・予測可能[^13]
- ゲームオーバーなし。失敗の概念なし
- 美学が「pixel art + 郷愁」。単調ではなくカラフル

**懸念**
- テキスト量が多い（8歳以下だと文字読み困難）
- NPCとのロマンス要素が若年層には不要
- UI が複雑（ルーチン優先で、意思決定が多い）

#### 7-9. Toca Boca / Sago Mini / Khan Academy Kids

**高評価の根拠**
- Toca Boca：「開放的クリエイティブプレイ」実証済み[^14]
- Sago Mini：「若年向けに最適化」
- Khan Academy Kids：「無料 + 教育的」

**懸念**
- 3〜5歳向け設計が多く、Nolla の想定 3〜18歳範囲では「幼稚」に見える可能性
- 「療育アプリ」としてのスティグマが強い（親が「これは勉強」と認識）
- ゲーム性（ハマり感）が弱い

#### 10-11. トミカ / プラレール

**ASD児への有効性**
- コレクション心 + 予測可能な動き（電車走行）[^16]
- 感覚フィードバック（走行音、振動）

**Nolla との距離**
- 物理玩具のため、デジタル化には限界
- 但し「コレクション心」「予測可能性」は参考

<a name="共通パターン分析"></a>

## 共通パターン分析：ASD児に共通する「ハマるメカニズム」

### 1. 予測可能性（CRITICAL — 全S/A評価コンテンツの共通要素）

**定義**：ルール・振る舞い・結果が100%事前予測可能

| コンテンツ | 予測可能性の仕組み |
|-----------|------------------|
| Minecraft | ブロック振る舞い + 非変更ルール |
| Roblox | ゲーム内経済ルール + 操作システム固定 |
| どうぶつの森 | イベント日程 + NPCルーチン |
| LEGO | ブリックシステム + 説明書 |

**実装への示唆**
- ゲーム内ルール変更は予告必須
- 難度上昇も「予測可能」（9段階など明確化）
- チュートリアルで全ルール明示

---

### 2. 完全な非競争性

**定義**：敵がいない / 失敗概念がない / ゲームオーバーがない

| コンテンツ | 非競争性の形態 |
|-----------|----------------|
| どうぶつの森 | ゲームオーバーなし |
| Minecraft | ピースフルモード選択可 |
| Stardew Valley | 失敗ペナルティなし |

**Nolla との整合性**
- 難度は「成功率 75～80%」で自動維持（設計書確定）
- 「敵」概念は不要
- 正解/不正解フィードバックはサイレント修正で

---

### 3. 自由度＋目標の両立

**定義**：「正解」がないが、目標は自分で設定可能

| コンテンツ | 自由度 | 目標 |
|-----------|--------|------|
| Minecraft | 完全自由 | 建築・採掘など自選 |
| どうぶつの森 | 完全自由 | 島完成・図鑑埋め等自選 |
| Roblox | 部屋カスタマイズは自由 | ペット育成（提案型） |

**示唆**
- Nolla ゲームループは「クリア不可能」にする？
- 又は「クリア条件が複数」（正解複数）

---

### 4. 即座フィードバック＋ビジュアル明晰さ

**定義**：200ms以内にビジュアル反応。色＆形状で情報伝達

| コンテンツ | フィードバック例 |
|-----------|------------------|
| Minecraft | ブロック破壊 → 粒子効果 |
| どうぶつの森 | 操作 → 音声 + アニメ |
| LEGO | ブリック接合 → クリック感 |

**Nolla設計との一致**
- 200ms 即座フィードバックは既設計に一致
- 音声 + 粒子 + 振動の組み合わせ

---

### 5. コレクション心理（長期エンゲージメント）

**定義**：「全て揃える」という目標が長期的に動機づける

| コンテンツ | コレクション対象 |
|-----------|-----------------|
| Pokemon | 図鑑 + カード |
| Minecraft | 全ブロック種類 + 建築パターン |
| どうぶつの森 | 家具 + 服 + 魚・虫 |
| トミカ | 全車種 |

**Nolla 実装**
- ゲーム種類の拡充予定（6→9+ST+動画）
- 報酬ショップの「図鑑」化
- スター＆バッジのコレクション設計

---

### 6. ビジュアル言語の共通特性

#### 6.1 色彩

**データに基づいた選択（研究より）**[^16]
- **推奨色調**：冷色系（青・緑・紫）
- **推奨彩度**：高彩度（但し蛍光色は禁止）
- **推奨明度**：高明度（明るい）
- **非推奨**：黄色（ASD児が苦手傾向）

**実際の成功例**
- どうぶつの森：彩度高・明度高のパステル（黄色も含むが、くすんだ黄）
- Minecraft：ミュートトーン（彩度制御）
- Roblox：パステル（彩度抑制）

**Nolla との関連**
- `nolla_color_regulation.md` の「自由度」に基づいて問題なし
- 但し黄色の使い方（くすませる）が重要

#### 6.2 形状言語

**ASD児の視覚処理特性**
- 直線 + 立方体：予測可能感
- 曲線 + 有機形状：親和性・優しさ
- ドット/ブロック：「デジタル感」が心地よい

**実装パターン**
- Minecraft：立方体（構造性）
- どうぶつの森：有機形状（優しさ）
- Roblox：混在（立方体 + 丸さ）

**Nolla 設計**
- 「Minecraft + ACパステル融合」は両立可能
- 建物は立方体系、キャラは丸め、背景は有機形状

#### 6.3 キャラクター造形

**ASD児が好む特性**
- 目玉焼きのような目（不気味の谷回避）
- デフォルメ（リアルな人間顔は避ける）
- 感情表現は3段階（笑う・中立・困る）以内

**実例**
- どうぶつの森：動物キャラ + 目玉焼き目
- Minecraft：Steve（ドット人間 + 黒目）
- Roblox：アバター（顔なし + 体の形状で表現）

**Nolla キャラ設計との整合性**
- 「幾何学的デフォルメ」既確定（CLAUDE.md）
- わが家キャラの造形は既に最適化

#### 6.4 世界観の「物語性」

**ASD児の特性**
- 「複雑な背景ストーリー」は不要
- 「何ができるか」が明白な環境が好ま

しい
- 世界観は「自然+人工」のハイブリッド

**実装パターン**
- Minecraft：採掘→構築→生存 = シンプル
- どうぶつの森：島生活 = わかりやすい日常
- Pokemon：冒険 + 図鑑埋め = 目標が明示的

**Nolla 適用**
- 「遊園地」or「テーマパーク」の世界観が候補（目的地が明白）
- 「自宅+学校」より「体験の場」が適切か

## Nolla MVP 実装への示唆

### 1. 「Minecraftモデル」と「どうぶつの森モデル」の選択

| 属性 | Minecraft | どうぶつの森 |
|------|-----------|-------------|
| 自由度 | 100%（正解なし） | 80%（提案あり） |
| 難度調整 | ゲームモード | 自動適応 |
| ビジュアル | ブロック調 | パステル曲線 |
| 社会性 | マルチプレイ必須でない | シングルプレイOK |
| 年齢範囲 | 3～18歳全域 | 4～18歳 |

**Nolla の位置付け**
- Minecraft の「自由度＋予測可能性」 ✅
- どうぶつの森の「パステル美学＋日常性」 ✅
- 両者のハイブリッドが最適

**実装提案**
- ゲーム内容：視空間認知訓練（既設計）
- 世界観：遊園地 or 学習施設（オープンワールド的）
- ビジュアル：Minecraft + ACパステルの融合（既確定）

---

### 2. コレクション＆報酬ショップ設計への応用

**実装すべき要素**
- スター・バッジ・ロック解除のドット進行表示
- 「あと5個でバッジ3が開放」などの可視化
- 図鑑機能（各ゲーム / スター獲得状況）

**参考パターン**
- Pokemon：図鑑 + 捕獲率表示
- どうぶつの森：クリエイトレシピ図鑑
- Minecraft：実績システム（ただし提案型に）

---

### 3. 難度調整の「透明性」

**実装方針**
- 現在の難度レベルを明示（「LEVEL 3 of 9」など）
- 成功率を親画面で可視化（専用ダッシュボード）
- 難度上昇を事前告知（「次のゲームは難度UP」などのアニメ）

**参考**
- Minecraft：ゲームモード選択画面の明晰さ
- Khan Academy Kids：進捗バー + 次の学習トピック予告

---

### 4. 「ゲーム性」と「療育効果」の両立

**Nolla の重要な決定（CLAUDE.md より）**
- 「療育アプリ」ではなく「ASD児がハマるゲーム」
- 療育効果は「裏側」で実現

**実装への応用**
- ゲーム画面に「学習目的」を表示しない
- 難度 = 認知負荷 = RCT検証済みゲーム仕様（非表示）
- UI は「楽しさ」優先

**参考**
- どうぶつの森：教育的なのに「遊び」に見える
- Minecraft：建築は創造的学習だが「ゲーム」に見える

**必須要件**
- スマホ横向き（812×375）対応
- iPad 横向き（1194×834）対応

**参考**

**長期ロードマップ（CLAUDE.md ＆ 既設計より）**

**実装への示唆**

- 報酬ショップは「ゲーム追加で自動拡張」なデータ構造

**参考**
- Pokemon：世代更新で新ポケモン追加 → 図鑑拡張
- Roblox：新ゲーム続々追加 → プラットフォーム成立

## 結論

Nolla MVP のビジュアル方向性は、以下の「3層構造」を推奨：

1. **ゲーム設計（内部ロジック）**：Minecraft の「完全予測可能性」 + 「自由度」

この組み合わせにより、ASD 児が「ハマるゲーム」になりながら、療育効果（視空間認知訓練）を「裏側」で実現できる。
