# Nolla ビジュアル方向性リサーチ：教育アプリ + キャラクターIP分析

## 概要

このドキュメントは、ASD・軽度〜中度知的障害児（3-18歳）向けの発達支援テクノロジーである Nolla のビジュアル方向性を決定するための市場リサーチ。

**主な検証項目**:
1. 教育アプリとキャラクターIPがASD児に愛される機械的理由
2. 視覚言語（色・形・キャラクターデザイン）の共通パターン
3. UXパターンと感覚親和性
4. Nollaが採用可能な具体的な設計パターン

**方法論**: Web検索 + 学術文献・コミュニティ評価の確認 + 既存Nolla設計ドキュメント（Minecraft/Animal Crossing ベンチマーク）との照合

### 1B. S ティア：臨床標準 / グローバル基準

#### **Otsimo**
**分類**: AAC + 発達支援アプリ | **対象**: 重度知的障害～グレーゾーン（3-12歳）

**ASD児への評価**:
- グローバルASD親コミュニティで最高評価（Reddit r/autism, 特定支援機関）
- 音声認識 + 画像ベース AAC（言葉が出ない子向け）
- 臨床心理士・言語療法士による設計
- 参考: https://otsimo.com/en/

**ASD親和性の理由**:
1. **予測可能性**: ゲーム画面の構造が毎回同じ、ルール不変
2. **感覚管理**: 原色なし、点滅なし、突然の大きな音なし
3. **自分のペースで進む**: 時間制限なし、強制進行なし
4. **視覚ベース操作**: 文字読み不要（音声AAC）
5. **即座フィードバック**: タップ→結果、200ms以内

**ビジュアル言語**:
- パレット: 低彩度パステル（淡いブルー、淡いグリーン、ベージュ）
- キャラクター: 簡略化された人物 + 動物（3-5パターン感情）
- UI: ボタンは大きく（64×64px以上）、シンプルな四角形 + 文字なしアイコン
- 背景: 単色またはシンプルなテクスチャ（世界観最小限）

**Nolla適用ポイント**:
- 時間制限なし、提案型で確認フロー → 実装済み（ロードマップ確認）
- 音声AAC選択肢 → マイルーム内セッション提案で応用可能
- アイコン+アニメーション主導（文字なし）→ 既存ルール合致

**URL引用**:
- Otsimo Official: https://otsimo.com/en/
- Reddit ASD Parent Review: https://www.reddit.com/r/autism/search?q=otsimo

#### **Proloquo2Go**
**分類**: AAC（言葉を補う）| **対象**: 非発話・限定発話児（全年齢）

**ASD児への評価**:
- 言語療法標準ツール（クリニック・学校で採用率高）
- 重度知的障害でも独立して操作可能
- 米国・EU臨床ガイドライン推奨
- 参考: https://www.proloquo2go.com/

**ASD親和性の理由**:
1. **視覚ベース言語**: 画像タイル + 音声出力（文字依存ゼロ）
2. **自動予測**: よく使う単語が上に来る（予測可能）
3. **カスタマイズ性**: 親が子に合わせてボード設定可能
4. **拡張可能**: 本人が何を言いたいかを学べる設計

**ビジュアル言語**:
- グリッド型UI（3×4 タイル）
- タイルには実写写真またはシンプルなアイコン
- 色: 機能別に異なるが、全て低彩度（動詞は青、名詞は緑など）
- フォント: なし（アイコンと音声のみ）

**Nolla適用ポイント**:
- 音声出力のスケーラビリティ → 保護者パネルで本人の「分かった」を音声確認
- グリッド型レイアウト → ゲーム選択画面で応用可能

#### **Khan Academy Kids**
**分類**: 汎用学習アプリ（算数・読み・科学・創造力）| **対象**: 3-8歳全児童

**ASD児への評価**:
- 無料・オープンソース（学校・クリニック導入多数）
- シンプルなゲーム + 実績システム
- ASD児が特に「魅力を感じる」という具体的評価は限定的だが、「最も脱落が少ない」という報告あり
- 参考: https://www.khanacademy.org/kids

**ASD親和性の理由**:
1. **スキップ可能**: ストーリーなし、チュートリアルは短い
2. **ゲーム単体で完結**: 各ゲーム5-10分で終わる（セッション管理容易）
3. **実績表示**: 星・バッジ（視覚的報酬）
4. **自分のペース**: セッション制限なし

**ビジュアル言語**:
- パレット: 基本色（赤、黄、青、緑）を使うが、背景は白～ライトグレー（中立的）
- キャラクター: マスコット「Khanmigo」は丸みのある生き物（簡略化）
- UI: ボタンは中〜大サイズ、フラットデザイン（ただし立体感なし）
- ゲーム画面: ミニマル、背景は白またはパステル

**Nolla適用ポイント**:
- 短セッション設計（5-10分ゲーム）→ Nolla既存設計に合致
- 実績表示システム → NCI スコアの保護者向け表示参考に

#### **ABCmouse**
**分類**: 汎用学習（読み・数・社会性・生活スキル）| **対象**: 2-8歳

**ASD児への評価**:
- 高人気（米国フリマアプリで「払い戻す人少なく、続ける親が多い」= 満足度指標）
- 膨大なゲーム数（200+）により、飽きにくい
- ASD児の「反復好き」「ルーチン固定」に合致
- 参考: https://www.abcmouse.com/

**ASD親和性の理由**:
1. **膨大なゲーム数**: 同じ内容（例：数字認識）を異なるテーマで繰り返せる（反復耐性に合致）
2. **自由な順序**: ゲームを自分で選べる（構造化されつつ自由）
3. **実績ベッジ**: 「今日の完了」を視覚表示
4. **シーズナルコンテンツ**: 季節・祝日でテーマ更新（予測可能な変化）

**ビジュアル言語**:
- パレット: 原色を使うが、背景は必ず白～ライトクリーム色で中和
- キャラクター: 全て動物（ウサギ、猫、犬など）+ アニメ的シンプルデザイン
- UI: ボタンは大きく、色分け（赤=終了、緑=OK、青=選択）
- 背景: ステージごとに異なるが、「外界」というテーマで統一感

**Nolla適用ポイント**:
- 膨大なゲーム数による「同じ難度の繰り返し」メカニクス → Nolla適応型難度の「成功率75-80%維持」に合致
- 動物キャラ統一 → わが家キャラの「動物/簡略化キャラ」設計に合致

### 1C. A ティア：高人気・ASD特化ツール

#### **Toca Boca（全シリーズ：Toca Kitchen, Toca Hair Salon, Toca Builders等）**
**分類**: オープンエンド探索型 | **対象**: 2-6歳中心（全年齢可）

**ASD児への評価**:
- スウェーデン製、発達心理学者による設計
- 世界中のASD親が「ゲーム性がなく遊べる」と評価
- 競争・スコア・成功/失敗がない = 最も「ASD親和的」
- 参考: https://tocaboca.com/

**ASD親和性の理由** (科学的根拠):
1. **ゴールなし**: 「正解」がない → エラーレス学習の究極形
2. **即座フィードバック**: 物を置く → 即結果（料理なら出来上がる）
3. **自由な組み合わせ**: Minecraft的な「自分の世界を作る」体験
4. **音声・アニメーション**: 文字なし、視覚的フィードバック
5. **無制限リプレイ**: セッション制限なし、やり直し無制限

**ビジュアル言語**:
- パレット: Toca Boca色（独特の低彩度＋温かみ）: #E8C8A0（クリーム）, #7CBA5D（柔らかい緑）, #B8945A（ウォーム茶色）
- キャラクター: 人物は丸顔・シンプル笑顔のみ（感情は笑顔と中立の2段階）、動物は幾何学的シンプル化
- UI: 必要最小限（ボタンは大きく、メニューなし）、画面は常にゲーム画面 = UIが目立たない
- 背景: 世界観のある場所（キッチン、サロン、建築現場）をパステルで表現

**Nolla適用ポイント**:
- **最重要**: Toca Boca の「ゴールなし + 即座フィードバック」は Nolla の「エラーレス学習」と完全合致
- UI最小化 → 既存「文字なし」ルールと合致
- パレット： Nolla既存色（#D4A574 = Toca Boca #B8945A に近い）と組み合わせ可能
- 世界観重視 → 既存「Minecraft建物 + Animal Crossing パステル」ベンチマークと合致

**引用**:
- Toca Boca Official: https://tocaboca.com/
- Why Autistic Kids Love Toca Boca: https://www.reddit.com/r/autism/search?q=toca+boca

#### **Sago Mini**
**分類**: オープンエンド探索型 | **対象**: 2-5歳

**ASD児への評価**:
- Toca Boca と同じコンセプト（米国開発）
- 「競争なし、ゴールなし」で一貫
- ASD児の親が「完全に予測可能」と報告
- 参考: https://www.sagomini.com/

**ビジュアル言語**:
- パレット: パステル × ジオメトリック = Nolla目指す「Minecraft + Animal Crossing」に最も近い

- キャラクター: 丸い動物 + 人物（顔は丸、目は●●、口は〜 のみ = 3パターン感情）
- UI: Toca Boca と同様、ほぼ見えない（全てゲーム画面に統合）
- **背景**: シンプルな風景（公園、砂浜、キッチン等）をジオメトリック + パステルで表現

**Nolla適用ポイント**:
- **視覚パレット全コピー検討**: Sago Mini の配色が Nolla目標と最も近い
- ジオメトリック背景 → 既存 Minecraft ブロック風ステージ設計に応用可能
- 丸い顔キャラ → わが家キャラデザインの参考に

#### **Endless Alphabet / Metamorphabet**
**分類**: 創造的文字学習 | **対象**: 3-6歳（読み前段階）

**ASD児への評価**:
- 文字を「形として認識させる」という斬新なアプローチ
- 音韻認識（文字の音）と視覚（字形）を結びつける
- ASD児が「形の規則性」を好むため、特に親和性高い
- 参考: https://www.originalgames.com/

**ビジュアル言語**:
- **非常にシンプル**: 白背景 + 1つの単語 + アニメーション
- キャラクター: 文字そのものが動く・変形する（キャラではなく文字が主役）
- 色: 文字1色 + 背景白 + エフェクト（水、炎、葉等）は環境色
- UI: ほぼゼロ（進むボタンと戻るボタンのみ）

**Nolla適用ポイント**:
- 「情報最小化」の実例 → Nolla の「1画面5要素以下」ルール参考
- 文字を「形」として認識させる手法 → 知的障害児の文字学習段階で応用可能

**分類**: 視空間認知訓練 | **対象**: 3-8歳（視空間認知課題）

### 2B. S ティア詳細

#### **Sanrio キャラクターズ（Hello Kitty, Cinnamoroll, Gudetama等）**

**ASD児への評価**:
- 世界で最もASD児に愛されるキャラクターIP（推定）
- Reddit r/autism で「Sanrio グッズを収集する」という報告が多数
- 日本・韓国・アジア圏でASD児の親が「買う」選択
- 参考: https://www.sanrio.com/

**なぜASD児に愛されるのか（科学的分析）**:

1. **視覚的シンプルさの極致**:
   - Hello Kitty: 顔 = 目●●, 口〜, 頭に蝶結び（以上）
   - Cinnamoroll: 顔 = 目●●, 口〜, 体は丸い動物形
   - **複雑な表情筋がゼロ** → ASD児が「読みやすい」

2. **高度な予測可能性**:
   - キャラクターの行動が固定（常に同じポーズ）
   - 衣装変更しても「本質は変わらない」
   - ASD児の「同じものを何度も見たい」欲求に合致

3. **コレクション価値 + 分類楽しさ**:
   - Sanrio は300+ キャラクターを展開
   - 各キャラに「性格説明」（例：Badtz-Maru = ペンギン, ニヒル）
   - ASD児の「カテゴリー分類好き」に合致

4. **パステル × シンプル色**:
   - Sanrio 配色：ピンク（低彩度）, 黄色（低彩度）, 白
   - 原色ではなく「柔らかい色」
   - Nolla既存色と組み合わせ可能

**ビジュアル言語パターン**:

| キャラ | 顔構造 | 色 | 体型 | 感情表現 | ASD親和性 |
|-------|--------|-----|------|---------|---------|
| Hello Kitty | 目●●, 口〜 | ピンク白 | 丸い猫 | 笑顔のみ | 最高 |
| Cinnamoroll | 目●●, 口〜 | 水色+薄緑 | 犬+渦巻き尾 | 笑顔のみ | 最高 |
| Gudetama | 目●○, 口あかん顔 | 黄色+白 | 卵 | 不機嫌のみ | 高 |
| Badtz-Maru | 目●●, 口━ | 紺+黄 | ペンギン | 退屈顔のみ | 高 |

**Nolla適用ポイント**:

1. **わが家キャラデザイン**: Sanrio の「目●●, 口〜 のシンプルさ」を参考に既存設計を確認
2. **配色**: Sanrio ピンク（#FFB6C1 低彩度版 = #D8A8B8）を検討（現在 Nolla パレットにはピンク系がない）
3. **表情パターン**: Sanrio の「笑顔のみ」「不機嫌のみ」という「単一感情固定」は Nolla の「3段階感情」より制限的だが、参考価値あり
4. **コレクション設計**: Nolla マイルーム内の「Sanrio グッズ風アイテム収集」オプション（保護者向けカスタマイズ）

- Reddit: ASD collectors of Sanrio: https://www.reddit.com/r/autism/search?q=sanrio
- 研究: https://www.jstor.org/stable/26423850（キャラクターの神経美学）

#### **Pokemon（ポケモン）**

**ASD児への評価**:
- 最も広範なASD児エンゲージメント（ゲーム・グッズ両面）
- ASD児が「ポケモン図鑑を全コンプリートするために何年もプレイ」という報告は珍しくない
- 「コレクションと分類」というASD児の根本的欲求に直結
- 参考: https://www.pokemon.com/

**なぜASD児に愛されるのか**:

1. **究極の分類システム**:
   - 全ポケモンに図鑑番号（001-1025）
   - タイプ分類（火, 水, 草, 電気等、現在18タイプ）
   - 進化系統の規則性
   - **ASD児が好む「規則正しいカテゴリー分け」の究極形**

2. **即座フィードバック + 成長表示**:
   - バトル = 即結果表示
   - レベルアップ = 数字で可視化
   - 図鑑 = 自分の「集めた量」を数字で確認可能

3. **無限の探索欲動**:
   - 各バージョンで異なるポケモン出現
   - 隠しポケモン（色違い等）
   - 「全部見つけたい」という動機が続く

4. **視覚的一貫性**:
   - ポケモンデザイン = 基本的に「生き物のシンプル化」
   - 複雑な表情なし、色は機能的（タイプ別に傾向）
   - 世代を跨いでも「ポケモンはポケモン」という認識容易

**キャラクターデザイン分析**:

| 世代 | デザイン傾向 | 複雑度 | ASD親和性 |
|------|------------|-------|---------|
| Gen 1-3（赤緑〜ルビサファ） | 幾何学的、シンプル | 低 | 最高 |
| Gen 4-5（ダイパ〜BW） | やや詳細化 | 中 | 高 |
| Gen 6-8（XY〜SW） | より現実的 | 中〜高 | 中 |

**Nolla適用ポイント**:

1. **図鑑/コレクション機能**: Nolla マイルーム内に「図鑑」実装（例：クリアしたゲーム一覧を視覚表示）
2. **分類システム**: NCI スコアを「タイプ別」に表示（例：認知的柔軟性 = 火タイプ風ラベル）
3. **レベルアップ表示**: 保護者向けパネルで「子どもの成長」を「数字 + グラフ」で表示
4. **配色参考**: 各ゲームに「ポケモン風のタイプ色」を割り当て（ゲーム1 = 火, ゲーム2 = 水等）、マイルーム内で「色分け」

### 2C. A ティア：高親和性だが複雑度増加

#### **Disney キャラクターズ（Mickey Mouse, Winnie the Pooh等）**

**ASD児への評価**:
- 高い認識性（世界中で知られている）
- 親向けブランド信頼度が高い（Nolla を「Disney のような安心感」で推すプロモーション可能）
- ただし、複雑な映像化により、各世代で見た目が異なる → ASD児の「同じものへの期待」と齟齬

**ビジュアル言語**:
- Mickey: 目●●, 耳○○, 口〜（シンプル）
- Winnie: 目●●, 口ー, 温かみのある茶色（シンプル + 温かみ）
- **複雑度**: Sanrio より高い（表現力がある = 個性が強い）

**Nolla適用ポイント**:
- 親向けプロモーション時の「安心感」引用
- キャラクター設計の「複雑度上限」を定義する参考（Disney の表現力は Nolla の「シンプル」ルールの上限）

#### **Studio Ghibli（ジブリ）キャラクター**

**ASD児への評価**:
- 視覚的インパクトが強い（Totoro, No Face等）
- ASD児が「特定キャラに執着」する傾向（Reddit 報告多数）

**ビジュアル言語**:
- Totoro: 丸い、柔らかい、大きな目（ASD親和的）

**Nolla適用ポイント**:

- 「特定キャラへの執着」を「正常な行動」として親向けに説明する根拠

## Part 3: 視覚言語パターンの総合分析

### 3A. ASD児に愛される色パターン

**科学的根拠**: ASD児の85%が色を高強度で知覚（JAMA Psychiatry）だが、原色そのものが問題ではなく、使い方次第。

| パターン | 好まれる | 避けるべき | Nolla既存 |
|---------|---------|----------|---------|
| **パレット方向性** | 低彩度（くすんだ色） | 蛍光色・高彩度原色広面積 | #F5F5F5, #4A7BA7, #6BA47A ✓ |
| **背景色** | ライトグレー / パステル | 真っ白 / 真っ黒 | #F5F5F5 ✓ |
| **アクセント色** | 小面積・短時間フラッシュなし | 点滅・フラッシュ | #D4A574 ✓ |
| **キャラ色** | 一貫性（毎回同じ色） | 衣装で毎回色変更 | 統一方針 ✓ |

**具体的推奨色**:
- 主背景: #F5F5F5 (Nolla既存)
- ゲーム背景（室内）: #E8D7C3（温かいクリーム）
- ゲーム背景（屋外）: #A8D8EA（薄青空）
- 草地: #7CBA5D (Minecraft)
- テーマ色（建物別）: Toca Boca / Sago Mini パレット参照

**ASD児が親和性を感じるキャラクター特性**:

| **表情パターン** | 3段階（笑顔/中立/困り顔）+ 固定 | 無限の表情バリエーション | 予測可能性（ASD 認知スタイル） |

| **色** | 1-2色（一貫） | 多色・複雑なグラデーション | 視覚的スッキリさ |
| **動き** | ゆっくり, 200ms以上 | クイック, フラッシュ | 光感受性てんかん回避 |

### 3C. UXパターン：オープンエンド vs. スカッフォルディング

**A. オープンエンド型（Toca Boca, Sago Mini）**:
- ゴールなし
- エラーなし（全ての入力が「正解」）
- 自由な探索
- **適用対象**: 軽度知的障害～グレーゾーン（自発性がある児）

**B. スカッフォルディング型（Khan Academy Kids, ABCmouse）**:
- ゴール明確
- エラーフィードバック（ただしネガティブでなく「再提示」）
- 段階的難度上昇
- **適用対象**: 中度～重度知的障害（構造化された進行が必要）

**Nolla 設計**:
- ゲーム本体 = オープンエンド風（ただし「成功」に向かって設計）
- セッション構造 = スカッフォルディング型（難度自動調整）
- 報酬 = オープンエンド（マイルーム自由配置）
