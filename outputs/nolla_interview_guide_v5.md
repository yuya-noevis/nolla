# Nolla ユーザーインタビューガイド v5

**前版**: nolla_interview_guide_v4.md（_archive/へ移動）
**更新日**: 2026-04-20
**設計原則**: The Mom Test（Rob Fitzpatrick）準拠

---

## v4→v5の設計転換

### v4の反省（1件目実施後に判明）
1. **診断前〜診断直後の話が全く引き出せなかった**。現状の困りごとしか取れず、早期介入に寄与するプロダクトとして必要な「診断プロセスの空白」「親が不安で動けない時期に何を欲していたか」の一次情報が欠落していた。
2. **可視化ニーズの深掘りに時間を使いすぎた**。可視化機能は実装予定で、あって困るものではない。検証の必要性が低い。
3. **ソリューション説明をしてしまいバイアス混入**。「AI駆動の成長支援プロダクトの説明に対して即座に前向きな反応」といった記述はThe Mom Test違反。以降の回答が汚染される。

### v5での変更
- **新セクション S2「診断までの歩み」を追加**（7分）。現状→遡って診断前後の情報収集行動・感情・ギャップを事実ベースで引き出す。
- **S4「成長実感」を3分に短縮**、可視化の需要検証は最小限（1問のみ）。
- **「絶対に聞いてはいけない質問」にソリューション説明禁止を明記**。インタビュアー向け行動規範も強化。
- **早期介入寄与プロダクトとしての検証軸 H4 を追加**。

## このガイドが検証する4仮説

| # | 仮説 | 検証の意味 |
|---|------|-----------|
| H1 | 親は家庭内の発達支援に課題を感じており、既存の代替手段では解決できていない | 家庭向けツールの需要の実在性 |
| H2 | 子どもの成長の「現在地」を把握できていない親が多い | 可視化機能の需要（参考情報扱い。深掘り不要） |
| H3 | 「子ども直接利用型」と「親支援ツール型」で、需要の強さが異なる | MVPの方向性決定 |
| H4 | **診断前〜診断直後に、親の情報・支援アクセスに空白がある** | **早期介入寄与プロダクトとしての価値検証** |

## インタビュー対象者

### 保護者（10〜15名）

| 条件 | 内容 |
|------|------|
| 子どもの年齢 | 3〜12歳 |
| 障害種別 | ASD・ADHD・知的障害・ダウン症・境界知能・グレーゾーンのいずれか（診断の有無は問わない） |
| 必須条件 | 過去1年以内に何らかの発達支援を利用した経験がある、または診断を受けた経験がある |
| 除外条件 | 直接の友人・家族（The Mom Test原則） |

### リクルーティング経路

| 経路 | 対象 | 目標人数 |
|------|------|---------|
| 放デイ・療育教室経由 | 既に制度を利用している層 | 7〜10名 |
| SNS・親の会・オンラインコミュニティ | 施設未接続層・軽度/グレーゾーン層 | 3〜5名 |

### サンプル構成の注意
- 「子どもがデバイスを使える層」と「使えない層」の両方を含むこと
- 診断時期で「早期診断層（0-3歳）」「就学前診断層（3-6歳）」「就学後診断層（6歳以降）」を分散させる
- 施設経由のみだと高リテラシー層に偏る。SNS・親の会から未診断・未接続層を含める

| セクション | 目的 | 時間 |
|-----------|------|------|
| S0. オープニング | ラポール構築・録音許可 | 2分 |
| S0.5. 背景確認 | セグメント情報の取得 | 2分 |
| S1. 日常と文脈 | 生活リズム・既存支援 | 5分 |
| **S2. 診断までの歩み（新設）** | **H4検証：診断前後の情報・支援アクセスの空白を特定** | **7分** |
| S3. 家庭での関わりとデバイス | H1・H3の核心 | 15分 |
| S4. 成長実感 | H2の軽検証（深掘り不要） | 3分 |
| S5. 優先順位と支出 | 問題の序列・支払い実績 | 5分 |
| クロージング | 想定外情報・紹介・MVP協力者 | 3分 |
| バッファ | 余白 | 3分 |

## S0. オープニング（2分）

> 「今日はお子さんの日々の生活や、発達支援のことについてお話を聞かせてください。正解や不正解はありませんので、思ったままお話しいただければ大丈夫です。メモと録音をさせていただいてもよろしいですか？」

**※ この時点でプロダクト名・機能・コンセプトは一切言わない。**

---

## S0.5. 背景確認（2分）

**目的：セグメント分析に必要な基本情報を取得する。**

> 「最初に、差し支えない範囲でお子さんのことを教えてください」

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 0-1 | 「お子さんは今おいくつですか？」 | 年齢帯の確認 |
| 0-2 | 「お子さんの特性や診断について、差し支えない範囲で教えていただけますか？」 | 障害種別。診断の有無を含む |
| 0-3 | 「ご家族の構成を教えてください。きょうだいはいますか？」 | 家族構成・きょうだい児の有無 |

**※ 運用注意：** 深掘りしない。事実の確認のみ。「大変ですね」等の共感コメントは入れない（バイアス防止）。

## S1. 日常と文脈（5分）

**目的：生活リズムと既存の支援手段を把握する。**

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 1-1 | 「お子さんの平日は、普段どんなふうに過ごしていますか？」 | 時間の使い方。支援に割ける余白 |
| 1-2 | 「今通っている支援サービスは何ですか？」 | 代替手段の把握 |

**→ フォローアップ:**
- 「始めたきっかけは？」
- 「続けている理由は？（不満はありますか？）」
- 「先生から『家でこれをやってください』と言われていることはありますか？」

---

## S2. 診断までの歩み（7分） ← v5 新設

**目的：H4検証。診断前〜直後の情報・支援アクセスの空白を事実ベースで把握する。**

**※ 設計意図：** 早期介入に寄与するプロダクトとしての価値を検証するには、「現在の困りごと」だけでなく「過去に何を知りたかったか」「何ができず困ったか」が必要。ここでは仮定の未来ではなく、過去の行動事実を引き出す。

### Part 1: 最初の違和感〜診断まで（4分）

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 2-1 | 「最初に『あれ？』と思ったのは、いつ、どんな場面でしたか？」 | 気づきのトリガー。年齢・具体エピソード |
| 2-2 | 「その時、最初にしたことを教えてください。誰かに相談しましたか、調べましたか？」 | 情報探索の事実。手段（検索／相談相手／書籍等） |
| 2-3 | 「診断（または診断疑い）に至るまで、どのくらいの時間がかかりましたか？その間、どうしていましたか？」 | 期間の長さ・その間の行動 |
| 2-4 | 「その時期、『これが知りたかった』『これがあれば良かった』と思うものはありましたか？」 | 未充足ニーズの過去事実。「今欲しい」ではなく「その時欲しかった」を聞く |

### Part 2: 診断直後（3分）

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 2-5 | 「診断を受けた時、または『支援が必要』と言われた時、どんなお気持ちでしたか？」 | 感情の事実。脚色せず聞く |
| 2-6 | 「診断直後の数週間〜数ヶ月で、具体的に何をしましたか？（情報収集・相談・療育探しなど）」 | 行動の事実。時系列の事実を引き出す |
| 2-7 | 「その頃、一番困ったこと、もしくは『何から手をつけていいかわからない』と感じたことは何ですか？」 | 空白の具体化 |

### Part 3: 未診断層への質問（2-1〜2-4 で診断未確定と判明した場合のみ）

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 2-8 | 「今、受診や相談をまだしていない／迷っているとしたら、何が引っかかっていますか？」 | 受診障壁の事実 |
| 2-9 | 「具体的に今、一番知りたいことは何ですか？」 | 未充足情報ニーズ |

**※ S2全体の運用注意:**
- プロダクトを持ち出さない。「こんなアプリがあったら〜」は絶対に言わない
- 「大変でしたね」等の共感コメントは最小限（ラポール程度）。バイアス源
- 感情の質問（2-5）では5秒以上の沈黙を許す。先回りしない
- 重要発言は **exact quote** で記録

## S3. 家庭での関わりとデバイス（15分）

**目的：家庭内支援の実態と課題を把握した上で、デバイス利用の可否を確認し、「子ども直接利用型」と「親支援ツール型」の需要を測る。**

### Part 1: 家庭での取り組みと課題（7分）

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 3-0 | 「療育や支援サービス以外で、家で何かお子さんの成長のためにやっていることはありますか？」 | 家庭内支援の実態 |
| 3-1 | 「お子さんの特性や行動で、最近一番大変だったことは何ですか？それはいつ起きましたか？」 | 具体的エピソード |
| 3-2 | 「その問題に対して、何か試しましたか？結果どうでしたか？」 | 既存手段のギャップ |
| 3-3 | 「その時、どう対応すればいいかわからない、と感じたことはありましたか？」 | 親支援ツール需要の測定（H3） |

### Part 2: デバイスの利用実態（8分）

**ゲート質問**

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 3-4 | 「お子さんは一人でスマホやタブレットを操作できますか？」 | デバイス利用の可否 |
| 3-4b | （No/補助ありの場合）「具体的にどういった部分が難しいですか？」 | 運動機能・認知・感覚過敏等の切り分け |

**※ デバイス利用レベル:** [一人で可能 / 補助があれば可能 / 不可] の3段階で記録

**ルートA: 子どもがデバイスを使える場合**

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 3A-1 | 「普段どんなふうに使っていますか？何を見たり遊んだりしていますか？」 | 現在の利用実態 |
| 3A-2 | 「普段どのくらいの時間使っていますか？」 | スクリーンタイム |
| 3A-3 | 「使いすぎて止めたことや、時間のルールを決めていますか？」 | 許容ラインの行動ベース確認 |
| 3A-4 | 「発達支援に使えるアプリやサービスがあること自体は知っていましたか？」 | 認知の有無 |
| 3A-5 | （知っていた場合）「実際に使ったことはありますか？」 | 認知→行動の転換率 |
| 3A-6 | （使った場合）「使ってみて、お子さんはどんな反応でしたか？」 | 子どもの食いつき |
| | → フォローアップ:「今も続けていますか？」→ Noなら「何がきっかけでやめましたか？」 | 離脱理由 |
| 3A-7 | （知っていて使わなかった場合）「知った上で使わなかったのは何か理由がありますか？」 | 不採用理由 |

**ルートB: 子どもがデバイスを使えない場合**

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 3B-1 | 「お子さんの療育や成長支援について、ご自身で情報を調べたり、誰かに相談したりしていますか？」 | 親自身の情報探索行動 |
| 3B-2 | 「最近、療育や家での関わり方について調べたり、誰かに相談したりしましたか？いつ、どこで（誰に）でしたか？」 | 情報探索の事実 |
| 3B-3 | 「家で何か独自に取り組んでいることはありますか？どんなことを、どのくらいの頻度で？」 | 家庭内支援の実態 |

## S4. 成長実感（3分） ← v4のS3を短縮

**目的：H2の軽検証のみ。可視化機能は実装予定のため深掘り不要。**

| # | 質問 | 聞きたいこと |
|---|------|-------------|
| 4-1 | 「お子さんがこの半年で変わったなと思うことがあれば、教えてください」 | 成長認知の解像度。即答性（5秒以内に具体的回答が出るか）を記録で判定 |
| 4-2 | 「お子さんの発達が『順調かどうか』を、今どうやって判断していますか？」 | 現在の代替手段。1問で済ませる |

**※ v4のQ3-2（効果確認行動の有無）は削除。情報密度が低かった。**

## 絶対に聞いてはいけない質問・言ってはいけないこと

| ❌ 禁止 | 理由 | ✅ 代替 |
|---------|------|--------|
| 「このアプリ使いたいですか？」 | 社交辞令Yes | 「実際に試したアプリはありますか？どうでしたか？」 |
| 「月額980円なら払いますか？」 | 仮定の支払い意思 | 「今、発達支援に月いくら使っていますか？」 |
| 「こういうアプリがあったら便利だと思いますか？」 | 仮定の未来 + 全員Yes | 「発達支援に使えるアプリがあること自体は知っていましたか？」 |
| 「Nollaというアプリを作っています」 | 以降の回答が汚染される | **ソリューションの話は全体を通じて出さない** |
| **「AI駆動の成長支援を作っていて〜」「こんな機能があったら〜」** | **v4で実際に汚染が発生した。絶対禁止** | **最後まで質問に徹する。説明したい衝動に負けない** |
| 「〜と思いますか？」系の意見を聞く質問 | 願望・推測は事実ではない | 「〜をしたことがありますか？いつ、どこで？」事実を聞く |

## インタビュー中の行動規範

| ルール | 理由 |
|--------|------|
| 話す量は全体の20%以下 | 80%は相手に話させる |
| 具体的エピソードが出るまで深掘り | 「大変です」は情報ゼロ。「先週〇〇が起きて…」が情報 |
| 沈黙は5秒待つ | 相手が考えている。埋めると浅い回答になる |
| 褒め言葉・「いいですね！」は無視 | 社交辞令はデータではない |
| 「他に何かありますか？」を3回聞く | 1回目は社交辞令。3回目に本音が出る |
| **プロダクト説明は終始しない** | **最後にC-2「試作段階のもの」と言う以外、アプリ・AI・機能の具体性を出さない** |
| 過去の事実で質問する | 「その時どうしましたか？」「最後に〇〇したのはいつ？」 |
