# Nolla Cognitive Index（NCI）アルゴリズム設計

**作成日**: 2026-04-05
**ステータス**: CONFIRMED（2026-04-05 Yuya承認）
**参照**: nolla_game_mechanics_design.md、nolla_target_user_iq_distribution_research.md

## 0. このドキュメントの位置づけ

NCIは Nolla の最重要技術資産。ゲームプレイデータから認知機能を**臨床テスト以上の精度で、継続的に、見えない形で**測定するシステム。

- 保護者向け: 成長の可視化 → 継続率向上
- 施設・学校向け: 個別支援計画の数値的根拠
- 研究機関向け: 匿名化データAPIの基盤（将来収益源）
- 事業価値: 特許対象になり得る知的財産

---

## 1. NCIの4軸

| 軸 | 名称 | 測定元ゲーム | 測定内容 |
|----|------|------------|---------|
| NCI-M | Memory（記憶） | おぼえて + まねして | ワーキングメモリ容量 + 空間記憶 |
| NCI-F | Flexibility（柔軟性） | なかまわけ | 認知的柔軟性 + 抑制制御 |
| NCI-A | Attention（注意） | みつけて | 選択的注意 + 視覚探索効率 |
| NCI-S | Speed（処理速度） | 全ゲーム共通 | 認知処理速度（運動速度を分離済み） |

各軸は **0.000〜999.999** の連続値。内部用。保護者には変換後のスコアを見せる。

## 2. 数学的基盤

### 2.1 IRT 2PLモデル（能力推定の核）

各試行を「テスト問題」として扱い、子どもの能力パラメータ θ を推定する。

```
正答確率:
  P(正解 | θ, a, b) = 1 / (1 + exp(-a(θ - b)))

  θ: 子どもの認知能力（推定対象）
  b: その試行の難易度（ゲームパラメータから算出）
  a: その試行タイプの識別力（能力の違いをどれだけ見分けるか）
```

**パラメータ b（難易度）の算出**:

各ゲームのパラメータ値を1つの難度スコアに変換する関数。

```
神経衰弱:
  b = w1 * log(pairs) + w2 * (similarity/100) + w3 * (1 - flip_delay/2000)
  初期重み: w1=0.5, w2=0.3, w3=0.2（実データで校正）

分類:
  b = w1 * log(categories) + w2 * log(items) + w3 * criterion_complexity + w4 * switching_factor
  criterion_complexity: color=0.1, shape=0.3, size=0.4, category=0.6, multi=0.9
  switching_factor: none=0, between_rounds=0.3, within_round=0.6

視覚探索:
  b = w1 * log(scene_items) + w2 * diff_count_factor + w3 * (diff_subtlety/100)
  diff_count_factor: 1個=0.1, 2個=0.2, 3個=0.35, 4個=0.5, 5個=0.65, 6個=0.8, 7個=0.9

Corsi Block:
  b = w1 * log(blocks) + w2 * seq_length_factor + w3 * (1 - display_ms/1500)
  seq_length_factor: seq_length / 9（最大シーケンス長で正規化）
```

**パラメータ a（識別力）の算出**:

初期値は全試行タイプで a=1.0。実データ蓄積後に試行タイプごとに校正。
識別力が高い試行タイプ = 能力の高低をよく見分ける問題。

### 2.2 ベイズ逐次更新（精度の向上）

各試行後に θ の事後分布を更新する。

```
初期（ベースライン確立前）:
  事前分布: θ ~ N(μ_0, σ²_0)
  μ_0 = 500（スケール中間値）
  σ_0 = 150（広い不確実性）

試行 t 後の更新:
  θ_t | データ ~ N(μ_t, σ²_t)

  μ_t = (σ²_likelihood * μ_{t-1} + σ²_{t-1} * θ_MLE) / (σ²_likelihood + σ²_{t-1})
  σ²_t = (σ²_likelihood * σ²_{t-1}) / (σ²_likelihood + σ²_{t-1})

  θ_MLE: 今回の試行結果から推定される最尤能力値
  σ²_likelihood: 今回の試行の情報量の逆数

効果:
  試行が増える → σ_t が小さくなる → 推定精度が上がる
```

### 2.3 精度推移の目安

| プレイ量 | 累計試行数（目安） | 標準誤差 SE | 信頼区間幅 | 精度レベル |
|---------|-----------------|-----------|----------|----------|
| 初回セッション | ~10 | ±45 | 90 | 粗い推定 |
| 8日分ベースライン | ~50 | ±15 | 30 | 実用的 |
| 1ヶ月 | ~200 | ±5.0 | 10 | 高精度 |
| 3ヶ月 | ~800 | ±1.8 | 3.6 | 超高精度 |
| 6ヶ月 | ~2000 | ±0.8 | 1.6 | 微細な変化を検出可能 |

**重要**: 上記はプレイ回数ベース。暦日数ではない。週1プレイの子は6ヶ月後でも累計~100試行で、精度は「1ヶ月相当」。

## 3. 反応時間の運動/認知分離（DDM）

### 3.1 問題

反応時間には「頭の速さ」と「手の速さ」が混在している。運動機能に課題がある子ども（発達性協調運動症: ASD児の63%に併存）は、認知は速くても反応時間が遅い。

### 3.2 Drift Diffusion Model

```
全体の反応時間 RT = 認知処理時間 Td + 運動実行時間 Tr + ノイズ

DDMパラメータ:
  v（ドリフト率）: 情報蓄積の速さ = 認知処理速度
  a（境界分離）: 反応に必要な確信度の閾値
  t0（非決定時間）: 知覚入力 + 運動出力にかかる時間

Nollaで使うのは:
  v → NCI-S（認知処理速度）に直接反映
  t0 → 運動ベースラインとして除外
```

### 3.3 運動ベースラインの測定

```
タイミング: チュートリアル時 + 各セッション冒頭
方法: 「光ったブロックをすぐタップ」の単純反応課題（認知負荷ゼロ）
回数: 10回
算出: 中央値を採用（外れ値に頑健）

更新: セッションごとに新しい計測値を加重平均で統合
  motor_baseline_new = 0.7 * motor_baseline_old + 0.3 * 今回の中央値
  → 体調や成長による運動速度の変化に追従
```

### 3.4 認知処理時間の算出

```
各試行:
  cognitive_RT = total_RT - motor_baseline

  cognitive_RT < 0 の場合:
    → motor_baseline が過大推定の可能性。その試行のRT情報は使わない

  個人内相対化:
    z_cognitive_RT = (cognitive_RT - 個人平均cognitive_RT) / 個人SD
    → この z値 が NCI-S の更新に使われる
```

## 4. 異常セッション検出

### 4.1 検出パターンと処理

| パターン | 検出条件 | 解釈 | NCI更新への影響 |
|---------|---------|------|----------------|
| 体調不良 | 正答率が個人平均から 2SD 以上低い | パフォーマンス低下は一時的 | 重み 0.2（ほぼ無視） |
| 適当プレイ | 反応時間が個人平均の 30% 以下 | 考えずにタップしている | 重み 0（NCI更新に使わない） |
| 疲労 | セッション後半の正答率がセッション前半より 20%以上低下 | 認知疲労 | 前半のみ使用 |
| 連打 | 同じ選択肢を 3回以上連続タップ | ゲームとして遊んでいない | 重み 0 |
| 外的要因 | セッション中に長い中断（>30秒）が複数回 | 注意が他に向いている | 重み 0.5 |

### 4.2 異常検出のアルゴリズム

```
各セッション終了時:
  1. 正答率を個人の移動平均（直近20セッション）と比較
  2. 反応時間の分布を個人の累積分布と比較
  3. セッション内の正答率推移を分析（前半/後半比較）
  4. タップパターンを分析（連打、規則的パターン等）
  5. タッチダイナミクス異常検出（圧力急増 = 不安/フラストレーション）
  6. デバイスモーション異常検出（揺れ急増 = 注意散漫/常同行動）

異常スコア = f(正答率乖離, RT乖離, 疲労指標, タップ異常度,
              タッチ圧力乖離, デバイス安定度乖離)

異常スコア > 閾値 → そのセッションのNCI更新重みを低下
データ自体は全て保存（行動分析、長期トレンドには使用する）
```

## 5. ベースライン確立

### 5.1 条件

```
ベースライン確立の最低条件:
  - 各ゲーム最低 8セッション以上
  - 8日間（暦日）以上経過
  - 異常セッションを除外した有効セッションが 6以上

ベースライン確立前:
  - NCI は内部で計算されるが、保護者には非表示
  - 保護者画面: 「お子さまのデータを計測中です（あと○回）」
  - 進捗バーでカウントダウン表示
```

### 5.2 精度の段階的向上

```
Phase 1（ベースライン確立〜1ヶ月）:
  NCI は粗い推定値。保護者には大まかな方向性のみ表示
  「記憶力: 計測中（もう少しデータが必要です）」

Phase 2（1〜3ヶ月）:
  NCI の精度が実用的になる。スコア表示開始
  「記憶力: 68 ↑ 伸びています」

Phase 3（3ヶ月〜）:
  高精度。微細な変化を検出可能。トレンド分析が有効に
  「記憶力: 72 ↑ 3ヶ月間で着実に伸びています」

Phase 4（6ヶ月〜）:
  超高精度。長期成長曲線が描ける
  「記憶力: 半年前と比べて22%向上しました」
```

## 6. NCI → 保護者・施設向けスコア表示

### 6.1 基本方針: 絶対評価（θのみ）

```
【確定方針（2026-04-05）】
保護者向けも施設向けも、同じθ（能力値）を表示する。
混合スコア（エンゲージメント等を混ぜた加工値）は使わない。

理由:
  1. 保護者と施設で同じ数字 → 信頼性・一貫性
  2. 障害児の保護者は「低い=ダメ」とは捉えない。
     正確な現在地を知ることが適切な支援につながる
  3. 施設・教師には加工されていない正確な数値が必須
  4. 計算がシンプル。余分な重み付け不要
```

### 6.2 表示スコアへの変換

```
内部 NCI（0-999.999）→ 表示スコア（0-100）

変換式:
  display_score = NCI_current / scaling_factor

  scaling_factor: 母集団の校正データから決定（ローンチ後に精緻化）
  
  ※相対評価ではない。個人のθを固定スケールに変換するだけ。
  ※他の子との比較・偏差値ではない。
```

### 6.3 表示の段階的開放

```
ベースライン確立前（~32セッション）:
  NCI は非表示。保護者画面:
  「お子さまのデータを計測中です（あと○回）」
  + ゲーム結果・プレイ履歴・獲得報酬は表示OK

ベースライン確立直後（信頼区間が広い）:
  スコアは表示するが、表記を控えめに:
  「記憶力: 計測中（おおまかな傾向が見え始めました）」
  → 数値の横に信頼区間の広さを「精度」として可視化

精度が実用的になった後（~1ヶ月）:
  スコア + トレンド矢印を表示:
  「記憶力: 68 ↑」

高精度（~3ヶ月以降）:
  スコア + トレンド + 長期比較:
  「記憶力: 72 ↑ 開始時と比べて着実に伸びています」
```

### 6.4 トレンド表示の色

| トレンド | 色 | 表現 |
|---------|-----|------|
| 上昇傾向 | 緑 | 「伸びています」 |
| 横ばい | 黄 | 「安定しています」 |
| 一時的低下 | 青 | 「今はじっくり取り組んでいます」 |

**赤は使わない。** 下がっても「青」。理由: 不安を与えない。

### 6.5 θが横ばいの子どもへの対応

能力θが横ばいでも、**別の情報として**（スコアに混ぜずに）以下を表示する:

```
認知スコアとは別枠で表示:

「今週のプレイ」
  - プレイ回数: 5回
  - ヒントなしで解けた率: 60% → 75%
  - 新しく覚えたカード: 3種類
  - 集中できた時間: 平均8分

これらはNCIスコアには混ぜない。独立した情報として見せる。
保護者がスコア以外の「がんばり」を確認できる場所を用意する。
```

**重要**: エンゲージメントや自立度を認知スコアに混ぜない。混ぜると「認知機能の指標」としての信頼性を失う。

## 10. 校正計画

### 10.1 内部校正（ローンチ前）

```
IRT パラメータ（a, b）の初期値:
  - ゲームパラメータから理論的に算出（2.1節の数式）
  - 初期重みは仮設定。実データで校正必須
  - bの初期値は保守的に設定する（循環依存バイアスの抑制。13.1参照）

DDM パラメータの初期値:
  - 定型発達児の文献値をベース
  - ASD児特有の補正は実データから推定
  - MVPでは簡易版を使用（13.2参照）
```

### 10.2 外部校正（ローンチ後）

```
Phase 1（ローンチ後3ヶ月）:
  - 50-100名の被験者にWISC-V（臨床IQテスト）を実施
  - NCIとWISC-Vの相関を検証
  - 目標: r > 0.7（高い相関）

Phase 2（ローンチ後6ヶ月）:
  - NCI予測精度の検証（NCI変化 → 実際の認知変化の一致度）
  - IRTパラメータの校正（実データから a, b を再推定）
  - DDMの補正係数をASD児データで更新

Phase 3（1年後〜）:
  - 大規模データで校正精度を向上
  - 年齢×障害種別×IQ帯域ごとのノーム作成
  - 論文発表 → 学術的妥当性の確立
```

## 11. 実装上の注意

### 11.1 計算コスト

```
IRT更新: 試行ごとに O(1)。リアルタイム処理可能
DDM推定: セッション単位で O(n)。バッチ処理可能
異常検出: セッション終了時に O(n)。バッチ処理可能

全て端末側で計算可能。サーバー側は保存のみ。
ただしDDMのフルモデル推定はサーバー側で日次バッチ推奨。
```

### 11.2 プライバシー

```
NCI値は個人の認知機能を高精度で推定する → 要配慮個人情報

- NCI値はサーバー上で暗号化保存
- 研究機関向けAPIは完全匿名化（個人特定不可能な状態）
- 保護者のみがNCI値を閲覧可能（PINで保護）
- 児童本人にはNCI値を見せない
```

### フェーズ別の閲覧手段

## 12. 拡張データソース（タッチ・センサー統合）

タッチスクリーンの正誤データだけでなく、デバイス内蔵センサーを活用してNCIの精度と多次元性を大幅に向上させる。詳細な取得仕様は `nolla_game_mechanics_design.md` の8.2-8.4節を参照。

### 12.1 データソース一覧と実装時期

| # | データソース | 取得手段 | NCI統合先 | 実装時期 |
|---|------------|---------|----------|---------|
| 1 | 正誤 + 反応時間 | タッチイベント | NCI全4軸（基幹データ） | MVP |
| 2 | タッチダイナミクス | タッチAPI拡張 | NCI-S精度向上 + 覚醒/不安推定 | MVP |
| 3 | デバイスモーション | ジャイロ/加速度 | セッション品質 + 注意集中度 | MVP or Phase 1 |
| 4 | 環境騒音 | マイク（dB値のみ） | セッション品質補正 | Phase 2 |
| 5 | 眼球追跡 | フロントカメラ | NCI-A精度向上 + 注視パターン | Phase 2-3 |
| 6 | 発声分析 | マイク（要同意） | 言語発達モニタリング | Phase 2-3 |

### 12.2 タッチダイナミクスのNCI統合

```
【NCI-S（処理速度）への統合】
  従来: reaction_time のみ → DDM で運動/認知分離
  拡張: reaction_time + touch_force + touch_duration + inter_tap_interval
        → 多変量DDMで、より精密な認知処理速度推定

【確信度推定（新指標）】
  touch_duration が短い + touch_trajectory が直線的 = 高確信度
  touch_duration が長い + touch_trajectory に迷いがある = 低確信度
  → 確信度は NCI の補助指標として記録
  → 「正解だが確信度が低い」= まぐれ当たりの可能性 → NCI上昇幅を抑制

【覚醒・不安レベル推定】
  touch_force の個人内偏差 → 覚醒レベルの間接指標
  圧力が個人平均から 1.5SD 以上 → 不安/フラストレーションの可能性
  → 異常セッション検出の補助データ
```

### 12.3 デバイスモーションのNCI統合

```
【セッション品質スコア（新指標）】
  device_stability = 1 / (gyro_variance + accel_variance)
  高い = デバイスが安定 = 集中している
  低い = デバイスが揺れている = 注意散漫 or 常同行動

  セッション品質スコア = f(device_stability, session_duration, 正答率推移)
  → NCI更新の重み付けに使用（品質が低いセッション = 重みを下げる）

【注意持続時間の推定】
  セッション開始からdevice_stabilityが閾値以下に落ちるまでの時間
  → 「集中持続時間」として保護者レポートに表示可能
  → 「集中できている時間が先月より2分伸びました」

【常同行動検出】
  加速度データの周波数分析（FFT）→ 規則的な繰り返しパターン検出
  → ASD児の常同行動（手の振り、体の揺れ）の頻度・強度を定量化
  → 行動データとして保存。将来の行動分析・研究データに活用
```

### 12.4 設計原則

```
1. MVPでは「取得して保存する」に徹する
   分析アルゴリズムは後から載せればよい。
   今は使わなくても、将来分析するためのデータを最初から取っておく。
   取らなかったデータは後から取り直せない。

2. バッテリー消費を最小化
   ジャイロ/加速度: 10Hz（100ms間隔）で十分。高頻度計測は不要。
   マイク: dB値のみ取得（音声録音しない）。1秒間隔。

3. プライバシーファースト
   タッチ/モーション: 追加同意不要（既存の行動データ同意でカバー）
   マイク: dB値のみなら追加同意不要。音声分析は要追加同意。
   カメラ: 要追加同意 + オンオフ切替必須。
```

## 13. 設計上の確定事項（2026-04-05 議論結果）

### 13.1 θの測定・調整の統合（循環依存対策）

```
【確定】1つのθで能力測定と難度調整の両方を行う。

分離（測定用θと調整用θを別にする）は不要。
IRTモデルは難度パラメータbが既知ならθを正しく推定できる設計。

ただしbの精度がMVP初期は低いため:
  - bの初期値を保守的に設定（過大推定方向に寄せる）
  - ベースライン確立前のNCI非表示（既存設計のまま）
  - 信頼区間が広いうちは「計測中」表記を残す
  - 校正データ蓄積でbの精度が上がれば循環バイアスは自然解消
```

### 13.2 DDM: MVP簡易版 → Phase 1フルモデル

```
【確定】MVPでは簡易版を使用。

MVP（簡易版）:
  cognitive_RT = total_RT - motor_baseline
  精度: r ≈ 0.75-0.80（定型発達児のDDMフルモデルとの相関）
  影響: 運動障害が重い子（ASD児の20-30%）で誤差が大きくなる可能性
  実装コスト: 低（数時間）

Phase 1（フルDDM）:
  DDMのv（ドリフト率）を直接推定
  精度: r ≈ 0.90-0.95
  実装コスト: 高（数週間）

データ保存方針:
  タッチの生データ（タイムスタンプ、座標、圧力）はDay1から全保存
  → DDMフルモデル導入時に過去データも遡及計算可能
```

### 13.3 NCI 4軸の独立性

```
【確定】MVPでは4軸を独立に推定する。

軸間の相関（例: NCI-Mが低い子はNCI-Aも低くなりやすい）は存在するが、
相関補正は実データ蓄積後のPhase 2で対応する。

理由:
  - 相関構造はASD+知的障害児の実データなしには推定困難
  - 文献値（定型発達児ベース）を使うとバイアスが乗るリスク
  - MVPで独立推定 → Phase 2で実測相関を使って補正する方が正確
```

### 13.4 保護者向けスコア: 絶対評価（θのみ）

```
【確定】保護者・施設向けスコアはθ（能力値）のみで構成する。

混合スコア（エンゲージメント・自立度等を混ぜた加工値）は廃止。
詳細は6節を参照。
```
