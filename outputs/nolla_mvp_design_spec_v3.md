# Nolla MVP 設計仕様書 v3.0（完全版）

**作成日**: 2026-04-03
**対象**: ASD + 軽度〜重度知的障害児（3-18歳）、日本語
**根拠**: 40件以上の査読済み論文 + ASD人気ゲーム分析 + 全フィードバック反映
**重要**: このドキュメントを読まずに開発を開始するな。全セクションを通読してから作業に入ること。

## 0. 最重要コンセプト

### 「療育アプリ」を作るな。「ASD児がハマるゲーム」を作れ。

療育効果はゲーム設計の裏側で実現する。子どもが見る画面は**Minecraft・どうぶつの森レベルに楽しいゲーム**でなければならない。安っぽい療育ツールは子どもが触らない。触らなければ効果はゼロ。

### 画面の向き: 横向き（ランドスケープ）固定
- 子ども向けゲーム・療育アプリは全て横型が業界標準（Otsimo、Jade ND、全知育アプリ）
- フロントカメラで眼球データ取得するため、横持ちが適切（iPadはカメラが上辺中央）
- 縦にするな

## 1. デザインの方向性（最も重要なセクション）

### 1A. なぜMinecraft・どうぶつの森・Robloxがベンチマークなのか

この3つはASD児に世界的に最も人気のあるゲーム。科学的に分析された理由:

**Minecraft:**
- ルール不変・物理法則一定 → 予測可能性（ASD児の最重要ニーズ）
- ピクセルアート・ブロック型 → 幾何学的シンプルさ（ASD児は14ヶ月から幾何学的刺激に親和性: JAMA Psychiatry）
- 建築自由 → 自分の世界を作れる（最大の動機付け要因: GamesLearningSociety）
- 自分のペース → 時間制限なし、競争なし
- 抑制型ブロック色 → 感覚安全性

**どうぶつの森:**
- 日課固定 → ルーティンの満足（ASD児はルーティンを愛する）
- パステルカラー・丸みを帯びたデフォルメ → 視覚的安全性
- 島カスタマイズ → 自分の世界構築
- 家具・虫・魚コレクション → 収集行動の満足（ASD児の25%: Springer 2024）
- 勝ち負けなし → 非競争

**Roblox（Adopt Me!/Brookhaven）:**
- 想像力を表現できるカスタマイズ環境
- NPCとの単純な関係性（複雑な社交なし）
- メタバース型ソーシャル（直接対面の不安なし）

### 1B. ビジュアルデザインの具体指針

#### カラーパレット
```
ベースカラー:
  背景:      #F5F5F5 (ライトグレー) — 白すぎず目に優しい
  プライマリ: #4A7BA7 (カームブルー) — どうぶつの森の空・海
  セカンダリ: #6BA47A (ソフトグリーン) — どうぶつの森の草原
  アクセント: #D4A574 (ウォームベージュ) — 木の温かみ
  テキスト:   #333333 (ダークグレー) — 親向け画面のみ
  成功:      #6BA47A + チェックマーク

ゲーム世界カラー:
  ベースの上に、くすんだ（低彩度の）カラフルさを実現する。
  Minecraftの土・石・木のブロック色 + どうぶつの森のパステルフルーツ色を参考に、
  単色ではなくバリエーションのある世界観を作る。
  
  例: ゲーム背景は草原テクスチャ（#8FBC8F系のくすんだグリーン）
      箱やカードには木目テクスチャ（#D2B48C系のウォームブラウン）
      報酬スターは柔らかい金色（#DAA520の低彩度版）

絶対禁止色: 蛍光色のみ（ASD児の85%が色を高強度で知覚）
  ※ 赤・黄等の色自体は禁止ではない。「彩度x面積x動き」で判断
  ※ 詳細は nolla_color_regulation.md 参照
```

#### 背景デザイン
```
NG: 白背景、単色背景、無地
OK: 世界観のある背景

具体例:
  ホーム画面: くすんだ緑の草原 + 柔らかい青空 + 遠くに丸い木（どうぶつの森風）
  ゲーム画面: テーマに合った背景（なかまわけ→森の中、おぼえて→星空等）
  マイルーム: 木の壁と床のテクスチャ（温かみのある室内）
  
背景のルール:
  - テクスチャ・影・奥行きを持たせる（フラットにしない）
  - ただし情報量は抑える（背景は静的、動く要素はゲーム部分だけ）
  - くすんだトーン（彩度を抑える）
```

#### UIコンポーネントのデザイン
```
NG: フラットなボタン、影なし、テクスチャなし → 安っぽい療育アプリに見える
OK: Minecraftのブロック感 + どうぶつの森の丸みを融合

具体例:
  ボタン:
    - 角丸（border-radius: 16-24px）
    - 柔らかい影（box-shadow with low opacity）
    - 木目やブロックのテクスチャを薄く適用
    - ホバー/タップ時に少し拡大（scale: 1.05）+ 短い効果音
    - 最小サイズ: 48×48px（3-8歳は64×64px）

  カード:
    - 角丸 + 薄い影 + 境界線
    - 内部にイラスト（写真ではなく）
    - 選択可能なカードは光るエフェクト

  ゲーム内の「箱」（なかまわけ等）:
    - 木箱のテクスチャ（Minecraft的）
    - 分類先の色はくすんだトーン
    - タップ可能な箱は柔らかく脈動（pulse animation、ゆっくり）
```

#### キャラクターデザイン
```
方向性: 幾何学的デフォルメ

Minecraftの影響:
  - 対称的、シンプル、ブロック的
  - 複雑な表情筋なし、単純な目と口
  - 認知負荷が低い → ASD児が安心

どうぶつの森の影響:
  - 丸みを帯びた柔らかいシルエット
  - 愛らしいがミニマルな表情
  - 同じキャラが同じ行動を繰り返す → 予測可能

Nollaのキャラ設計:
  - ナビキャラ1体: アプリ全体を案内。デフォルメ動物（丸い形状、大きな目、シンプルな口）
  - 感情表現: 笑顔、中立、困り顔の3段階のみ
  - リアル禁止: 写実的な人間の顔はNG（不気味の谷）
  - ビデオモデリング用キャラ: AI生成の同年齢風キャラ（リップシンク付き）
  
  ユーザーの「好きなもの」をマイルームに組み込む:
  - 電車好き → 電車アイテムが報酬ショップに
  - 数字好き → 数字ブロックの装飾
  - 動物好き → ペットアイテム
```

### 1C. 感覚オーバーロード防止ルール

| 要素 | NG | OK |
|------|-----|-----|
| アニメーション | 自動再生、ストロボ、フラッシュ | ユーザートリガー、200ms以上のスロー |
| 音声 | 突然の大きな音、BGM自動再生 | 調整可能な音量、穏やかな効果音 |
| 情報密度 | 画面いっぱいの要素 | 1画面5要素以下、十分な余白 |
| 形状 | シャープなコーナー、複雑な形 | 丸角（16-24px）、シンプル形状 |
| 色 | 蛍光色 | くすんだパステル推奨。赤・黄等は「彩度x面積x動き」で判断 |
| 遷移 | 急な画面切り替え | フェード（300-500ms）で穏やかに |

### 1D. 文字なしUI設計

ゲーム内・ナビゲーションに文字を使わない。知的障害があると文字認識ができない。

```
意味の伝え方:
  アイコン + 色 + 形 + アニメーション + 音声の組み合わせ

成功の表現:
  緑チェックマーク + ポップアップ拡大（200-400ms）+ 「ピンポン♪」+ 短い振動

ナビゲーション:
  ホーム = 家アイコン
  ゲーム = 各ゲーム固有のアイコン（色と形で識別）
  設定 = ギアアイコン（親向け画面のみ）
  戻る = ← 矢印

チュートリアル:
  文字を一切使わない。以下の4段階で教える:
  Step 1: 対象が発光（1秒）→ 周辺が暗くなる（視線誘導）
  Step 2: ハンドジェスチャーアニメーション（タップを示す）
  Step 3: ユーザーが操作 → 成功フィードバック
  Step 4: 2-3回繰り返し、毎回ガイドを薄くする → 4回目でガイドなし（独立実行）
```

## 2. 初回アセスメント（オンボーディング）

### 2A. 保護者への質問（2分以内、10-12問）

#### Step 1: 基本情報（3問）
- お子さまの年齢（スライダー: 3-18歳）
- 性別（男/女/回答しない）
- 「あてはまることを選んでください（複数選択OK）」— 診断名は聞かない
  - よく同じ動作を繰り返す / 音や光に敏感 / 新しい環境で不安 / 興味が限定的 / 指示の理解が難しい時がある

#### Step 2: コミュニケーション能力（4問）
- 発語レベル: 発語なし / 単語のみ / 二語文 / 短い文章 / 会話できる
- 理解力: 簡単な指示 / 文で理解 / 文脈で理解 / よくわからない
- AAC使用経験: 使用していない / 試したことがある / 毎日使用
- 使用中のツール名（任意）

#### Step 3: 療育と好み（3問）
- 受けている支援（複数選択）
- 日常の困りごと（最大3つ）
- 好きなもの・興味（複数選択: 電車 / 数字 / 動物 / 音楽 / 回転するもの / その他）→ マイルームのアイテム出し分けに使用

※「アプリに期待すること」は聞かない（質問削除でサインアップ率+40%: Zigpoll調査）

#### 法的対応
- 障害情報 = 要配慮個人情報 → 明示的同意必須（個人情報保護法）

### 2B. 子ども向け実技アセスメント（ゲーム形式、時間制限なし）

**時間制限は一切設けない。** ASD児は精度重視で応答時間が長い。制限は真の能力を過小評価する（PMC Sensory Processing研究）。各ミニゲームは5-8試行で自動完了（時間ではなく試行数で区切る）。

| ミニゲーム | 測定項目 | 指示方法 |
|-----------|---------|---------|
| タップチャレンジ | タップ反応速度・精度 | 光るボタンをタップ（視覚のみ） |
| おなじものどーれ | 視覚的マッチング能力 | 同じ絵を見つけてタップ（視覚のみ） |
| おとをきいて | 聴覚理解力 | 「りんごはどれ？」→ 絵を選ぶ |
| おぼえてね | ワーキングメモリ | 光った順番にタップ（視覚のみ） |

- 全てゲームの世界観内で実施（「はじめての冒険」）
- 即座報酬あり、ネガティブフィードバックなし
- 中断OK

### 2C. 名前の収集（初回セッション後）

オンボーディングでは聞かない。初回セッション完了後に:
```
「たのしかった？ もっとたのしくするために、おなまえをおしえてね！」
[ニックネーム入力欄]  [あとで]
```
入力後はホーム画面で「〇〇ちゃん、きょうもあそぼう！」と表示。
根拠: 名前の個人化は感情的絆を向上（European Journal of Psychology of Education）

### 2D. パーソナライゼーション
```
発語レベル → コンテンツ比率
├ 発語なし    → AAC 50% / 認知ゲーム 40% / 音声模倣 10%
├ 単語のみ    → AAC 30% / 認知ゲーム 40% / 語彙拡張 30%
├ 二語文〜    → AAC 10% / 認知ゲーム 40% / 構音練習 50%
└ 会話できる  → 認知ゲーム 50% / 構音練習 30% / 応用会話 20%

実技結果 → ゲーム難度初期値
├ 成功率 0-30%   → Level 1（選択肢2つ）
├ 成功率 31-60%  → Level 2（選択肢3つ）
├ 成功率 61-80%  → Level 3（選択肢4つ）
└ 成功率 81%+    → Level 4（複合タスク）
```

## 3. 認知機能強化ゲーム（4種類）

**注意（2026-04-05更新）**: MVP搭載ゲームは4種類に確定。旧「みつけてタッチ」（マッチ3パズル）は不採用。代わりに視覚探索（間違い探し型）とCorsi Block（光るブロック記憶）を採用。確定事項の詳細は `.claude/projects/.../memory/project_game_design_decisions.md` を参照。

### 確定4ゲーム

| # | ゲーム | 認知機能 | 建物 | エビデンス |
|---|---|---|---|---|
| 1 | 神経衰弱（メモリマッチング） | ワーキングメモリ | カードスート城 | ASD適合4.8/5, d=0.64 |
| 2 | 分類ソーティング | 認知的柔軟性+抑制制御 | ソーティング小屋 | d=0.62 |
| 3 | 視覚探索（間違い探し型） | 選択的注意 | 虫めがねタワー | g=1.2 |
| 4 | Corsi Block（光るブロック記憶） | 空間ワーキングメモリ | 光るグリッド塔 | d=0.61 |

### 選定根拠（参考）

| 順位 | 認知機能 | Effect Size | 根拠 |
|------|---------|------------|------|
| 1位 | 認知的柔軟性 | 中〜大 (g>0.50) | メタ分析59研究n=2,122 |
| 2位 | ワーキングメモリ | 中 (g=0.63-0.64) | CogMed RCT |
| 3位 | 注意制御 | 小〜中 | CPAT研究 |
| 4位 | 空間ワーキングメモリ | 中 (d=0.61) | Corsi Block RCT |

### 共通仕様

- **セッション時間制限なし**: 連続60分で「休憩しませんか？」と提案（強制ではない）。保護者が設定で上限変更可能。デフォルトは制限なし
- **適応型難度調整**: 成功率75-80%を自動維持（直近10試行の成功率50% + 反応時間25% + エラーパターン15% + 疲労度10%）
- **強化スケジュール**: 初期CRF（全正解に報酬）→ 習得後VR3-5
- **フィードバック**: 200ms以内、ビジュアル+音声+振動
- **チュートリアル**: アニメーション主導、文字なし、4段階削減法
- **エラー処理**: エラーレス学習（サイレント修正→ヒント→自動正解実行。エラーはスコアに影響しない）

### 報酬システム: 報酬ショップ + マイルーム

```
ゲームプレイ → スター獲得
  ↓
報酬ショップ（スターとアイテムを交換 = トークンエコノミー + 等価交換学習）
  ├ 小アイテム: スター3個（机、椅子、花等）
  ├ 中アイテム: スター10個（ベッド、テレビ、ペット等）
  └ 大アイテム: スター30個（特別な家具、乗り物等）
  ↓
マイルーム（どうぶつの森の部屋のような温かみのある空間）
  ├ アイテムを自由に配置（自分の世界を作る）
  ├ 壁紙・床を変更
  └ 図鑑で一覧表示（コレクション要素）

ショップのアイテムは子どもの「好きなもの」に応じて変える:
  電車好き → 電車模型、線路
  動物好き → ペット、動物のぬいぐるみ
  数字好き → 数字ブロック
```

根拠:
- トークンエコノミーはABA療育の確立された手法
- 「自分の世界構築」はMinecraft/どうぶつの森がASD児に人気な最大の理由
- コレクション行動はASD児の25%に見られる健全な特別関心

### ショップ・マイルームの画面デザイン
```
ショップ:
  - どうぶつの森の「たぬきち商店」的な温かみのある木造の店内
  - アイテムは棚に並ぶ（グリッド表示ではなく、世界観のある陳列）
  - 購入可能なアイテムは柔らかく光る
  - 所持スター数を常時表示

マイルーム:
  - どうぶつの森の自室のような俯瞰視点
  - 木の壁・木の床のテクスチャ（温かみ）
  - アイテムをドラッグで配置
  - 配置時にポトッという効果音
```

### Game 1: 「なかまわけ」（認知的柔軟性）

**鍛えられる力**: ルールを理解して切り替える力
→ 日常例: 遊園地で順番を忍耐強く待てるようになる。お着替えの手順を場面に応じてできるようになる

#### ゲーム画面のデザイン
```
背景: 森の中（くすんだグリーンの木々、柔らかい木漏れ日）
箱: 木箱テクスチャ（Minecraft的なブロック感）、各箱に色/形のシンボル
アイテム: デフォルメされた果物・動物・乗り物のイラスト
ルールカード: 画面上部に木の看板風のフレームで表示
```

#### ゲームフロー
```
1. 画面上部にルールカード（木の看板風: 色アイコン = 「いろでわける」）
2. 画面中央にアイテム1つ（デフォルメイラスト）
3. 画面下部に2-3つの木箱
4. 子どもが正しい箱にドラッグ or タップ
5. 正解 → 箱が開いてアイテムが入る + 「ピンポン♪」+ スター飛び出す
6. 不正解 → 正解の箱が柔らかく光る（サイレント修正）
```

#### 難度レベル

| Level | ルール | 選択肢 | ルール切替 |
|-------|--------|--------|-----------|
| 1 | 色だけ（赤/青） | 2つの箱 | なし |
| 2 | 色（赤/青/黄） | 3つの箱 | なし |
| 3 | 形だけ（丸/四角/三角） | 3つの箱 | なし |
| 4 | 色→形に切り替え | 3つの箱 | あり（5-10秒の静的予告） |
| 5 | カテゴリ（食べ物/動物/乗り物） | 3つの箱 | あり（5-10秒の静的予告） |

Level 1-3: ルール切り替えなし（難度は選択肢の数・複雑さのみ）
Level 4+: 5-10秒の静的予告画面 + 新ルールで2試行の練習（エラーレス）

### Game 2: 神経衰弱・メモリマッチング（ワーキングメモリ）— 旧名「おぼえてタッチ」

**注意（2026-04-05更新）**: 確定版では「神経衰弱（メモリマッチング）」として採用。下記のシーケンス記憶方式ではなく、カードペアマッチング方式に変更。建物は「カードスート城」。詳細メカニクスはPhase 0-D-1で策定。

**鍛えられる力**: さっき見たものの位置を覚えておく力
→ 日常例: お着替えの順番を覚えて自分でできる。「りんごとパンを持ってきて」と言われて両方覚えていられる

#### ゲーム画面のデザイン
```
背景: 夜空（くすんだネイビー、柔らかい星）
カード: 丸角の木のカード、裏面に星の模様
動物イラスト: デフォルメされた可愛い動物（どうぶつの森風）
光るエフェクト: カードが光る時は柔らかい金色のグロー
```

#### ゲームフロー
```
1. 画面にカード（動物の絵）が2-6枚並ぶ
2. カードが順番に1枚ずつ柔らかく光る（1枚あたり1-2秒）
3. 全部光り終わったら ナビキャラが「やってみよう！」のジェスチャー
4. 子どもが光った順番にカードをタップ
5. 正解 → 動物が動く + 「すごい！」+ スター
6. 不正解 → もう一度光る（ヒント）→ 再挑戦
```

#### エラー処理
```
1回目: 正解カードが薄く光る
2回目: 正解カードが明確に光る + ハンドアニメーション
3回目: 自動で正解実行 → 次の試行へ
※ エラー回数はスコアに影響しない
```

### Game 3: 視覚探索・間違い探し型（選択的注意）— 旧名「みつけてタッチ」

**注意（2026-04-05更新）**: 確定版では「視覚探索（間違い探し型）」として採用。下記の草原探索方式をベースに、間違い探し要素を強化。建物は「虫めがねタワー」。詳細メカニクスはPhase 0-D-1で策定。

**鍛えられる力**: たくさんの中から目的のものを見つける力
→ 日常例: おもちゃ箱から欲しいものを見つけられる。スーパーで好きなお菓子を棚から見つけられる

#### ゲーム画面のデザイン
```
背景: 草原（くすんだグリーン、遠くに丸い木、どうぶつの森風）
ターゲット表示: 画面上部に木のフレームでターゲットアイコン
アイテム: 草原に散らばったデフォルメアイテム（花、虫、果物等）
発見エフェクト: タップするとキラキラ + アイテムが浮き上がる
```

#### ゲームフロー
```
1. 画面上部にターゲット表示（木のフレーム内）
2. 画面にアイテムが散在（ターゲット + ディストラクタ）
3. 子どもがターゲットをタップ
4. 正解 → キラキラ + アイテムが浮き上がる + スター
5. 不正解 → ディストラクタが薄くなる（ヒント）
6. 10秒以上操作なし → ターゲットが少し揺れる（ヒント）
```

### Game 4: Corsi Block・光るブロック記憶（空間ワーキングメモリ）— 新規追加

**追加日（2026-04-05）**: エビデンスベースで新規追加。建物は「光るグリッド塔」。d=0.61。

**鍛えられる力**: 光った順番を覚えて再現する力（空間ワーキングメモリ）
→ 日常例: 道順を覚えられる。部屋の中でものの位置を覚えていられる

**基本メカニクス**:
- ブロックが順番に光る → 子どもが同じ順番でタップして再現
- 正解: 視覚効果 + 振動フィードバック
- 不正解: もう一度光る（ヒント）→ 再挑戦
- レベルに応じてシーケンス長・グリッドサイズが変化

**詳細メカニクス・画面デザイン**: Phase 0-D-1で策定予定

## 4. ST系コンテンツ（Phase 2）

### 4A. AAC（発語なし〜単語レベル）

#### 画面デザイン
```
背景: 温かみのある室内（木の壁、柔らかい照明）
シンボル: デフォルメイラスト（写真ではなく）
カテゴリボタン: 大きな丸角ボタン（64×64px以上）、各カテゴリに固有の色（くすんだトーン）
メッセージバー: 画面上部、木のフレーム内に選択シンボルが並ぶ
```

#### 構成
- コア語彙50語（日常の80%をカバー）
- カテゴリ: たべもの / きもち / どうさ / ばしょ / ひと / もっと
- 音声出力: AI TTS（子どもの声）、タップ→100ms以内に再生
- 複数シンボル → 文として読み上げ（「りんご」+「ほしい」→「りんごがほしい」）
- カスタマイズ: 保護者がシンボル追加・非表示設定可能

#### 段階設計
```
Stage 1: 1シンボルタップ → 音声再生
Stage 2: 2つから1つを選ぶ
Stage 3: 2シンボルで文を作る
Stage 4: 3シンボル以上の文構築
```

### 4B. 音声模倣・発語促進

#### ビデオモデリング
```
1. AI生成キャラクター（同年齢風、リップシンク付き）が画面に表示
2. キャラが口を動かして音を出す（母音から開始）
3. マイクアイコンが光る → 子どもが真似
4. 音声認識ON: AIが判定 → 「すごい！」
   音声認識OFF: 親が「できた」ボタン → 「すごい！」
5. スター + キャラが拍手
```

#### 音声認識の仕様
- API: SpeechSuper（日本語対応、音素レベル判定）
- ON/OFF切替可能（設定画面にトグル）
- OFF時: 親タップ方式に自動切替（子どもの体験は同じ）
- 初回表示で注意書き:
```
「音声認識機能について」
お子さまの声を聞いて、発音をチェックする機能です。
発話の特性によっては正しく認識できない場合があります。
うまく機能しない場合は、設定からOFFにできます。
     [OK、使ってみる]    [OFFにする]
```

### 4C. 構音練習（発語あり）

対象音は日本語の構音発達順:
Level 1: 母音 → Level 2: 口唇音（ま、ぱ、ば）→ Level 3: 歯茎音 → Level 4: 軟口蓋音 → Level 5: 摩擦音 → Level 6: 弾き音
